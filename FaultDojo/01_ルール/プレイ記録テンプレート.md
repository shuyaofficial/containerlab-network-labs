# プレイ記録テンプレート

<!-- プレイのたびに本ファイルを複製して使う。
     ファイル名: プレイ記録_YYYY-MM-DD_シナリオNN.md （例: プレイ記録_2026-07-06_シナリオ01.md）
     複製先はプレイヤーが管理しやすい場所でよい（本フォルダ配下に置く場合はサブフォルダを切ること）。

     過去の教訓（Obsidian Mistakes.md）に基づき、以下の順序を必ず守ること:
     1. ライブ状態を先に採取する（設定ファイルを読む前に、まず実機の今の状態を見る）
     2. 層別（L2→L3→プロトコル→アプリ）に切り分ける
     3. 単一原因を決め打ちせず、複数の独立原因を並行して疑う -->

## 4. 層別切り分け表

<!-- L2到達 → VLAN存在 → SVI Up → L3到達 → プロトコル隣接 の順で埋める。

| 層 | 確認項目 | コマンド | 期待 | 実測 | 判定 |

| L2 | SVI Up | `show ip interface brief \| include Vlan` | up/up | | ⬜ |
| L3 | IF疎通 | `ping <対向IP>` | 成功 | | ⬜ |
| プロトコル | 隣接確立 | `show ip ospf neighbor` 等 | FULL/UP/Established | | ⬜ |

| アプリ/E2E | エンドツーエンド疎通 | `ping` / `traceroute` | 成功 | | ⬜ |
