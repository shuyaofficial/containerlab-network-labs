# ベースライン構築手順

NW Fault Dojoのベースライン（障害注入前の正常状態）を構築する手順です。土台はテーマ21キャプストンのトポロジ（`clab-capstone-*`）で、既存テーマのファイルは変更せず参照のみ行います。

## 0. 前提

- テーマ21のトポロジファイル・スクリプト（`capstone.clab.yml` / `configure_all.sh` 等）は一切変更しません。参照のみです。
- 本手順はプレイヤー本人が実行します。
- 実行場所はOrbStackのclab VMです。接続方法は [../00_ログイン/ログインコマンド.md](../00_ログイン/ログインコマンド.md) を参照してください。

## 1. ラボのデプロイ

```bash
ssh clab@orb
cd /Users/shuya/Documents/claude/Mac仮想環境構築/21_cisco_capstone
sudo containerlab deploy -t capstone.clab.yml
```

テーマ21のREADME（`README_Lab_Challenge.md`）記載の、実証済みのデプロイ方式です。

## 2. IOL起動待ち

対象ノード（isp / hq-edge1 / hq-edge2 / hq-core1 / hq-core2）はx86エミュレーションのCisco IOLです。6ノードの起動には**10〜15分程度**を見込んでください。

## 3. ベースライン設定の投入

```bash
sudo bash configure_all.sh
```

テーマ21に同梱のスクリプトです。VPN（IPsec/GRE）以外の全設定（NAT・OSPF・BGP・HSRP等）を投入し、各機器で `write memory` まで実行します。

## 4. VPN設定の復元

`configure_all.sh` はVPN（IPsec VTI + IKEv2 + OSPF）を含みません。ここでは、テーマ21のパラメータシート（`03_パラメータシート_セキュリティ・NAT編.md`）および構築記録（`05_ipsec_nat_troubleshooting_log.md`）と整合する内容で、同等機能のVPNを再構成します。**IKEv2の内部名称（proposal/policy/keyring/profile名）は当時の構築記録に残っていないため、今回同等機能で命名し直しています。初回検証プレイで実機確認します。**

### 4.1 hq-edge1

```text
conf t
crypto ikev2 proposal IKEV2_PROP
 encryption aes-cbc-256
 integrity sha256
 group 14
crypto ikev2 policy IKEV2_POL
 proposal IKEV2_PROP
crypto ikev2 keyring IKEV2_KEY
 peer 200.0.3.1
  address 200.0.3.1
  pre-shared-key CiscoSecretKey123!
crypto ikev2 profile IKEV2_PROF
 match identity remote address 200.0.3.1 255.255.255.255
 authentication remote pre-share
 authentication local pre-share
 keyring local IKEV2_KEY
crypto ipsec transform-set TS_AES256_SHA256 esp-aes 256 esp-sha256-hmac
 mode tunnel
crypto ipsec profile IPSEC_PROF
 set transform-set TS_AES256_SHA256
 set ikev2-profile IKEV2_PROF
interface Tunnel0
 ip address 192.168.100.1 255.255.255.252
 ip ospf 1 area 0
 tunnel source Ethernet0/1
 tunnel destination 200.0.3.1
 tunnel mode ipsec ipv4
 tunnel protection ipsec profile IPSEC_PROF
 no shutdown
end
write memory
```

### 4.2 br-edge

hq-edge1の鏡像設定です。トンネル終端・keyring peer・宛先が入れ替わります。

```text
conf t
crypto ikev2 proposal IKEV2_PROP
 encryption aes-cbc-256
 integrity sha256
 group 14
crypto ikev2 policy IKEV2_POL
 proposal IKEV2_PROP
crypto ikev2 keyring IKEV2_KEY
 peer 200.0.1.1
  address 200.0.1.1
  pre-shared-key CiscoSecretKey123!
crypto ikev2 profile IKEV2_PROF
 match identity remote address 200.0.1.1 255.255.255.255
 authentication remote pre-share
 authentication local pre-share
 keyring local IKEV2_KEY
crypto ipsec transform-set TS_AES256_SHA256 esp-aes 256 esp-sha256-hmac
 mode tunnel
crypto ipsec profile IPSEC_PROF
 set transform-set TS_AES256_SHA256
 set ikev2-profile IKEV2_PROF
interface Tunnel0
 ip address 192.168.100.2 255.255.255.252
 ip ospf 1 area 0
 tunnel source Ethernet0/1
 tunnel destination 200.0.1.1
 tunnel mode ipsec ipv4
 tunnel protection ipsec profile IPSEC_PROF
 no shutdown
interface Ethernet0/2
 ip ospf 1 area 0
end
write memory
```

`interface Ethernet0/2`（支社LAN側）に `ip ospf 1 area 0` を追加することで、支社LAN（10.2.40.0/24）がOSPFで広報されます。

## 5. ベースライン健全性チェック

プレイ開始前（シナリオ選択の前）に、毎回以下を確認します。

| No. | 確認項目 | 実行ノード | コマンド | 期待結果 |
|---|---|---|---|---|
| 1 | IPsecセッション | hq-edge1 / br-edge | `show crypto session` | UP-ACTIVE |
| 2 | OSPFネイバー | hq-edge1 | `show ip ospf neighbor` | core1/core2/br-edge(Tunnel0) すべてFULL |
| 3 | 本社側疎通 | hq-pc-sales | `ping 8.8.8.8` | 成功（NAT経由） |
| 4 | 拠点間疎通 | br-pc | `ping 8.8.8.8` および `ping 10.1.10.100` | 両方成功（後者はVPN経由） |
| 5 | HSRP状態 | hq-core1 / hq-core2 | `show standby brief` | core1=Active / core2=Standby |

## 6. 注意事項

- **プレイ中（シナリオ注入後）の `write memory` は禁止です。** 障害注入はrunning-configのみを対象とするため、`write memory` をしてしまうと切り戻しの安全網（コンテナ再起動によるstartup-config復帰）が機能しなくなります。
- VPN設定は、テーマ21のパラメータシートと同等機能での再構成であり、内部名称の一致は保証されません。初回検証プレイで実機確認します。
- テーマ21（`21_cisco_capstone/`）配下のファイルは参照のみとし、変更しません。
