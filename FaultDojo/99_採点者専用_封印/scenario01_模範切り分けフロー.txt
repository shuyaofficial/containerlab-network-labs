== 採点者専用・封印 ==
プレイヤーは開かないこと。開いた場合、このシナリオのプレイは無効となる。

対応シナリオ: シナリオ01 初級「営業部PCがゲートウェイに届かない」
フォルダ名: シナリオ01_初級_営業部PC疎通不可

----------------------------------------------------------------------
注入内容（平文）
----------------------------------------------------------------------

対象機器: hq-core1 のみ

hq-core1:
conf t
interface e1/3
 switchport access vlan 99
end

※ VLAN99はDBに存在しない。IOSのバージョンによっては自動作成されるが、どちらでもPCはVLAN10から隔離され症状は同一。

----------------------------------------------------------------------
切り戻し（平文）
----------------------------------------------------------------------

hq-core1:
conf t
interface e1/3
 switchport access vlan 10
no vlan 99
end

（自動作成された場合のVLAN99は `no vlan 99` で掃除する）

----------------------------------------------------------------------
症状全容マトリクス
----------------------------------------------------------------------

- 提示する初期観測: 「営業部PC(10.1.10.100)からデフォルトGW(10.1.10.254)へのpingが100%失敗する。PCのリンクはUP」
- 全容: PC→GW失敗 / PC→どこへも到達不可 / core1のe1/3はup/up / HSRPはcore1 Active(正常) / SVI正常 / WAN・支社・VPNすべて正常

----------------------------------------------------------------------
模範切り分けフロー
----------------------------------------------------------------------

1. PC側事実採取: `ip addr`/`ip route`/`ip neigh`（ARP: 10.1.10.254 が FAILED/incomplete → L2疑い）
2. core1: `show interfaces status`（e1/3 connected だが VLAN列が99）← 最短ここで確定
3. 裏取り: `show vlan brief`（e1/3がVLAN10メンバーに不在）／`show mac address-table vlan 10`（PCのMAC不在）
4. 反証（正常証明）: `show standby brief`（Active正常）／`show ip interface brief | include Vlan`（SVI up/up）→ L3以上は無罪
5. 根本原因宣言: 「e1/3のaccess VLANが10以外に変更されている（L2・ポートVLAN帰属の障害）」

----------------------------------------------------------------------
ヒント（3段階・全文）
----------------------------------------------------------------------

1. 方角: 「障害は本社・営業部セグメント内で完結している。WAN・支社・エッジルーターは無関係」
2. 層: 「L3(SVI/HSRP)より下。アクセスポートのVLAN帰属を疑え」
3. 観測ポイント: 「hq-core1で show interfaces status と show vlan brief を突き合わせ、e1/3の所属VLANを確認せよ」

----------------------------------------------------------------------
採点重みとアンカー（A25/B20/C20/D20/E15）
----------------------------------------------------------------------

- A満点条件: PC側L1/L2確認→スイッチL2→(必要なら)L3の順。いきなり `show run` を読んだら-10（ライブ優先違反はBでも減点）
- B: ライブ状態(show系)を設定ファイル・記憶より先に採取したか
- C満点条件: HSRP/SVIの正常性をコマンドで証明して除外
- D: 単一原因シナリオだが「他に原因が残っていないか」の最終確認（症状全消失の確認）で判定
- E: 症状と無関係な機器・コマンドの乱発がないか
