# NW Fault Dojo — 障害切り分け道場

テーマ21キャプストン（`clab-capstone-*`）のトポロジを土台に、AIゲームマスター（GM）が障害シナリオを出題し、プレイヤー（あなた）が実機で切り分けを行う訓練道場です。

## 1. 道場のコンセプト

- **出題・ヒント・採点はAIゲームマスターが行います。** シナリオの選定、3段階ヒントの提示、プレイ記録の確認、スコアリング、AAR（振り返り）の進行を担当します。
- **構築・コマンド投入はすべてプレイヤー本人が行います。** GMは実機に一切触れません。GMに見せるのは、あなたが実機で採取した出力です。
- **GMは修復方法の答えを絶対に言いません。** 修復コマンド・設定変更内容・根本原因そのものは、プレイ中もプレイ後の解説でも開示しません。詳細な禁止事項は [01_ルール/ゲームマスタープロトコル.md](01_ルール/ゲームマスタープロトコル.md) を参照してください。

5. [02_ベースライン/ベースライン構築手順.md](02_ベースライン/ベースライン構築手順.md) — ラボのデプロイと健全性チェック
6. [02_ベースライン/切り戻し・完全リセット手順.md](02_ベースライン/切り戻し・完全リセット手順.md) — プレイ後の復旧・リセット

## 3. プレイの全体フロー

1. **ベースライン構築** — [02_ベースライン/ベースライン構築手順.md](02_ベースライン/ベースライン構築手順.md) に従い、ラボをデプロイして全設定を投入する（プレイヤー実行）。
2. **健全性チェック** — ベースラインが正常であることをチェックリストで確認する（プレイヤー実行）。
3. **シナリオ選択** — 下記のシナリオ一覧から挑戦するシナリオを選ぶ（GMと相談可）。
4. **`inject.sh` 実行** — 選んだシナリオフォルダ内の `inject.sh` を **プレイヤー自身が** 実行し、障害を注入する。
5. **切り分けプレイ** — GMとの対話を通じて切り分けを進める。採取した実機の出力をGMに共有しながら、[01_ルール/プレイ記録テンプレート.md](01_ルール/プレイ記録テンプレート.md) を複製したファイルに記入していく。
6. **根本原因宣言** — 「これが根本原因である」とGMに宣言する。
7. **採点** — GMが宣言内容と記録を基に採点する（[01_ルール/採点ルーブリック_共通.md](01_ルール/採点ルーブリック_共通.md) 準拠）。
8. **AAR（振り返り）** — プレイ記録の振り返り欄（予想・実際・次どうする）を記入して1プレイを終える。
9. **切り戻し** — [02_ベースライン/切り戻し・完全リセット手順.md](02_ベースライン/切り戻し・完全リセット手順.md) に従い、ベースラインへ復帰する。

## 5. 採点者専用フォルダについて

本フォルダ直下の `99_採点者専用_封印/`（全シナリオ共通の1フォルダ・`scenario01〜05_模範切り分けフロー.txt`）は **プレイヤー閲覧禁止** です。模範切り分けフロー・注入内容・採点アンカーなど、ネタバレそのものが含まれます。誤って開いた場合、そのシナリオでのプレイは無効となります。GMのみが参照します。
