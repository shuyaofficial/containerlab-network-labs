# 現行機コンフィグ集: 社内LAN更改・移行・移設（移行元スナップショット）

これはテーマ28完了時点の現行環境の設定です。Mission 1でこのまま6台（old-rt / old-core / old-sw1 / old-sw2 / mon-core / cap-sw）へ手動投入し、テーマ28完了時と同じ状態（老朽LAN内部の疎通・監視セグメントの疎通・SNMPv3ポーリング・v2cトラップ受信・SPAN）を移行元として再現します。

本シートはテーマ28のMission5・Mission6完了後の状態を反映しています。**SNMPv2cのcommunity（`public`）はREADOnly設定ごと削除済み**で、ポーリングはSNMPv3（authPriv、group `NMS-GROUP`、user `nmsuser`）のみで行います。トラップ送信のみ引き続きv2c（community `public`）で運用しています（「ポーリングはv3化したが、トラップはv2cのまま」という、テーマ28 Mission6で確認した現実的な混在パターンです）。

> SNMPv3の認証・暗号化パスフレーズ（`SnmpAuth28!` / `SnmpPriv28!`）はテーマ28からの引き継ぎ値です。実運用環境では絶対に使い回さないでください。

## 1. old-rt（老朽WANルータ）
### インターフェース
| IF | IP / モード | 備考 |
|---|---|---|
| Ethernet0/1 | 10.28.1.1/30 | old-core向けWAN transit |
| Loopback0 | 203.0.113.1/32 | 疑似インターネット |

### 設定例
```
hostname old-rt
!
interface Loopback0
 ip address 203.0.113.1 255.255.255.255
!
interface Ethernet0/1
 ip address 10.28.1.1 255.255.255.252
 no shutdown
!
ip route 10.28.0.0 255.255.0.0 10.28.1.2
!
! --- Mission3: SNMPv2c ---
snmp-server location "Server-Room(WAN Router)"
snmp-server contact noc@example.jp
!
! --- Mission5: SNMPv3(authPriv)へ移行 ---
snmp-server group NMS-GROUP v3 priv
snmp-server user nmsuser NMS-GROUP v3 auth sha SnmpAuth28! priv aes 128 SnmpPriv28!
!
! --- Mission6: トラップ ---
snmp-server enable traps snmp linkdown linkup coldstart
snmp-server host 10.28.100.10 version 2c public
```

## 6. cap-sw（Wireshark用SW・SPAN）
### インターフェース
| IF | IP / モード | 備考 |
|---|---|---|
| Ethernet0/1 | アクセス VLAN900 | old-core向け（SPAN source） |
| Ethernet0/2 | アクセス VLAN900 | mon-core向け |
| Ethernet0/3 | VLAN無所属 | cap向け（SPAN destination） |
| VLAN900 SVI | 10.28.0.3/29 | in-band管理 |

### 設定例
```
hostname cap-sw
!
vlan 900
 name MONITORING_TRANSIT
!
interface Ethernet0/1
 switchport access vlan 900
 switchport mode access
 no shutdown
!
interface Ethernet0/2
 switchport access vlan 900
 switchport mode access
 no shutdown
!
interface Ethernet0/3
 switchport mode access
 no shutdown
!
interface Vlan900
 ip address 10.28.0.3 255.255.255.248
 no shutdown
!
ip default-gateway 10.28.0.1
!
! --- Mission2: SPAN(ミラーリング) ---
! E0/1のみをsourceにする理由: インライン構成では通過フレームが必ずE0/1を1回通過するため、
! E0/1・E0/2の双方をsourceにすると同一フレームが2重にミラーされてしまう。
monitor session 1 source interface Ethernet0/1 both
monitor session 1 destination interface Ethernet0/3
!
! --- Mission3: SNMPv2c ---
snmp-server location "Monitoring-Rack"
snmp-server contact noc@example.jp
!
! --- Mission5: SNMPv3(authPriv)へ移行 ---
snmp-server group NMS-GROUP v3 priv
snmp-server user nmsuser NMS-GROUP v3 auth sha SnmpAuth28! priv aes 128 SnmpPriv28!
!
! --- Mission6: トラップ ---
snmp-server enable traps snmp linkdown linkup coldstart
snmp-server host 10.28.100.10 version 2c public
```
