# 要件定義書: 社内LAN更改・移行・移設

## 1. 背景と目的
前テーマ28「監視基盤とSNMP深堀り」では、老朽社内LAN（old-rt/old-core/old-sw1/old-sw2）に監視基盤（mon-core/nms/cap-sw）を後付けし、SNMPポーリング・トラップによって以下の老朽ポイントが可視化された。

- **単一コアのSPOF**: old-core 1台に全VLAN（10/20/30/90/900/901）のSVIとstatic routeが集約されており、old-core停止時に全セグメントが通信不能になる。
- **VLAN20のL2延伸**: 総務部セグメント（VLAN20）がold-sw1・old-sw2の2台にまたがってトランクで延伸されており、本来フロアBで閉じてよいブロードキャストドメインが不要に広がっている。
- **static一辺倒のルーティング**: old-rt⇔old-core間、old-core⇔mon-core間ともにstatic routeのみで構成され、動的ルーティングプロトコルが使用されていない。
- **旧IOS**: old-rt（IOS 15.6.3M3a）・old-core（L2-15.2）ともに旧バージョンのIOSで稼働している。

監視によって「見えていなかった老朽ポイントが見えるようになった」段階であり、可視化しただけでは課題は解決しない。本テーマ29では、これらの老朽ポイントに対して更改判断を行い、テーマ28で構築した監視基盤（NMS/SNMPv3ポーリング/v2cトラップ/SPAN）を判定装置として使いながら、コア・WANルータの冗長化、動的ルーティング化、VLAN構成の是正を計画的に移行する。

実務でも、Catalyst系スイッチのEoL更改、ルータ更改、GWのセグメント移行、LAN配線更改といったパターンは更改案件として頻出する（`ロードマップ/実務案件ログ/C案件.log`参照）。本テーマは、これらの実案件で頻出する更改パターンを教材化したものであり、老朽LANの改修そのものよりも、**計画移行のプロセス（移行方式の比較・段階移行・カットオーバー・切り戻し）を体系的に体験すること**に主眼を置く。

## 2. 更改要件
以下の内容で、老朽構成を更改後構成へ切り替える。

| 項目 | 現行 | 更改後 |
|---|---|---|
| コアSW | old-core 単一（L2-15.2、SPOF） | new-core1/2 冗長ペア＋HSRP＋STPルート（L2-advipservices-2017） |
| WANルータ | old-rt（IOS 15.6.3M3a、Lo0=203.0.113.1） | new-rt（IOS 15.7.3M2、Lo0=203.0.113.2） |
| 各VLANのGW | old-core SVI（.1） | HSRP VIP（.1）＝**IP温存でGW移行** |
| ルーティング | 全static | コア⇔new-rt間OSPF（area 0）＋`default-information originate` |
| VLAN20 | old-sw1/old-sw2両方に延伸、pc-bはold-sw1収容 | pc-bをold-sw2へ移設しold-sw2のみに整理 |
| 監視 | mon-core defaultはold-core向き | 新機器3台を監視対象に追加、mon-core defaultをnew-core1へ切替 |
| 旧機器 | ― | old-core/old-rtはshut→docker stopで撤去 |

フロアSW（old-sw1/old-sw2）とmon-core/cap-sw/エンドポイントは更改対象外（存置）とする。

## 3. 移行要件
- **許容断時間**: ユーザーVLANあたり**60秒以内**（カットオーバー1回につき）とする。
- **作業時間帯**: 業務時間外を想定する（本ラボでは擬似的に、業務時間外作業として一連の手順を実施する）。
- **移行方式**: VLAN単位の**段階移行**とする（方式比較は移行計画書§1〜2を参照）。
- **IP温存**: ユーザーセグメントのIPアドレス・デフォルトゲートウェイアドレス（各VLANの`.1`）は変更しない。端末側の設定変更はゼロとする。
- **監視継続**: 移行作業中もnmsの監視（連続ping・トラップ受信）を止めない。断時間の測定とエビデンス取得は監視継続を前提として行う。

## 4. 切り戻し要件
以下のいずれかに該当する場合、該当フェーズの切り戻し手順（切り戻し手順書参照）を実施する。

1. 断時間が60秒を超過した場合
2. カットオーバー判定試験（疎通マトリクス）にNG項目がある場合
3. HSRP・STPが期待状態にならない場合（例: Active/Standbyの逆転、STPが期待したルートブリッジ／ブロッキングポートにならない等）

切り戻し後は、原因究明が完了するまで同一フェーズの再カットオーバーを行わない（切り戻し手順書の原則を参照）。

## 5. 移設・撤去要件
- pc-bをold-sw1からold-sw2へ移設し、VLAN20をold-sw2のみに整理する（L2延伸の解消）。
- old-core・old-rtは、更改後構成での総合試験合格後に、インターフェースをshutdownしたうえでcontainerlabノードをdocker stopで撤去する。
- mon-coreの監視対象を新機器3台（new-core1/new-core2/new-rt）追加し、defaultルートをnew-core1向けに切り替える。
