# パラメータシート: 社内LAN更改・移行・移設（解答例）

本シートは、新機器3台（new-core1/new-core2/new-rt）の投入設定と、既存機器（mon-core・old-sw1・old-sw2）の変更差分の**解答例**です。まずは要件定義書・基本設計書・IPアドレス管理表・移行計画書をもとに自力でCLIを構成し、試験計画書のチェック項目に合格することを優先してください。本シートは、詰まったときの確認・答え合わせ用に使用してください。

> SNMPv3の認証・暗号化パスフレーズ（`SnmpAuth28!` / `SnmpPriv28!`）はテーマ28と同一の、本ラボ専用の学習用値です。実運用環境では絶対に使い回さないでください。
> old-coreのSVI shutdown等、カットオーバー当日の投入タイミングを伴う変更は、本シートではなく**カットオーバーMOP**・**切り戻し手順書**を参照してください。本シートに記載する新機器の設定例は、Mission3（新環境事前構築）の時点で投入し終える内容（フロアSW向けトランクとユーザーVLAN SVIはshutdownのまま）です。

## 1. new-core1（新コアL3SW・優先系／新設）
### インターフェース
| IF | IP / モード | 備考 |
|---|---|---|
| Ethernet0/1 | トランク（VLAN10,20,30,90） | new-core2向け（新コア間トランク） |
| Ethernet0/2 | トランク（VLAN10,20,30,90） | old-sw1向け。**P1までshutdown** |
| Ethernet0/3 | トランク（VLAN10,20,30,90） | old-sw2向け。**P1までshutdown** |
| Ethernet1/0 | 10.29.1.2/30（ルーテッド） | new-rt向け |
| Ethernet1/1 | 10.29.0.2/30（ルーテッド） | mon-core向け |
| VLAN10 SVI | 10.28.10.2/24 | HSRP standby 10、priority 110、preempt。**P2までshutdown** |
| VLAN20 SVI | 10.28.20.2/24 | HSRP standby 20、priority 110、preempt。**P4までshutdown** |
| VLAN30 SVI | 10.28.30.2/24 | HSRP standby 30、priority 110、preempt。**P5までshutdown** |
| VLAN90 SVI | 10.28.90.2/24 | HSRP standby 90、priority 110、preempt。**P6までshutdown** |

### 設定例
```
hostname new-core1
!
ip routing
!
vlan 10
 name SALES
vlan 20
 name GENERAL_AFFAIRS
vlan 30
 name SERVER
vlan 90
 name MGMT
!
interface Ethernet0/1
 switchport trunk encapsulation dot1q
 switchport mode trunk
 switchport trunk allowed vlan 10,20,30,90
 no shutdown
!
interface Ethernet0/2
 switchport trunk encapsulation dot1q
 switchport mode trunk
 switchport trunk allowed vlan 10,20,30,90
 shutdown
! --- カットオーバーMOP P1で no shutdown ---
!
interface Ethernet0/3
 switchport trunk encapsulation dot1q
 switchport mode trunk
 switchport trunk allowed vlan 10,20,30,90
 shutdown
! --- カットオーバーMOP P1で no shutdown ---
!
interface Ethernet1/0
 no switchport
 ip address 10.29.1.2 255.255.255.252
 no shutdown
!
interface Ethernet1/1
 no switchport
 ip address 10.29.0.2 255.255.255.252
 no shutdown
!
interface Vlan10
 ip address 10.28.10.2 255.255.255.0
 standby 10 ip 10.28.10.1
 standby 10 priority 110
 standby 10 preempt
 shutdown
! --- カットオーバーMOP P2で no shutdown ---
!
interface Vlan20
 ip address 10.28.20.2 255.255.255.0
 standby 20 ip 10.28.20.1
 standby 20 priority 110
 standby 20 preempt
 shutdown
! --- カットオーバーMOP P4で no shutdown ---
!
interface Vlan30
 ip address 10.28.30.2 255.255.255.0
 standby 30 ip 10.28.30.1
 standby 30 priority 110
 standby 30 preempt
 shutdown
! --- カットオーバーMOP P5で no shutdown ---
!
interface Vlan90
 ip address 10.28.90.2 255.255.255.0
 standby 90 ip 10.28.90.1
 standby 90 priority 110
 standby 90 preempt
 shutdown
! --- カットオーバーMOP P6で no shutdown ---
!
! --- STPルート設計 ---
spanning-tree vlan 10,20,30,90 root primary
!
! --- OSPF設計 ---
router ospf 1
 passive-interface Vlan10
 passive-interface Vlan20
 passive-interface Vlan30
 passive-interface Vlan90
 network 10.28.10.0 0.0.0.255 area 0
 network 10.28.20.0 0.0.0.255 area 0
 network 10.28.30.0 0.0.0.255 area 0
 network 10.28.90.0 0.0.0.255 area 0
 network 10.29.1.0 0.0.0.3 area 0
 redistribute static subnets
!
! --- 監視戻り経路(mon-core向け)。mon-coreはOSPFを話さないためstatic+redistributeで広報 ---
ip route 10.28.100.0 255.255.255.0 10.29.0.1
!
! --- SNMPv3(authPriv)。テーマ28と同値 ---
snmp-server group NMS-GROUP v3 priv
snmp-server user nmsuser NMS-GROUP v3 auth sha SnmpAuth28! priv aes 128 SnmpPriv28!
snmp-server location "Server-Room(New-Core1)"
snmp-server contact noc@example.jp
!
snmp-server enable traps snmp linkdown linkup coldstart
snmp-server host 10.28.100.10 version 2c public
```

## 4. 既存機器の変更差分

### 4.1 mon-core（監視L3SW）
Mission3（新環境事前構築）の時点で、新規リンク分の設定と並行稼働用の経路を追加する。defaultルートの切替はMission6（P8）で実施する。

```
! --- 新規: mon-core⇔new-core1間（監視拡張transit） ---
interface Ethernet0/3
 no switchport
 ip address 10.29.0.1 255.255.255.252
 no shutdown
!
! --- 新規: 並行稼働期間中は既存defaultに加えて10.29系への経路を追加 ---
ip route 10.29.0.0 255.255.0.0 10.29.0.2
!
! --- 既存: 並行稼働期間中は残す ---
! ip route 0.0.0.0 0.0.0.0 10.28.0.1
!
! --- カットオーバーMOP P8で実施: defaultをnew-core1向けへ切替 ---
! no ip route 0.0.0.0 0.0.0.0 10.28.0.1
! ip route 0.0.0.0 0.0.0.0 10.29.0.2
```

### 4.2 old-sw1（フロアA L2SW）
Mission3の時点で、新コア向けの2本のトランクポートを追加投入する（shutdownのまま）。`no shutdown`のタイミング、およびpc-b移設に伴うEthernet0/3の`shutdown`はカットオーバーMOP（P1・P7）を参照。

```
! --- 新規: new-core1向けトランク ---
interface Ethernet1/0
 switchport trunk encapsulation dot1q
 switchport mode trunk
 switchport trunk allowed vlan 10,20,30,90
 shutdown
! --- カットオーバーMOP P1で no shutdown ---
!
! --- 新規: new-core2向けトランク ---
interface Ethernet1/1
 switchport trunk encapsulation dot1q
 switchport mode trunk
 switchport trunk allowed vlan 10,20,30,90
 shutdown
! --- カットオーバーMOP P1で no shutdown ---
```

### 4.3 old-sw2（フロアB L2SW）
Mission3の時点で、新コア向けの2本のトランクポートに加え、pc-b移設先のアクセスポートを追加投入する（いずれもshutdownのまま）。`no shutdown`のタイミングはカットオーバーMOP（P1・P7）を参照。

```
! --- 新規: new-core1向けトランク ---
interface Ethernet0/3
 switchport trunk encapsulation dot1q
 switchport mode trunk
 switchport trunk allowed vlan 10,20,30,90
 shutdown
! --- カットオーバーMOP P1で no shutdown ---
!
! --- 新規: new-core2向けトランク ---
interface Ethernet1/0
 switchport trunk encapsulation dot1q
 switchport mode trunk
 switchport trunk allowed vlan 10,20,30,90
 shutdown
! --- カットオーバーMOP P1で no shutdown ---
!
! --- 新規: pc-b移設先アクセスポート ---
interface Ethernet1/1
 switchport access vlan 20
 switchport mode access
 shutdown
! --- カットオーバーMOP P7で no shutdown ---
```

### 4.4 old-core（単一コアL3SW）
old-core自体への設定投入はない（既存構成のまま維持し、カットオーバーMOPの各フェーズでSVIを順次shutdownしていくのみ）。具体的な投入コマンド（`interface Vlan10` → `shutdown`等）はカットオーバーMOPを参照。
