# Theme 29 機器別ログインコマンド

## 1. 実行場所

以下はすべてOrbStack Linux VM `clab`で実行します。

```bash
ssh clab@orb
cd /Users/shuya/Documents/claude/Mac仮想環境構築/29_lan_refresh_migration
```

## 2. 稼働確認

```bash
cd 04_構築
./deploy.sh inspect
```

すべてのノード（IOLおよびLinuxコンテナ、計14台）が`running`であることを確認します。

## 3. Cisco IOLへのログイン

IOLをPID1として起動し、`docker attach`の標準入力をIOSへ直結しています。
画面がすぐ表示されない場合は、接続後にEnterを1〜2回押してください。

### old-rt (老朽WANルータ)
```bash
sudo docker attach --sig-proxy=false clab-lan-refresh-lab-old-rt
```

### old-core (単一コアL3SW / SPOF)
```bash
sudo docker attach --sig-proxy=false clab-lan-refresh-lab-old-core
```

### old-sw1 (フロアA L2SW)
```bash
sudo docker attach --sig-proxy=false clab-lan-refresh-lab-old-sw1
```

### old-sw2 (フロアB L2SW)
```bash
sudo docker attach --sig-proxy=false clab-lan-refresh-lab-old-sw2
```

### mon-core (監視L3SW)
```bash
sudo docker attach --sig-proxy=false clab-lan-refresh-lab-mon-core
```

### cap-sw (Wireshark用SW / SPAN)
```bash
sudo docker attach --sig-proxy=false clab-lan-refresh-lab-cap-sw
```

### new-core1 (新コアL3SW #1 / HSRP Active)
```bash
sudo docker attach --sig-proxy=false clab-lan-refresh-lab-new-core1
```

### new-core2 (新コアL3SW #2 / HSRP Standby)
```bash
sudo docker attach --sig-proxy=false clab-lan-refresh-lab-new-core2
```

### new-rt (新WANルータ)
```bash
sudo docker attach --sig-proxy=false clab-lan-refresh-lab-new-rt
```

### コンテナを止めずに離脱
`Ctrl-p`を押し、続けて`Ctrl-q`を押します。`Ctrl-c`は離脱には使いません。

## 4. Linux端末へのログイン

### nms (監視サーバ)
```bash
sudo docker exec -it clab-lan-refresh-lab-nms bash
```

### cap (キャプチャホスト)
```bash
sudo docker exec -it clab-lan-refresh-lab-cap bash
```

### pc-a (営業部PC)
```bash
sudo docker exec -it clab-lan-refresh-lab-pc-a bash
```

### pc-b (総務部PC)
```bash
sudo docker exec -it clab-lan-refresh-lab-pc-b bash
```

### srv-file (ファイルサーバ)
```bash
sudo docker exec -it clab-lan-refresh-lab-srv-file bash
```

Linuxシェルからは`exit`で抜けます。

## 5. ログインできない場合

### 5.1 コンテナ確認
```bash
sudo docker ps --format 'table {{.Names}}\t{{.Status}}' | grep lan-refresh-lab
```

### 5.2 起動ログ確認
```bash
sudo docker logs --tail 50 clab-lan-refresh-lab-old-core
```

### 5.3 ラボを作り直す場合
> ⚠️ 保存していないIOS設定は消えるため、必要な設定を保存してから実行します。`./deploy.sh destroy`は`--cleanup`付きでnvramも削除されます（詳細は[../04_構築/構築手順書.md](../04_構築/構築手順書.md)を参照）。
```bash
cd 04_構築
./deploy.sh destroy
./deploy.sh deploy
```
