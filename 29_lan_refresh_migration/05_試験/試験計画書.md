# 試験計画書: 社内LAN更改・移行・移設

本試験は、老朽LAN（テーマ28の移行元）から新環境（new-core1/new-core2/new-rt）への段階移行が、Mission 1〜6の各段階で要件通りに実施できていることを確認するためのものです。

## 1. 現行環境as-is再現＋移行前ベースライン取得 (Mission 1)
Given: テーマ28の老朽LAN（old-rt/old-core/old-sw1/old-sw2）と監視基盤（mon-core/nms/cap-sw）が、テーマ28の構成のまま再現されていること。
- [ ] **pc-a**（10.28.10.10）から `ping -c 4 203.0.113.1` → 4/4成功
- [ ] **pc-b**（10.28.20.10）から `ping -c 4 203.0.113.1` → 4/4成功
- [ ] **srv-file**（10.28.30.10）から `ping -c 4 203.0.113.1` → 4/4成功
- [ ] **old-core**で `show ip route static` → デフォルトルート `0.0.0.0/0 via 10.28.1.1` が存在する
- [ ] **old-sw1**・**old-sw2**それぞれで `show vlan brief` → 両SWにVLAN20が存在する（総務部VLANのL2延伸＝老朽ポイントを再確認）
- [ ] **nms**から `snmpwalk -v3 -l authPriv -u nmsuser -a SHA -A 'SnmpAuth28!' -x AES -X 'SnmpPriv28!' <各IP> system` を老朽LAN側4台（old-rt/old-core/old-sw1/old-sw2）に対して実行 → 全て成功する（テーマ28の監視基盤が引き続き機能していることを確認）
- [ ] **nms**から `snmpwalk -v2c -c public 10.28.90.1 system` → 応答がなくTimeoutになる（テーマ28 Mission5でv2cコミュニティを削除済みであり、本テーマの移行元はSNMPv3のみで運用されている状態を再確認）
- [ ] 疎通マトリクス（pc-a/pc-b/srv-file × old-rt Lo0）と経路情報（`show ip route`）・ARP/MACアドレステーブル（`show arp`, `show mac address-table`）を記録し、移行前ベースラインとして保存する

## 2. 移行計画策定 (Mission 2) — 文書ミッション
Given: 要件定義書・基本設計書・IPアドレス管理表が整備されていること。本ミッションはコマンド投入を伴わない、ドキュメント完成のミッションである。
- [ ] 移行計画書§1の移行方式比較表（一括切替／段階移行／並行稼働）を読み、それぞれの断時間・切り戻し容易性・作業量・リスクの違いを自分の言葉で説明できる
- [ ] カットオーバーMOP中の「（Mission 2で記入）」と明記された空欄（P1-7の許容再収束時間、P2-6の許容損失数、P4-5のHSRP期待状態、P5のT+目安、P6-10のold-core依存解消の確認方法）を、要件定義書・基本設計書・IPアドレス管理表・移行計画書の記載内容から導出して記入する
- [ ] 切り戻し手順書の内容を読み、各フェーズの戻しコマンドがカットオーバーMOPの対応する作業を正しく打ち消す内容になっていることを確認する
- [ ] 移行計画書§5のリスク一覧を読み、各リスクがどのフェーズで発生し得るか、対策がどの文書（切り戻し手順書等）に対応しているかを説明できる

## 3. 新環境事前構築＋新環境単体試験 (Mission 3)
Given: new-core1/new-core2/new-rtが、パラメータシート§1〜3のとおり構築済みであること（フロアSW向けトランク・ユーザーVLAN SVIはshutdownのまま）。
参考: 書籍『ネットワーク「動作試験」入門』の`参考資料/ネットワーク技術/ネットワーク「動作試験」入門/_index/text/p091.md`（FHRP冗長化機能試験の内容・確認項目）
- [ ] **new-core1**・**new-core2**で `show standby brief` → 全VLAN（10/20/30/90）でnew-core1=Active、new-core2=Standby
- [ ] **new-core1**・**new-core2**で `show ip ospf neighbor` → new-rtとの隣接がFULL
- [ ] **new-rt**で `show ip route ospf` → OSPF経由のdefault route（`default-information originate always`の効果）が広報されていることを確認する
- [ ] **nms**から `snmpwalk -v3 -l authPriv -u nmsuser -a SHA -A 'SnmpAuth28!' -x AES -X 'SnmpPriv28!' 10.28.90.2 system`（new-core1）・同`10.28.90.3`（new-core2）・同`10.29.1.1`（new-rt）→ いずれも成功する（新機器3台の監視対象追加）
- [ ] **HSRPフェイルオーバー試験**: **new-core1**で `interface Vlan10` → `shutdown` → **new-core2**で `show standby vlan 10 brief` → State=Active に昇格する。この時点でユーザーVLANのGW自体はまだold-core側にあるため、**pc-a**への疎通に影響がないこと（`ping -c 4 10.28.10.1`→4/4成功）を確認する（ユーザー無影響証明）
- [ ] フェイルオーバー確認後、**new-core1**で `interface Vlan10` → `no shutdown` → `standby 10 preempt` によりnew-core1がActiveへ復帰することを `show standby vlan 10 brief` で確認する

## 4. カットオーバー第1弾VLAN10＋断時間測定 (Mission 4)
Given: カットオーバーMOP P0〜P1が完了していること。
- [ ] カットオーバーMOP P2の手順に従い、VLAN10のGWをold-coreからnew-core1/2へ移行する
- [ ] **nms**で `ping -i 0.2 10.28.10.1` の実行結果から、損失数×0.2秒で算出した断時間が60秒以内である
- [ ] **nms**のsnmptrapd出力に、old-core Vlan10のlinkDownトラップと、new-core1・new-core2 Vlan10のlinkUpトラップが着信している
- [ ] **pc-a**から `ping -c 4 10.28.10.1` → 4/4成功、`ping -c 4 203.0.113.2` → 4/4成功
- [ ] **new-core1**で `show standby vlan 10 brief` → State=Active

## 5. 切り戻し訓練→再カットオーバー→VLAN30→WAN切替 (Mission 5)
Given: Mission4でVLAN10のカットオーバーが完了していること。
参考: 書籍『ネットワーク「動作試験」入門』の`参考資料/ネットワーク技術/ネットワーク「動作試験」入門/_index/text/p111.md`（物理リンク障害復旧試験・許容時間内での通信復旧確認の考え方）
- [ ] カットオーバーMOP P4の手順でVLAN20をカットオーバーする
- [ ] 意図的に切り戻し手順書P4の手順を実施し、VLAN20のGWをold-coreへ戻す → **pc-b**から `ping -c 4 10.28.20.1` → 4/4成功（移行前ベースラインと同等に復旧していることを確認）
- [ ] 切り戻し後、切り戻し手順書§2の原則に従い、原因（本ラボでは意図的な訓練である旨）を記録したうえで、カットオーバーMOP P4を再実行する → 断時間測定・トラップ確認・疎通確認のいずれも成功する
- [ ] カットオーバーMOP P5の手順でVLAN30をカットオーバーする → **srv-file**から `ping -c 4 10.28.30.1` → 4/4成功、断時間60秒以内、トラップ着信を確認する
- [ ] カットオーバーMOP P6の手順でVLAN90のGWを移行する → **old-sw1**・**old-sw2**から `ping -c 4 203.0.113.2` → いずれも4/4成功（WAN切替の完了、old-coreのdefault route依存解消の確認）
- [ ] **pc-a**から `ping -c 4 203.0.113.2` → 4/4成功（VLAN10/20/30/90の全GW移行完了後、社内のどのセグメントからも新・疑似インターネットへ到達できることの最終確認）

## 6. 移設・撤去・移行後総合試験 (Mission 6)
Given: Mission5でP2〜P6のカットオーバーが全て完了していること。
- [ ] カットオーバーMOP P7の手順でpc-bをold-sw1からold-sw2へ移設する → **pc-b**から `ping -c 4 10.28.20.1` → 4/4成功
- [ ] **old-sw1**・**old-sw2**で `show vlan brief` → VLAN20がold-sw2のみに存在する（L2延伸の解消を最終確認）
- [ ] **nms**から9台（old-rt/old-core/old-sw1/old-sw2/mon-core/cap-sw/new-core1/new-core2/new-rt）全てへSNMPv3ポーリングが成功する（旧機器撤去直前の最終全数確認）
- [ ] カットオーバーMOP P8の手順でold-core・old-rtを撤去し、**mon-core**のdefault routeをnew-core1（10.29.0.2）向けへ切り替える
- [ ] **nms**から残り7台（old-sw1/old-sw2/mon-core/cap-sw/new-core1/new-core2/new-rt）へのSNMPv3ポーリングが成功し、old-rt・old-core宛のポーリングが対象から除外されている
- [ ] **pc-a**・**pc-b**・**srv-file**から `ping -c 4 203.0.113.2` → いずれも4/4成功
- [ ] 全点疎通マトリクスを実施し、Mission1で記録した移行前ベースラインと比較して、意図した変更点（VLAN20の集約、GWの実体、WANの到達先）以外に差異がないことを確認する
