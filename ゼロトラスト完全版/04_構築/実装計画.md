# ゼロトラスト完全版｜実装計画

仮想企業への統合到達点に向けた実装ロードマップ（S0〜S6）・環境前提・進め方をまとめる。Phase番号（P0-P6）・N番号（N1-N4）は各部品の出自ラボにおける管理単位として存続する。**実装（config投入）はこのフォルダでは行わず、各実装テーマへ委譲する**。

## 実装方針

- 本フォルダは統合設計・検証エビデンスの集約であり、実装作業は各テーマ（[ZERO_zero_trust](../../ZERO_zero_trust/README_Lab_Challenge.md)・[31_nac_dot1x](../../31_nac_dot1x/README_Lab_Challenge.md)・[36_ztna_openziti](../../36_ztna_openziti/README_Lab_Challenge.md)・[42_ndr_flow](../../42_ndr_flow/README_Lab_Challenge.md)・[35_faucet_sdn](../../35_faucet_sdn/README_Lab_Challenge.md)・[microseg_cilium](../../microseg_cilium/README_Lab_Challenge.md)・[microseg_nftables](../../microseg_nftables/README_Lab_Challenge.md)）に委譲する。
- 「できそうな順（arm64確度・依存の少なさ）」を優先する方針は [NW-ZT_トラックロードマップ](../../ZERO_zero_trust/02_基本設計/NW-ZT_トラックロードマップ.md) を踏襲する。

## 統合ロードマップ（S0〜S6）

L7/NW-ZT別の依存順を個別に描く方式は廃止し、部品✅の現状から統合ポイントI-1〜I-8を接続していく単一のステージ表に統合する。

| ステージ | 内容 | 対応ゲート | 実装先テーマ |
|---|---|---|---|
| S0 | 部品✅の現状（達成済） | IG-L7-0・IG-N1・IG-N2・IG-N3・IG-N3-補助・IG-N4-cilium・IG-N4-nftables | ZERO_zero_trust／31_nac_dot1x／36_ztna_openziti／42_ndr_flow／35_faucet_sdn／microseg_cilium／microseg_nftables |
| S1 | keycloak実装＋I-1（RADIUSバックエンド接続） | IG-I1 | ZERO_zero_trust（Phase1実装）→31_nac_dot1x（RADIUS連携先） |
| S2 | SIEM全社統合＋I-7 | IG-I7（旧IG-F2） | ZERO_zero_trust（Phase3実装）→42_ndr_flow（既存eve→Loki統合と合流） |
| S3 | I-2（動的VLAN×μセグ同居）＋I-3（全社mirror） | IG-I2（旧IG-F1）・IG-I3 | 31_nac_dot1x×microseg_cilium／microseg_nftables、35_faucet_sdn（mirror拡張） |
| S4 | T-N1-7/T-N1-8（VLAN99自動割当・CoA） | IG-FB・IG-FC（前段） | 31_nac_dot1x |
| S5 | I-4 SOAR-lite（本丸） | IG-I4・IG-FC | 実装先未定（新規: soar-lite） |
| S6 | I-6 IAP+mTLS・I-8 SDN化（発展） | IG-I6・IG-I8 | ZERO_zero_trust（Phase2/6実装）、35_faucet_sdn（発展） |

各ステージの実configは実装先テーマの `04_構築/` へ委譲する（本表はステージ順と対応ゲートのみを扱う）。

## 環境前提

| 項目 | 値 |
|---|---|
| ホスト | macOS（Apple Silicon / arm64） |
| VM | OrbStack VM `clab`（Ubuntu 24.04 arm64） |
| 出自: ZERO_zero_trust（P0-P6） | docker完結（IOL非連携、D-1） |
| 出自: 31/35/36/42/microseg_*（N1-N4） | IOL連携解禁（D-7） |
| コンテナランタイム | Docker（OrbStack提供） |
| イメージ制約 | arm64マニフェスト必須。x86専用イメージ不可 |

## 進め方

1. **S0（現状）**: 各部品はP0（ZERO_zero_trust、2026-07-04）・N1-N4（31_nac_dot1x／36_ztna_openziti／42_ndr_flow／microseg_cilium／microseg_nftables、2026-07-05、N3補助は35_faucet_sdnで2026-07-07）が実装済・実機検証済。P1-P6は [ZERO_zero_trust/02_基本設計/段階ロードマップ.md](../../ZERO_zero_trust/02_基本設計/段階ロードマップ.md) のゲート条件に沿って、実装時にテーマ側で着手する。config投入はテーマ側の `04_構築/` で行う。
2. **S1〜S6（統合ロードマップ）**: 上記の統合ロードマップ表に沿って、統合ポイントI-1〜I-8を順次接続する。追加実装（CoA自動隔離等の発展課題）は各実装テーマの残課題を参照して着手する。
3. **委譲の原則**: 本フォルダでは進め方・依存順・環境前提のみを扱い、実際のconfig（faucet.yaml/RADIUS属性/nftablesルール等）は各テーマの `03_詳細設計/` `04_構築/` に置く。

## リスクと代替

| リスク | 内容 | 代替 |
|---|---|---|
| arm64非対応イメージ | FreeRADIUS・ntopng等、公式Dockerイメージがamd64のみのケース | arm64 OSSパッケージが存在する場合はapt導入のDockerfile自作で回避（N1-N4実測済） |
| 真の非対応（arm64パッケージ自体が無い） | ClamAV・osquery等、amd64専用でarm64版が存在しない | 機能代替（例: DLPはmitmproxyアドオンに一本化）または該当機能を設計のみに留める |
| NetFlowパイプライン制約 | softflowd(arm64)がフロー化できず処理件数0 | Suricata flow（5-tuple統計）で代替（N3で実証済、[42_ndr_flow試験結果](../../42_ndr_flow/05_試験/試験結果_2026-07-05.md)参照） |
| IOL起動コスト | IOLノード1台の起動に約2分、リソース消費が大きい | IOL連携は出自ラボ（31/35/36/42/microseg_*側）のみで許容し、ZERO_zero_trust側はdocker完結に限定（D-1） |
| P1-P6未実装のギャップ | 設計のみのため実機での統合ポイント（I-1・I-4・I-7等）が未検証 | ◐設計のみとして明示し、実装着手時に統合検証結果を更新する |
| CoA未実装がF-B/F-Cのブロッカー | RADIUS CoAが未実装（T-N1-8）のため、隔離VLAN自動移行・NDR→SOAR連携が成立しない | S4でT-N1-7/T-N1-8を先行実施し、S5でSOAR-lite本体を構築する |
| keycloak⇔FreeRADIUS連携方式の選定未着手 | I-1（ID統一）のLDAP/OIDC等の連携方式が未確定 | S1着手時にZERO_zero_trust側Phase1実装と合わせて選定する |
| soar-liteの実装先テーマ未定 | I-4のwebhook受信・CoA発行を担うsoar-liteはどの既存テーマにも属さない新規コンポーネント | S5着手時に実装先（新規ミニテーマ、または31_nac_dot1x拡張）を決定する |
