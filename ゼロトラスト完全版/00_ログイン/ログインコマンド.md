# ゼロトラスト完全版｜ログインコマンド

本ドキュメントは統合ショーケースとしての**入口の束**であり、各層固有のログイン手順は実装テーマ側の `00_ログイン/ログインコマンド.md` が正とする。ここでは代表例のみを示す（実設定・認証情報は書かない）。

## VM 接続

```bash
# OrbStack VM clab へ SSH
ssh clab@orb

# 接続確認
docker --version
containerlab version
```

## E2Eフロー ↔ 出自テーマ ↔ 該当セクション対応表

[統合アーキテクチャマップ](../02_基本設計/統合アーキテクチャマップ.md) のE2EフローF-A〜F-Eが、本ページのどの入口に対応するかの索引。

| フロー | 出自テーマ | 該当セクション（本ページ内） |
|---|---|---|
| F-A 社員PC入場 | 31_nac_dot1x | N1 NAC（31_nac_dot1x） |
| F-B 未認証端末 | 31_nac_dot1x | N1 NAC（31_nac_dot1x） |
| F-C 感染端末の検知→自動隔離 | 42_ndr_flow／35_faucet_sdn／31_nac_dot1x | N3 NDR（42_ndr_flow、補助: 35_faucet_sdn） |
| F-D リモート社員のZTNA到達 | 36_ztna_openziti | N2 SDP-ZTNA（36_ztna_openziti） |
| F-E サーバ間東西のμセグ+L7認可 | microseg_cilium／microseg_nftables | N4 μセグ（microseg_cilium／microseg_nftables） |

## 実装出自テーマ別の入口（テーマ横断の代表例）

コンテナ名は containerlab の命名規約により `clab-<lab名>-<node名>` になる（実際の名前は各テーマの `docker ps` で確認する）。

### P0-P6（ZERO_zero_trust、Phase 0 実装済）

```bash
# Phase 0 ノード（client / app / external）へのアクセス
docker exec -it clab-zero-client bash
docker exec -it clab-zero-app bash
```

詳細は [ZERO_zero_trust/00_ログイン/ログインコマンド.md](../../ZERO_zero_trust/00_ログイン/ログインコマンド.md) を参照。Phase 1-6 は設計のみのため未確定。

### N1 NAC（31_nac_dot1x）

```bash
# IOL スイッチへの attach（テーマ側 entrypoint 経由）
docker exec -it clab-nac-sw1 bash
docker exec -it clab-nac-radius bash
```

詳細は [31_nac_dot1x/README_Lab_Challenge.md](../../31_nac_dot1x/README_Lab_Challenge.md) を参照（テーマ側に専用ログインドキュメントはなく、README内のアクセス方法・構築ログを参照する）。

### N2 SDP-ZTNA（36_ztna_openziti）

```bash
# controller/router 一体コンテナへのアクセス
docker exec -it ziti bash
```

詳細は [36_ztna_openziti/README_Lab_Challenge.md](../../36_ztna_openziti/README_Lab_Challenge.md) を参照（テーマ側に専用ログインドキュメントはなく、README内のアクセス方法を参照する）。

### N3 NDR（42_ndr_flow、補助: 35_faucet_sdn）

```bash
# Suricata センサーへのアクセス（host mode）
docker exec -it suricata bash

# Faucet補助（テーマ35, mirror/gauge 連携）
docker exec -it clab-faucet-sdn-ovs1 bash
```

詳細は [42_ndr_flow/README_Lab_Challenge.md](../../42_ndr_flow/README_Lab_Challenge.md)（テーマ側に専用ログインドキュメントはなく、README内のアクセス方法を参照する）・[35_faucet_sdn/00_ログイン/ログインコマンド.md](../../35_faucet_sdn/00_ログイン/ログインコマンド.md) を参照。

### N4 μセグ（microseg_cilium／microseg_nftables）

```bash
# Cilium系（k3d クラスタ経由）
kubectl -n microseg exec -it deploy/frontend -- sh

# nftables系（IOL + Linux ホスト）
docker exec -it clab-microseg-pc10a bash
```

詳細は [microseg_cilium/README_Lab_Challenge.md](../../microseg_cilium/README_Lab_Challenge.md)・[microseg_nftables/README_Lab_Challenge.md](../../microseg_nftables/README_Lab_Challenge.md) を参照（両テーマとも専用ログインドキュメントはなく、README内の手順を参照する）。
