# 規約 — 索引と目的

`Mac仮想環境構築/` 配下のラボテーマを新規作成・運用するときの標準を定義する。
既存の `環境/`（現状の説明）・`ロードマップ/`（未来の計画）に対し、`規約/` は「あるべき形の定義」を担う三点セットの一角。

## 適用範囲

- **新規テーマ（テーマ28以降・テーマZERO含む）に適用する。**
- 既存テーマ02〜27には遡及しない（不整合の指摘は [提案_運用改善.md](提案_運用改善.md) に記録のみ）。
- 新規テーマの雛形生成は `containerlab-theme` スキルが本規約の `雛形/` を単一ソースとして参照する。

## 索引

| ファイル | 内容 |
|---|---|
| [ドキュメント標準.md](ドキュメント標準.md) | ディレクトリ構成・必須ドキュメント・章立て・記法規律・命名規約 |
| [frontmatterスキーマ.md](frontmatterスキーマ.md) | 全MDに付与するYAML frontmatterの値辞書とgrepレシピ |
| [clab運用規約.md](clab運用規約.md) | clab.yml配置・deploy.sh標準・VM実行パターン・禁止事項 |
| [構成図規約.md](構成図規約.md) | Mermaid第一・HTML併用・drawio新規非推奨の判断基準 |
| [Markdown採用判断.md](Markdown採用判断.md) | 「MDでよいのか」の検討結論（ADR） |
| [提案_運用改善.md](提案_運用改善.md) | 既存環境への改善提案（非改変・記録のみ） |
| [監査チェックリスト.md](監査チェックリスト.md) | 新規テーマの規約適合セルフ監査 |
| `雛形/` | テーマ生成用テンプレート一式（.tmpl） |
| `ビルド/` | MD→HTML ビルドシステム（build.mjs） |

## 品質ゲート（全新規テーマ共通）

1. `node 規約/ビルド/build.mjs --check` がpass（NFD命名・リンク切れ・frontmatter欠落・H1複数 = 0件）
2. [監査チェックリスト.md](監査チェックリスト.md) のセルフ監査を完了
3. `00_ログイン/ログインコマンド.md` の全機器実ログイン確認（2026-06-30決定事項）

## 改訂ルール

- 規約の変更は本フォルダ内ファイルの編集＋下表への追記で行う。雛形とスキルの二重管理は禁止（スキルは雛形を参照する）。
- 改訂が既存テーマに影響する場合も遡及修正はせず、`提案_運用改善.md` に移行推奨として記録する。
