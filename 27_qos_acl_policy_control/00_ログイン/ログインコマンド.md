## 3. Cisco IOLへのログイン

IOLをPID1として起動し、`docker attach`の標準入力をIOSへ直結しています。
画面がすぐ表示されない場合は、接続後にEnterを1〜2回押してください。

### hq-core (HQ Core)
```bash
sudo docker attach --sig-proxy=false clab-qos-acl-policy-lab-hq-core
```

### hq-dist (HQ Distribution)
```bash
sudo docker attach --sig-proxy=false clab-qos-acl-policy-lab-hq-dist
```

### hq-edge (HQ Edge/QoS/NAT)
```bash
sudo docker attach --sig-proxy=false clab-qos-acl-policy-lab-hq-edge
```

### hq-dmz (HQ DMZ Switch)
```bash
sudo docker attach --sig-proxy=false clab-qos-acl-policy-lab-hq-dmz
```

### br-core (Branch Core)
```bash
sudo docker attach --sig-proxy=false clab-qos-acl-policy-lab-br-core
```

### br-edge (Branch Edge)
```bash
sudo docker attach --sig-proxy=false clab-qos-acl-policy-lab-br-edge
```

### isp (ISP Router)
```bash
sudo docker attach --sig-proxy=false clab-qos-acl-policy-lab-isp
```

## 6. SSH直接ログイン（一括操作・自動化向け・2026-07-10追加）

各機器で `crypto key generate rsa` 済みなら、`docker attach`（対話・低速・不安定）でなく
**SSHで直接**入れる。複数機器への確認・採取を並列で回せるため高速。

OrbStack VM（`ssh clab@orb`）から実行。IOSは古い暗号のため**レガシーKEXオプションが必須**:

```bash
ssh -o KexAlgorithms=+diffie-hellman-group14-sha1 \
    -o HostKeyAlgorithms=+ssh-rsa -o PubkeyAcceptedAlgorithms=+ssh-rsa \
    -o Ciphers=+aes128-cbc admin@172.27.27.13     # ユーザー admin
```

**複数コマンドを非対話で流す**（`terminal length 0` でページャ無効化）:

```bash
printf 'terminal length 0\nshow ip ospf neighbor\nshow ip bgp summary\n' \
  | ssh <上記オプション> admin@172.27.27.11
```

**7台一括**（並列・全機器の状態を1秒未満で取得）:

```bash
for ip in 11 12 13 14 21 22 100; do
  ( printf 'terminal length 0\nshow ip ospf neighbor\n' \
    | ssh <上記オプション> admin@172.27.27.$ip ) &
done; wait
```

> 注意: SSHは `write memory`・設定投入も同方式で可能だが、本ラボは手動設定が学習目的のため、
> 一括投入は検証・採取に限定し、Mission設定は各自でコンソール投入すること。
