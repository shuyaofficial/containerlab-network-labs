# Theme 27 機器別ログインコマンド

## 1. 実行場所

以下はすべてOrbStack Linux VM `clab`で実行します。

```bash
ssh clab@orb
cd /Users/shuya/Documents/claude/Mac仮想環境構築/27_qos_acl_policy_control
```

## 2. 稼働確認

```bash
cd 04_構築
./deploy.sh inspect
```

すべてのノード（IOLおよびLinuxコンテナ）が`running`であることを確認します。

## 3. Cisco IOLへのログイン

IOLをPID1として起動し、`docker attach`の標準入力をIOSへ直結しています。
画面がすぐ表示されない場合は、接続後にEnterを1〜2回押してください。

### hq-core (HQ Core)
```bash
sudo docker attach --sig-proxy=false clab-qos-acl-policy-lab-hq-core
```

### hq-dist (HQ Distribution)
```bash
sudo docker attach --sig-proxy=false clab-qos-acl-policy-lab-hq-dist
```

### hq-edge (HQ Edge/QoS/NAT)
```bash
sudo docker attach --sig-proxy=false clab-qos-acl-policy-lab-hq-edge
```

### hq-dmz (HQ DMZ Switch)
```bash
sudo docker attach --sig-proxy=false clab-qos-acl-policy-lab-hq-dmz
```

### br-core (Branch Core)
```bash
sudo docker attach --sig-proxy=false clab-qos-acl-policy-lab-br-core
```

### br-edge (Branch Edge)
```bash
sudo docker attach --sig-proxy=false clab-qos-acl-policy-lab-br-edge
```

### isp (ISP Router)
```bash
sudo docker attach --sig-proxy=false clab-qos-acl-policy-lab-isp
```

### コンテナを止めずに離脱
`Ctrl-p`を押し、続けて`Ctrl-q`を押します。`Ctrl-c`は離脱には使いません。

## 4. Linux端末へのログイン

### hq-pc (Data endpoint)
```bash
sudo docker exec -it clab-qos-acl-policy-lab-hq-pc bash
```

### hq-voip (Voice endpoint)
```bash
sudo docker exec -it clab-qos-acl-policy-lab-hq-voip bash
```

### dmz-srv (Web Server)
```bash
sudo docker exec -it clab-qos-acl-policy-lab-dmz-srv bash
```

### br-pc (Branch endpoint)
```bash
sudo docker exec -it clab-qos-acl-policy-lab-br-pc bash
```

Linuxシェルからは`exit`で抜けます。

## 5. ログインできない場合

### 5.1 コンテナ確認
```bash
sudo docker ps --format 'table {{.Names}}\t{{.Status}}' | grep qos-acl-policy-lab
```

### 5.2 起動ログ確認
```bash
sudo docker logs --tail 50 clab-qos-acl-policy-lab-hq-core
```

### 5.3 ラボを作り直す場合
> ⚠️ 保存していないIOS設定は消えるため、必要な設定を保存してから実行します。
```bash
cd 04_構築
./deploy.sh destroy
./deploy.sh deploy
```
