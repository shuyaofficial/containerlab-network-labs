1. 試験方針#

1.1 目的

試験計画書.md の Mission 1〜6 に対応し、ゾーン間ACL・PBR・DSCPマーキング・LLQ優先制御・BGP経路フィルタが要件通りに動作すること（ST-N: 正常性試験）、および経路障害発生時・復旧時にもこれらのポリシーが変更なく継続動作すること（ST-F: 障害試験）を検証する。

1.2 範囲

  • ST-N: ゾーン間ACL（Mission 1）/ PBR（Mission 2）/ DSCPマーキング（Mission 3）/ LLQ・帯域制御（Mission 4）/ BGP経路フィルタ（Mission 5）
  • ST-F: hq-core–hq-edge リンク障害・復旧時の総合シナリオ（Mission 6）

1.3 前提条件

  • 単体試験仕様書・結合試験仕様書の全項目が合格していること
  • hq-edge の ACL OUTSIDE-IN、hq-dmz の ACL DMZ-OUT、hq-dist の PBR PBR_VOICE、hq-edge の QoS WAN-EDGE、br-edge の prefix-list BRANCH_NET が投入済みであること
  • 作業は OrbStack Linux VM clab（ssh clab@orb）上で実施すること

3.2 障害試験（ST-F）

i
本ラボのトポロジ補足campus.clab.yml はhq-core を中心としたスター構成であり、hq-core–hq-edge間に冗長経路は存在しない。したがって本リンクの障害は「別経路への迂回」ではなく、HQ内部からWAN/インターネット方向への単一障害点による通信断として現れる。試験計画書.md Mission6が指す「迂回後もポリシーが継続動作すること」は、本ラボでは「① 障害の影響範囲がhq-core–hq-edge間の到達性に限定され、他ポリシー（Area 0内部到達性、Branch/WAN側のBGP・QoS・ACL）は無停止で動作し続けること」「② リンク復旧後にACL/QoS/PBR設定が再投入なしで即座に機能すること」として検証する。

4. 合格基準#

  • ST-N・ST-F をあわせた全19項目が 合格 であること。
  • ST-N01〜05（ゾーン間ACL）は許可/拒否の方向性を1件でも取り違えると不合格とし、該当ACLのエントリ順序・条件を是正する。
  • ST-N08・N09（DSCP）は 0x2E / 46 以外の値が観測された場合、hq-distの set dscp ef 設定を再確認する。
  • ST-N11・N12（LLQ）はVOICEクラスに1パケットでもドロップが発生した場合不合格とし、priority帯域とWAN帯域設定の整合性を見直す。
  • ST-F04（Branch/WAN側の非依存性）が不合格の場合、hq-core側の障害がBGP/OSPFの再帰的な経路喪失を通じてBranch側に波及していないか設計を見直す。
  • 本試験の合格をもってTheme 27の全試験を完了とする。
