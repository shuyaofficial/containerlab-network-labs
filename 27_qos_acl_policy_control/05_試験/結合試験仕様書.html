1. 試験方針#

1.1 目的

単体で正しく構築された各ノード同士が、リンク・OSPF・BGPで正しく隣接し、経路を交換し、拠点間でエンドツーエンドの疎通ができることを検証する。ACL/QoS/PBRのポリシー内容そのものの動作確認は正常性障害試験仕様書で扱う。

1.2 範囲

  • ルータ間6リンクの対向ping疎通
  • OSPFネイバー確立（HQ Area 0 / Branch Area 1）
  • BGPピア確立（hq-edge⇔isp、br-edge⇔isp）
  • 経路受信確認（デフォルトルート・支社経路の伝播）
  • 拠点間エンドツーエンド疎通

1.3 前提条件

  • 単体試験仕様書の全項目が合格していること
  • OSPF: HQ側は Area 0（hq-core / hq-dist / hq-edge / hq-dmz）、Branch側は Area 1（br-edge / br-core）
  • BGP: hq-edge = AS 65000、br-edge = AS 65001、isp = AS 65100
  • 作業は OrbStack Linux VM clab（ssh clab@orb）上で実施すること

2. 試験環境#

4. 合格基準#

  • 本書の全19項目が 合格 であること。
  • OSPF/BGPネイバーが1件でも FULL / Established にならない場合、隣接インターフェースの設定（IPアドレス・エリア番号・AS番号・ACL誤設定の有無）を再確認する。
  • 経路受信（IT-13〜17）が不合格の場合、原因（フィルタ設定・redistribute未設定等）を 03_詳細設計/パラメータシート.md と突き合わせて是正する。
  • 本試験に合格したのち、正常性障害試験仕様書に進む。
