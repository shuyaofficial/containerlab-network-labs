# 基本設計書: QoS/ACL/route-map応用

## 1. ネットワークゾーンと論理構成
本システムは『ネットワーク・デザインパターン』のコンセプトに基づき、4つのゾーンを構成する。各ゾーンの境界（エッジルータやディストリビューションスイッチ）で適切なセキュリティ・QoSポリシーを適用する。

- **Trustゾーン**: 本社・支社の社内LAN (10.27.10.0/24, 10.27.20.0/24, 10.27.50.0/24)
- **DMZゾーン**: 本社公開サーバ (10.27.30.0/24)
- **Untrustゾーン**: ISP/インターネット (198.51.100.0/24)
- **WANゾーン**: 拠点間接続網

## 2. アクセス制御（ACL）設計
ACLは各インターフェースのできるだけ「送信元に近い場所（インバウンド）」または「境界ルータのアウトバウンド」に適用する。

- **hq-edge (Untrust境界)**:
  - 外部インターフェース（ISP側）のインバウンドにACLを適用。
  - DMZのパブリックIP宛てHTTP/HTTPSを許可。
  - 内部からの戻り通信（TCP established等）を許可し、他はDeny。
- **hq-core / hq-dmz (DMZ境界)**:
  - DMZインターフェースのインバウンドで、DMZからTrust宛てのパケットをDeny。

## 3. ポリシーベースルーティング（PBR）設計
- **実装箇所**: hq-dist
- **設計内容**: `hq-voip` (10.27.50.0/24) からのパケットをACLでキャッチし、`route-map` で `set ip next-hop` を用いて、通常のOSPF/EIGRPで計算された経路とは別の特定のネクストホップへ転送する。

## 4. QoS (Quality of Service) 設計
シスコの **MQC (Modular QoS CLI)** フレームワークを使用して設計する。

### 4.1 トラフィック分類とマーキング（class-map / policy-map）
- **実装箇所**: hq-dist (LAN側エッジ) または hq-edge (WAN側エッジ)
- **分類**:
  - `class-map match-all VOICE`: ACLで送信元 10.27.50.0/24 をマッチ。
- **マーキング**:
  - `policy-map MARKING` にて、VOICEクラスに対して `set dscp ef` を設定。

### 4.2 キューイングとスケジューリング（policy-map）
- **実装箇所**: hq-edge のWANインターフェース (アウトバウンド)
- **設計**:
  - `policy-map QUEUING` を作成。
  - `class VOICE` に対して `priority` コマンドを使用（LLQ）し、帯域幅（例: 2000 kbps）を保証。
  - 他のトラフィックに対しては `bandwidth` や `fair-queue` を設定。

## 5. BGP経路フィルタリング設計
- **実装箇所**: hq-edge, br-edge
- **設計内容**:
  - `ip prefix-list` を定義し、自拠点のサブネットのみを `permit` する。
  - BGPの `neighbor <IP> prefix-list <NAME> out` コマンドで適用し、意図しない経路漏洩を防ぐ。
