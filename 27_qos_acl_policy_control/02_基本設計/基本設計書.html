1. 設計方針（4ゾーンモデル）#

本システムは『ネットワーク・デザインパターン』のコンセプトに基づき、4つのゾーンを構成する。各ゾーンの境界（エッジルータやディストリビューションスイッチ）で適切なセキュリティ・QoSポリシーを適用する。

ゾーン該当サブネット境界実装ノード

DMZ10.27.30.0/24本社公開サーバ（hq-dmz配下）

4. アクセス制御(ACL)設計#

ACLは各インターフェースのできるだけ「送信元に近い場所（インバウンド）」または「境界ルータのアウトバウンド」に適用する。パケットが不要に内部ネットワークを横断してからdenyされる無駄を避けるための設計方針である。

4.1 hq-edge（Untrust境界）— OUTSIDE-IN

外部インターフェース（ISP側 Ethernet0/2）のインバウンドに適用する。

順序ルール意図
10permit tcp any host 10.27.30.10 eq 80 DMZ公開サーバへのHTTP許可
20permit tcp any host 10.27.30.10 eq 443 DMZ公開サーバへのHTTPS許可
30permit icmp any any echo-reply内部発ICMPの戻り許可（またはestablished句で代用）
40deny ip any anyその他は全拒否（暗黙denyの明示化）

4.2 hq-dmz（DMZ境界）— DMZ-OUT

DMZインターフェース（Ethernet0/2、to dmz-srv）のインバウンドに適用し、DMZからTrust宛てのパケットをdenyする。

順序ルール意図
5permit tcp host 10.27.30.10 eq 80 anyWebサーバ応答（80番発）を許可。Trust→DMZ 80/443要件の戻りを成立させる（ISS-13）
6permit tcp host 10.27.30.10 eq 443 any同HTTPS応答
7permit icmp host 10.27.30.10 10.27.10.0 0.0.0.255 echo-replyPC→srv の ping 返事のみ許可（片方向ping・運用疎通確認用の拡張 ISS-14）
10deny ip 10.27.30.0 0.0.0.255 10.27.10.0 0.0.0.255DMZ→HQ Trust（VLAN10）拒否
20deny ip 10.27.30.0 0.0.0.255 10.27.20.0 0.0.0.255DMZ→Branch Trust（VLAN20）拒否
25deny ip 10.27.30.0 0.0.0.255 10.27.50.0 0.0.0.255DMZ→HQ Trust（VLAN50/VoIP）拒否 ※初版で漏れ、レビューで追補（課題管理表 ISS-12）
30permit ip any anyその他（戻り通信含む）は許可

5. PBR設計#

実装箇所: hq-dist。hq-voip（10.27.50.0/24）からのパケットをACLでキャッチし、route-map で set ip next-hop を用いて、通常のOSPFで計算された経路とは別の特定のネクストホップへ転送する。

項目設定値備考
対象ACLpermit ip 10.27.50.0 0.0.0.255 anyhq-voipセグメントを抽出
route-maproute-map PBR_VOICE permit 10 PBRポリシー名
matchmatch ip address <ACL>上記ACLで抽出した通信にマッチ
set next-hopset ip next-hop 10.27.1.1 hq-core向けIF（学習上の例示値）
set dscpset dscp ef PBR適用と同時にマーキングも実施
適用Ethernet0/3 inbound: ip policy route-map PBR_VOICEhq-dist（to hq-voip）

next-hop 10.27.1.1 は通常経路（OSPF計算結果）のネクストホップと同一であり、「別経路への誘導」というPBR本来の教育効果は限定的だが、学習上の例示値として許容する（課題管理表 課題3: オープン）。

6. QoS設計#

シスコの MQC（Modular QoS CLI） フレームワークを使用し、「分類 → マーキング → キューイング（LLQ）」の3段構成で設計する。

  1. 分類（class-map）ACLで送信元 10.27.50.0/24（hq-voip）をマッチさせ、class-map match-all VOICE として定義する。実装箇所は hq-dist（LAN側エッジ）。
  2. マーキング（policy-map MARKING）policy-map MARKING にて VOICEクラスに対し set dscp ef を設定する。
  3. キューイング（policy-map WAN-EDGE / LLQ）実装箇所は hq-edge のWANインターフェース（アウトバウンド）。class-map match-any VOICE（match dscp ef）を policy-map WAN-EDGE に組み込み、class VOICE: priority 2000 （kbps・LLQ）で帯域を絶対保証し、class class-default: fair-queue で残りを公平配分する。Ethernet0/2 のアウトバウンドに service-policy output WAN-EDGE を適用する。

7. BGP経路フィルタリング設計#

実装箇所: hq-edge, br-edge。ip prefix-list を定義し、自拠点のサブネットのみを permit する。BGPの neighbor <IP> prefix-list <NAME> out コマンドで適用し、意図しない経路漏洩を防ぐ。

項目設定値備考
prefix-listip prefix-list BRANCH_NET permit 10.27.20.0/24 br-edge（AS 65001）が広告してよい自拠点経路
適用neighbor 198.51.100.13 prefix-list BRANCH_NET out br-edge → isp 方向のBGPネイバー
hq-edge側同様に自拠点サブネットのみをprefix-listでisp側へ広告意図しない他拠点経路の漏洩防止
