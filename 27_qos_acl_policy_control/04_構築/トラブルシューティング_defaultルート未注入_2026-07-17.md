# トラブルシューティングログ — デフォルトルートが社内に降りてこない（2026-07-17）

まっさら化した環境をユーザーが Phase 1〜11 まで自力で再構築する過程で踏んだ躓き2件。
どちらも `default-information originate` の「付け場所」ミスで、症状の出方が異なる好例。

---

## 事象1: hq-pc → ISP Loopback(10.27.255.100) が届かない

### 症状
- `hq-pc` から `ping 10.27.255.100` → `From 10.27.10.1 Destination Host Unreachable`
- ゲートウェイ(hq-dist)自身がエラーを返す＝hq-distに経路が無い
- `hq-dist# show ip route` → `Gateway of last resort is not set`（デフォルト無し）
- 一方 `hq-edge` には `B* 0.0.0.0/0 via 198.51.100.10` が存在（BGPは正常）

### 原因
`default-information originate` が **空のOSPFプロセス `router ospf 1`** に付与されていた。
実際に稼働してE0/1で隣接を張っているのは **`router ospf 10`**。空プロセスは注入先の
OSPFドメインを持たないため、コマンドを書いても何も起きない。

- 遠因: **ドキュメント不整合**。`コンフィグ解答編.html` は `router ospf 1`、
  `パラメータシート_設定順.md` は `OSPF 10 (Area 0)`。番号を混在させて拾うと不発になる。

### 対処
```
hq-edge# configure terminal
 no router ospf 1
 router ospf 10
  default-information originate
```

### 確認
`hq-dist# show ip route 0.0.0.0` → `O*E2 0.0.0.0/0 via 10.27.1.1` が出現。

## 事象2: hq-voip → br-pc(10.27.20.10) が無言ハング

### 症状
- `hq-voip` から `ping 10.27.20.10` → **エラーも応答も出ずハング**（事象1と違いUnreachableが出ない）
- 無言ハング＝「行きは相手に届くが、戻りが返れない」の典型
- `br-core# show ip route 0.0.0.0` → `% Network not in table`（Branch内部にデフォルト無し）
- br-edge側は準備万端: BGP UP(PfxRcd=4) / `B 10.27.50.0/24` 学習済 / br-coreとOSPF FULL

### 原因
`default-information originate` を **br-core** に付けてしまった。
br-core は自分がデフォルトルートを持っていない（BGP無し）ため、配るものが無く無効。
デフォルトを **実際に持っているのは、ISPからBGPで 0/0 を受けている br-edge のみ**。
配れるのは境界ルータ(br-edge)だけ。

### 対処
```
br-core# configure terminal
 router ospf 10
  no default-information originate      ! 無効なので撤去
br-edge# configure terminal
 router ospf 10                          ! Area 1を動かしている実プロセス
  default-information originate
```

### 確認
`br-core# show ip route 0.0.0.0` → `O*E2 0.0.0.0/0 via 10.27.2.1`、
`hq-voip# ping 10.27.20.10` 疎通OK。

---

## 教訓（次回チェックリスト）

1. **`default-information originate` は「自分がデフォルトを実際に持つ、BGP↔OSPFの境界ルータ」に付ける。**
   内側のコア/ディストリに付けても無効（`always` を付けない限り、RIBにデフォルトが無いと配らない）。
2. **付ける先は「実稼働しているOSPFプロセス」。** 空プロセスに付けると不発。
   `show run | section router ospf` で稼働プロセス番号（隣接を張っているIFの所属）を確認してから投入。
3. **症状で切り分ける**:
   - `Destination Host Unreachable (From ゲートウェイ)` = ゲートウェイに経路が無い（前方の経路欠落）
   - **無言ハング** = 行きは届くが戻りが返れない（戻り経路 or 遠端のデフォルト欠落）
4. **ドキュメントのOSPFプロセス番号を統一する**（解答編=1 / パラメータシート=10 の不整合は要修正）。
