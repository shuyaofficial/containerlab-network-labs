本書は参考解（一例）本ラボの学習方針は全設定を手動でCLI投入することである。本書はあくまで一つの構成例に過ぎない。まず自力でパラメータシートと詳細設計書を基に構成し、行き詰まった箇所だけ該当機器を開いて確認すること。最初から本書を書き写すと学習効果が失われる。

hq-dist#

Distribution層。hq-voip（VLAN50）向けEthernet0/3インバウンドにPBRでネクストホップ強制を適用し、同インバウンドのMQC入力ポリシーでDSCP EFマーキングを行う。

解答を見る — hq-dist 完成コンフィグ （まず自力で構成してから開くこと）
hq-dist — running-config相当
hostname hq-dist
no ip domain-lookup
interface Loopback0
 ip address 10.27.255.2 255.255.255.255
interface Ethernet0/1
 description to hq-core
 ip address 10.27.1.2 255.255.255.252
 no shutdown
interface Ethernet0/2
 description to hq-pc (VLAN10)
 ip address 10.27.10.1 255.255.255.0
 no shutdown
interface Ethernet0/3
 description to hq-voip (VLAN50)
 ip address 10.27.50.1 255.255.255.0
 ip policy route-map PBR_VOICE
 service-policy input MARKING
 no shutdown
! PBR: hq-voip発を分類するACL（PBRのmatchとMQCのmatch access-groupで共用）
ip access-list extended VOIP-SRC
 permit ip 10.27.50.0 0.0.0.255 any
! PBR: ネクストホップ強制のみ（DSCPマーキングはMQCへ分離。理由は本節末尾のcallout参照）
route-map PBR_VOICE permit 10
 match ip address VOIP-SRC
 set ip next-hop 10.27.1.1
! QoS: MQC入力ポリシーでDSCP EFマーキング（基本設計書.mdのMARKING方式）
class-map match-all VOICE-SRC
 match access-group name VOIP-SRC
policy-map MARKING
 class VOICE-SRC
  set dscp ef
router ospf 1
 router-id 10.27.255.2
 network 10.27.1.0 0.0.0.3 area 0
 network 10.27.10.0 0.0.0.255 area 0
 network 10.27.50.0 0.0.0.255 area 0
 network 10.27.255.2 0.0.0.0 area 0
end
!
DSCPマーキングをMQCへ分離した理由パラメータシート.mdはPBRのroute-map内で set dscp ef を行う記載だが、PBR用route-mapのDSCP設定は本来 set ip dscp 構文であり、これ自体をサポートしないIOSも多い（IOL 15.7.3M2でも弾かれる可能性が高い）。確実に動作する参考解として、基本設計書.mdが示すMQC入力ポリシー（class-map VOICE-SRC → policy-map MARKING → service-policy input MARKING）でマーキングを行う方式を採用する。対応環境が確認できれば、route-mapへset ip dscp efを統合する構成に置き換えても構わない。

hq-edge#

Untrust境界。ACL・QoS(LLQ)・NAT・BGP・OSPFデフォルト経路再配布のすべてが集中する最重要機器。Ethernet0/1がNAT inside、Ethernet0/2がNAT outsideかつWAN。

解答を見る — hq-edge 完成コンフィグ （まず自力で構成してから開くこと）
hq-edge — running-config相当
hostname hq-edge
no ip domain-lookup
interface Loopback0
 ip address 10.27.255.254 255.255.255.255
interface Ethernet0/1
 description to hq-core (inside)
 ip address 10.27.1.6 255.255.255.252
 ip nat inside
 no shutdown
interface Ethernet0/2
 description to isp (outside / WAN)
 ip address 198.51.100.9 255.255.255.252
 ip access-group OUTSIDE-IN in
 ip nat outside
 service-policy output WAN-EDGE
 no shutdown
! ACL: Untrust(ISP)側インバウンド。上から順に評価、暗黙denyで終端
ip access-list extended OUTSIDE-IN
 permit tcp any host 10.27.30.10 eq 80
 permit tcp any host 10.27.30.10 eq 443
 permit icmp any any echo-reply
 permit tcp any any established
 deny ip any any
! QoS: VOICEクラスをEFで拾いLLQで最優先、その他はfair-queue
class-map match-any VOICE
 match dscp ef
policy-map WAN-EDGE
 class VOICE
  priority 2000
 class class-default
  fair-queue
! NAT: HQ Trust(PC/VoIP)のみoverloadでインターネットへ変換
ip access-list extended NAT-INSIDE
 permit ip 10.27.10.0 0.0.0.255 any
 permit ip 10.27.50.0 0.0.0.255 any
ip nat inside source list NAT-INSIDE interface Ethernet0/2 overload
! BGP: ISPへは自拠点サブネットのみ広告
ip prefix-list HQ_NET seq 5 permit 10.27.10.0/24
ip prefix-list HQ_NET seq 10 permit 10.27.30.0/24
ip prefix-list HQ_NET seq 15 permit 10.27.50.0/24
router bgp 65000
 neighbor 198.51.100.10 remote-as 65100
 neighbor 198.51.100.10 prefix-list HQ_NET out
 network 10.27.10.0 mask 255.255.255.0
 network 10.27.30.0 mask 255.255.255.0
 network 10.27.50.0 mask 255.255.255.0
! OSPF: ISPからeBGPで学習したデフォルトルートをHQ内部へ再配布
router ospf 1
 router-id 10.27.255.254
 network 10.27.1.4 0.0.0.3 area 0
 network 10.27.255.254 0.0.0.0 area 0
 default-information originate
end

解答を見る — hq-dmz 完成コンフィグ （まず自力で構成してから開くこと）
hq-dmz — running-config相当
hostname hq-dmz
no ip domain-lookup
interface Loopback0
 ip address 10.27.255.3 255.255.255.255
interface Ethernet0/1
 description to hq-core
 ip address 10.27.1.10 255.255.255.252
 no shutdown
interface Ethernet0/2
 description to dmz-srv (VLAN30)
 ip address 10.27.30.1 255.255.255.0
 ip access-group DMZ-OUT in
 no shutdown
! ACL: Webサーバ応答(80/443発)を先に許可し、DMZ発→Trust宛てを拒否、他は許可
! establishedでなくポート限定なのは SSH等の戻りまで通さないため（ISS-13）
ip access-list extended DMZ-OUT
 permit tcp host 10.27.30.10 eq 80 any
 permit tcp host 10.27.30.10 eq 443 any
 permit icmp host 10.27.30.10 10.27.10.0 0.0.0.255 echo-reply
 deny ip 10.27.30.0 0.0.0.255 10.27.10.0 0.0.0.255
 deny ip 10.27.30.0 0.0.0.255 10.27.20.0 0.0.0.255
 deny ip 10.27.30.0 0.0.0.255 10.27.50.0 0.0.0.255
 permit ip any any
router ospf 1
 router-id 10.27.255.3
 network 10.27.1.8 0.0.0.3 area 0
 network 10.27.30.0 0.0.0.255 area 0
 network 10.27.255.3 0.0.0.0 area 0
end

主要確認コマンド#

投入後は以下のコマンドで各ポリシーが意図通り動作しているかを確認する。詳細な試験手順は単体試験仕様書・結合試験仕様書を参照。

コマンド実行機器の例確認内容
show ip ospf neighborhq-core, hq-dist, hq-edge, hq-dmz, br-core, br-edgeOSPF隣接がFULLであること（Area 0 / Area 1それぞれ）
show ip bgp summaryhq-edge, br-edge, ispeBGPピアがEstablished状態であること
show access-listshq-edge, hq-dmzOUTSIDE-IN / DMZ-OUTの各行のマッチ数（hitcount）が増加していること
show route-maphq-distPBR_VOICEのmatchカウントが増加していること
show policy-map interface Ethernet0/3hq-distMARKINGポリシー（VOICE-SRCクラス）でDSCP EFへのマーキングが実施されていること
show policy-map interface Ethernet0/2hq-edgeVOICEクラス（LLQ）のパケットカウントと、class-defaultのfair-queue動作状況
