# パラメータシート（ルーティング・HSRP編）

本ドキュメントは、本社内を連携させる「OSPF」、インターネットと経路交換する「BGP」、そしてデフォルトゲートウェイの冗長化を行う「HSRP」のパラメータを定義します。

## 1. OSPF 設計 (本社内ルーティング)
本社内の全ルーター・L3スイッチで OSPF プロセスを起動し、Area 0 (バックボーンエリア) として社内網の経路情報を動的に交換します。

### OSPF 基本パラメータ
| 機器名 | プロセスID | ルータID (Loopback0) | エリア | 参加するネットワーク (Network コマンド) |
| :--- | :--- | :--- | :--- | :--- |
| HQ-Edge1 | 1 | 1.1.1.1 | 0 | `10.0.0.0 0.0.0.3` (Core1向)<br>`10.0.0.4 0.0.0.3` (Core2向) |
| HQ-Edge2 | 1 | 2.2.2.2 | 0 | `10.0.0.8 0.0.0.3` (Core1向)<br>`10.0.0.12 0.0.0.3` (Core2向) |
| HQ-Core1 | 1 | 3.3.3.3 | 0 | `10.0.0.0 0.0.0.3` (Edge1向)<br>`10.0.0.8 0.0.0.3` (Edge2向)<br>`10.1.10.0 0.0.0.255` (VLAN10) |
| HQ-Core2 | 1 | 4.4.4.4 | 0 | `10.0.0.4 0.0.0.3` (Edge1向)<br>`10.0.0.12 0.0.0.3` (Edge2向)<br>`10.1.10.0 0.0.0.255` (VLAN10) |

### デフォルトルートの再配送 (Default Information Originate)
HQ-Edge1 および HQ-Edge2 はインターネットへの出口であるため、BGPや静的経路で得た「デフォルトルート(`0.0.0.0/0`)」を、OSPFを通じてCoreスイッチ側に注入（再配送）します。
- **HQ-Edge1**: `default-information originate` (デフォルトメトリック1) を設定し、メインのデフォルトルートとする。
- **HQ-Edge2**: `default-information originate metric 20` を設定し、バックアップのデフォルトルートとする。これにより、平時はHQ-Edge1にトラフィックが集中し、明示的なActive/Standbyフェイルオーバー構成となる。

## 2. BGP 設計 (インターネット接続)
本社エッジルーターとISPルーター間で、eBGP (External BGP) ピアリングを確立します。

### BGP 基本パラメータ
| 機器名 | 自AS番号 | ピア(隣接)先IP | ピア先AS番号 | 備考 / 役割 |
| :--- | :--- | :--- | :--- | :--- |
| ISP (疑似) | 65000 | 200.0.1.1 (Edge1)<br>200.0.2.1 (Edge2) | 65001 | デフォルトルートを本社へ広報 |
| HQ-Edge1 | 65001 | 200.0.1.254 (ISP) | 65000 | メイン回線ピア |
| HQ-Edge2 | 65001 | 200.0.2.254 (ISP) | 65000 | サブ回線ピア |

### 経路制御 (Local Preference)
本社のインターネット通信を「常に HQ-Edge1（メイン回線）を優先」させるため、BGP属性（Local Preference）を操作します。
- **HQ-Edge1**: Local Preference = `200` (優先度高)
- **HQ-Edge2**: Local Preference = `100` (デフォルト値・優先度低)

## 3. 冗長化プロトコル (HSRP)
HQ-Core1 と HQ-Core2 は、営業部PC (VLAN 10) に対して冗長化されたデフォルトゲートウェイを提供します。

### HSRP パラメータ (VLAN 10 向け)
| 項目 | 設定値 | 備考 |
| :--- | :--- | :--- |
| 対象インターフェース | Vlan 10 | SVIに対して設定 |
| HSRP グループ番号 | 10 | VLAN IDと合わせる |
| **仮想IPアドレス (VIP)** | **10.1.10.254** | PCに設定するデフォルトゲートウェイ |
| HQ-Core1 プライオリティ | 150 | `Active` (メイン) として動作させるため高く設定 |
| HQ-Core2 プライオリティ | 100 | `Standby` (サブ) として動作 (デフォルト値) |
| Preempt機能 | 常に有効 | 障害復旧時に自動的にActiveを奪い返す設定 |
