## 🆕 このテーマで新しく学ぶこと

| 技術 | なぜ必要か（21との差分） |
|---|---|
| 3階層設計（コア/ディストリ/アクセス） | 機器が増えても「どこに何の役割を置くか」が破綻しないため |
| OSPFマルチエリア＋経路集約 | 棟・フロアが増えてもコアの経路テーブルが肥大化しないため |
| discard route（集約の罠対策） | ABRが2台ある集約構成の古典的ブラックホール問題への対処 |
| MST | VLANが増えてもSTPの計算・運用負荷を増やさないため |
| FortiGate（専用FW） | ルーターACLからの卒業。ステートフルポリシー・ゾーン管理 |
| FortiGate⇔Cisco IPsec interop | 実務で頻出する「ベンダーが違う機器同士のVPN」のデバッグ経験 |

これまでの積み上げ（VLAN/トランク/EtherChannel/STP/HSRP/OSPF/ACL/NAT/IPsec）は**全部使います**。

## 🖥️ 環境メモ

### リソースと段階デプロイ
- ネットワーク機器11台（IOL×10＝約10GB、FortiGate＝2GB）＋エンドポイント5台。**合計約13GB**。
- IOLは1ノード約2分起動。**一気に全部起動せず、フェーズに合わせた段階デプロイを推奨**:
  - 方法A: clab.ymlのノードを段階的にコメントアウト解除して `deploy` し直す
  - 方法B: `containerlab deploy --node-filter <ノード名,...>` で部分起動
  - フェーズ1〜2はキャンパス内部（IOL 8台）だけで進められる。FGT/ISP/支社はフェーズ3以降でOK

### FortiGateの扱い（初登場！）
- 導入手順・イメージタグは [`../環境/他ベンダー追加手順.md`](../環境/他ベンダー追加手順.md) を参照（起動検証済み・約2分）
- 管理: 起動後、管理IPへ **HTTPSでGUI**、またはSSHでCLI（`get system interface physical` で物理IF一覧）
- ⚠️ **clab.ymlの `ethN` とFortiOSの `portN` はズレる**（eth0=管理）。結線表の「想定」を実機で必ず確認してから設定すること
- FortiGateのログはトラブルシュートの宝庫。VPNが上がらない時は**FGT側とCisco側の両方のログを突き合わせる**のが定石

### いつもの鉄則
1. **復路（帰り道）を必ず確認**（06の教訓）
2. 新環境はまず **no shutdown**
3. **障害試験までやって初めて「できた」**
4. EtherChannelは**LACP必須**・エンドポイントは**multitool**・linksは**ethN表記**（[`../切り分け/containerlab_iol_quirks.md`](../切り分け/containerlab_iol_quirks.md)）
