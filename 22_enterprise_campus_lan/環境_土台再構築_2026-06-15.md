## 2. 残っている壁（あなたの判断・素材が必要）

`campus.clab.yml` が要求するイメージのうち、**ライセンス4種が消失**している（このMac全体・Drive・ゴミ箱を探索済みだが無し）。

| イメージ（タグはclab.ymlと一致が必須） | 役割 | 台数 | 素材 |
|---|---|---|---|
| `vrnetlab/cisco_iol:L2-advipservices-2017` | L3SW（core1/2・dist-a1/a2/b1） | 5 | IOL `.bin`（L3スイッチ用） |
| `vrnetlab/cisco_iol:L2-15.2` | L2SW（acc-a1/a2/b1） | 3 | IOL `.bin`（L2スイッチ用） |
| `vrnetlab/cisco_iol:15.7.3M2` | Router（isp・br-edge） | 2 | IOL `.bin`（ルータ用） |
| `vrnetlab/vr-fortios:7.2.4` | 境界FW（fgt-edge） | 1 | FortiGate `qcow2`（要ログインDL） |
| `wbitt/network-multitool` | PC/サーバ ×5 | 5 | ✅ 取得済み |

### ⚠ 旧世代IOLのライセンス問題（重要）
modern vrnetlab(v0.21.0)のIOLビルダーは **iourcライセンス生成を持っていない**（iol.binを直接起動するだけ）。
旧環境の15.x世代IOLは **iourc必須**で、消えた旧VMでは自作entrypoint（キー自動生成 + expectで初期ダイアログ自動応答）で解決していた。
→ **同世代の.binを使う場合、このカスタムentrypointの復元が必要**。`.bin`を1つ用意してくれれば、実バイナリのhostid出力に合わせて復元・検証する（盲目で当てずに、実物で確実に通す）。

代替案（より入手容易・堅牢）: iourc不要の**新しめIOL（例: 17.12.x のL2/L3）**を入手 → modern定義のままビルド可。
その場合は `campus.clab.yml` の `image:` タグを入手版に合わせて差し替える（タグは単なるラベルなので問題なし）。

## 3. 素材が揃ったあとの再構築手順（コマンド）

```bash
# 1) 素材を所定の名前で配置（clab VM 内）
#    例:
ssh clab@orb
cp /path/to/iol_l3sw.bin  ~/vrnetlab/cisco/iol/cisco_iol-L2-advipservices-2017.bin
cp /path/to/iol_l2sw.bin  ~/vrnetlab/cisco/iol/cisco_iol-L2-15.2.bin
cp /path/to/iol_router.bin ~/vrnetlab/cisco/iol/cisco_iol-15.7.3M2.bin
cp /path/to/fortios.qcow2 ~/vrnetlab/fortigate/fortios-v7.2.4.qcow2

# 2) ビルド→状況確認→デプロイ（ヘルパー一発）
cd "/Users/shuya/Documents/claude/Mac仮想環境構築/22_enterprise_campus_lan"
./build_and_deploy.sh all

# 個別に:
./build_and_deploy.sh status     # 充足状況だけ確認
./build_and_deploy.sh build      # イメージビルドのみ
./build_and_deploy.sh deploy     # デプロイのみ
./build_and_deploy.sh destroy    # 撤去
```

デプロイ後の確認:
```bash
sudo containerlab inspect -t campus.clab.yml      # 全ノードの起動状態
sudo docker exec -it clab-campus-pc-sales bash    # 営業PCに入る
curl http://10.20.30.20/                          # 社内ポータル疎通（L3+HTTP）
```
> 機器のCLI設定（OSPF/VLAN/MST/HSRP/FortiGateポリシー等）は**あなたの学習範囲**。ここでは投入しない。

## 6. 2026-06-17 デプロイ完了

### 解決した問題
| 問題 | 原因 | 解決策 |
|---|---|---|
| IOLルータ(15.7.3M2) ARM64でSystem Config Dialogハング | IOLルータバイナリがARM64 QEMU上でダイアログ表示時にCPU 96%で無限ループ | `L2-advipservices-2017`イメージに切替（L3ルーティング完備・ダイアログ非表示） |
| docker attachでコンソール接続不可（全IOLノード） | entrypoint.shで`&`バックグラウンド起動→IOLのfd 0が`/dev/null`に | `exec`でIOLを直接起動（PID 1=iol.bin、fd 0=/dev/pts/0） |
| FortiGate QEMU crash | ARM64でcpu=host不可 | `env: QEMU_CPU: qemu64` をclab.ymlに設定 |

### 最終構成
- IOL Core/Dist/ISP/BR-edge（7台）: `vrnetlab/cisco_iol:L2-advipservices-2017`（type: l2）
- IOL Access（3台）: `vrnetlab/cisco_iol:L2-15.2`（type: l2）
- FortiGate: `vrnetlab/vr-fortios:7.4.2.F`（QEMU_CPU: qemu64）
- エンドポイント（5台）: `wbitt/network-multitool`
