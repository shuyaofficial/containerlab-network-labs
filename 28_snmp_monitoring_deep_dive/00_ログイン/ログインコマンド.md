# Theme 28 機器別ログインコマンド

## 1. 実行場所

以下はすべてOrbStack Linux VM `clab`で実行します。

```bash
ssh clab@orb
cd /Users/shuya/Documents/claude/Mac仮想環境構築/28_snmp_monitoring_deep_dive
```

## 2. 稼働確認

```bash
cd 04_構築
./deploy.sh inspect
```

すべてのノード（IOLおよびLinuxコンテナ、計14台）が`running`であることを確認します。

## 3. Cisco IOLへのログイン

IOLをPID1として起動し、`docker attach`の標準入力をIOSへ直結しています。
画面がすぐ表示されない場合は、接続後にEnterを1〜2回押してください。

### old-rt (老朽WANルータ)
```bash
sudo docker attach --sig-proxy=false clab-snmp-monitoring-lab-old-rt
```

### old-core (単一コアL3SW / SPOF)
```bash
sudo docker attach --sig-proxy=false clab-snmp-monitoring-lab-old-core
```

### old-sw1 (フロアA L2SW)
```bash
sudo docker attach --sig-proxy=false clab-snmp-monitoring-lab-old-sw1
```

### old-sw2 (フロアB L2SW)
```bash
sudo docker attach --sig-proxy=false clab-snmp-monitoring-lab-old-sw2
```

### mon-core (監視L3SW)
```bash
sudo docker attach --sig-proxy=false clab-snmp-monitoring-lab-mon-core
```

### cap-sw (Wireshark用SW / SPAN)
```bash
sudo docker attach --sig-proxy=false clab-snmp-monitoring-lab-cap-sw
```

### コンテナを止めずに離脱
`Ctrl-p`を押し、続けて`Ctrl-q`を押します。`Ctrl-c`は離脱には使いません。

## 4. Linux端末へのログイン

### nms (監視サーバ)
```bash
sudo docker exec -it clab-snmp-monitoring-lab-nms bash
```

### cap (キャプチャホスト)
```bash
sudo docker exec -it clab-snmp-monitoring-lab-cap bash
```

### pc-a (営業部PC)
```bash
sudo docker exec -it clab-snmp-monitoring-lab-pc-a bash
```

### pc-b (総務部PC)
```bash
sudo docker exec -it clab-snmp-monitoring-lab-pc-b bash
```

### srv-file (ファイルサーバ)
```bash
sudo docker exec -it clab-snmp-monitoring-lab-srv-file bash
```

Linuxシェルからは`exit`で抜けます。

### zbx-srv (Zabbix Server)
```bash
sudo docker exec -it clab-snmp-monitoring-lab-zbx-srv bash
```

### zbx-db (Zabbix用DB / postgres)
```bash
sudo docker exec -it clab-snmp-monitoring-lab-zbx-db bash
```
DB接続確認例（学習用値、実運用では使い回さないこと）:
```bash
psql -U zabbix -d zabbix
```

### zbx-web (Zabbix Web UI)
```bash
sudo docker exec -it clab-snmp-monitoring-lab-zbx-web bash
```

Web UIはMacから **http://clab.orb.local:8280** でアクセスします。初期ログインはZabbix既定値の **Admin / zabbix** です。初回ログイン後、必ずパスワードを変更してください。

## 5. ログインできない場合

### 5.1 コンテナ確認
```bash
sudo docker ps --format 'table {{.Names}}\t{{.Status}}' | grep snmp-monitoring-lab
```

### 5.2 起動ログ確認
```bash
sudo docker logs --tail 50 clab-snmp-monitoring-lab-old-core
```

### 5.3 ラボを作り直す場合
> ⚠️ 保存していないIOS設定は消えるため、必要な設定を保存してから実行します。`./deploy.sh destroy`は`--cleanup`付きでnvramも削除されます（詳細は[../04_構築/構築手順書.md](../04_構築/構築手順書.md)を参照）。
```bash
cd 04_構築
./deploy.sh destroy
./deploy.sh deploy
```
