# 試験計画書: 監視基盤とSNMP深堀り

本試験は、老朽LANへ後付けした監視基盤（mon-core / nms / cap-sw）とSNMP監視設定が、Mission 1〜6の各段階で要件通りに動作していることを確認するためのものです。

## 1. 老朽LANベースライン構築 (Mission 1)
Given: old-rt/old-core/old-sw1/old-sw2からなる老朽LAN（VLAN10/20/30/90、static routing）が構築済みであること。
- [ ] **pc-a**（10.28.10.10）から `ping -c 4 203.0.113.1` → 4/4成功（営業部→old-rt Lo0=疑似インターネットへ到達）
- [ ] **pc-b**（10.28.20.10）から `ping -c 4 203.0.113.1` → 4/4成功
- [ ] **srv-file**（10.28.30.10）から `ping -c 4 203.0.113.1` → 4/4成功
- [ ] **old-core**で `show ip route static` → デフォルトルート `0.0.0.0/0 via 10.28.1.1` が存在する
- [ ] **old-rt**で `show ip route static` → `10.28.0.0/16 via 10.28.1.2` の経路が存在する
- [ ] **old-sw1**・**old-sw2**それぞれで `show vlan brief` → 両SWにVLAN20が存在する（総務部VLANのL2延伸＝老朽ポイントを確認）

## 2. 監視セグメント増設 (Mission 2)
Given: mon-core/cap-swを増設し、VLAN100/900のIPアドレス設計（IPアドレス管理表）に基づき設定済みであること。
- [ ] **nms**から `ping -c 4 10.28.1.1`（old-rt） → 成功
- [ ] **nms**から `ping -c 4 10.28.90.1`（old-core） → 成功
- [ ] **nms**から `ping -c 4 10.28.90.13`（old-sw1） → 成功
- [ ] **nms**から `ping -c 4 10.28.90.14`（old-sw2） → 成功
- [ ] **nms**から `ping -c 4 10.28.0.2`（mon-core）／`ping -c 4 10.28.0.3`（cap-sw） → いずれも成功
- [ ] **cap-sw**で `show monitor session 1` → source=Et0/1(both), destination=Et0/3 と表示される

## 3. SNMPv2c＋MIBツリー深堀り (Mission 3)
Given: 6 IOL機器全てに `snmp-server community public RO` を投入済みであること。
参考: `参考資料/ネットワーク技術/ネットワーク「動作試験」入門/_index/text/p074.md`, `p075.md`（SNMPポーリング試験・snmpwalkコマンド）
- [ ] **nms**から `snmpwalk -v2c -c public 10.28.90.1 system` → old-coreのsystemサブツリー（sysDescr, sysUpTime, sysName等）が返る
- [ ] **nms**から `snmpget -v2c -c public 10.28.90.1 1.3.6.1.2.1.1.5.0` → sysName=old-core が返る
- [ ] **nms**から `snmpget -v2c -c public 10.28.90.1 sysName.0`（名前OID） → 数値OID指定時と同じ値が返る
- [ ] **nms**から `snmptranslate -Tp -IR system` → MIB-IIのsystemサブツリー構造がツリー表示される
- [ ] **nms**から `snmpwalk -v2c -c public 10.28.1.1 system`（old-rt） → sysDescrに旧IOS 15.6のバージョン文字列が含まれる

## 4. IF-MIBカウンタ監視 (Mission 4)
Given: Mission3のSNMPv2c設定が有効であること。
- [ ] **nms**から `snmpwalk -v2c -c public 10.28.90.14 ifDescr` → old-sw2の各インターフェースのifIndexを確認する（以降このifIndexを使用）
- [ ] **srv-file**で `iperf3 -s` を起動し、**pc-a**から `iperf3 -c 10.28.30.10 -t 30` を実行してトラフィックを生成する
- [ ] トラフィック生成の前後で **nms**から `snmpget -v2c -c public 10.28.90.14 1.3.6.1.2.1.2.2.1.10.<ifIndex>`（ifInOctets、old-sw2のold-core向けトランクポート）を2回ポーリングし、差分値がトラフィック生成量相当に増加している
- [ ] 同区間・同時刻で `snmpget -v2c -c public 10.28.90.14 1.3.6.1.2.1.31.1.1.1.6.<ifIndex>`（ifHCInOctets、64bit版）も2回ポーリングし、32bit版（ifInOctets）と64bit版（ifHCInOctets）の差分値が一致する

## 5. 平文の危険性→SNMPv3移行 (Mission 5)
Given: Mission2で構築したSPAN（cap-sw、destination=cap）が有効であること。
参考: `参考資料/ネットワーク技術/ネットワーク「動作試験」入門/_index/text/p051.md`, `p052.md`（tcpdump/Wiresharkによるキャプチャ試験）
- [ ] **cap**で `tcpdump -i eth1 -w /tmp/snmp_v2c.pcap udp port 161` を実行しながら、**nms**から `snmpget -v2c -c public 10.28.90.1 1.3.6.1.2.1.1.5.0` を実行 → capのキャプチャファイルをWiresharkで開くと、UDPペイロード中にコミュニティ名 `public` がASCII平文で確認できる
- [ ] 監視対象機器に `snmp-server group NMS-GROUP v3 priv` と `snmp-server user nmsuser NMS-GROUP v3 auth sha SnmpAuth28! priv aes 128 SnmpPriv28!` を投入する
- [ ] **nms**から `snmpwalk -v3 -l authPriv -u nmsuser -a SHA -A 'SnmpAuth28!' -x AES -X 'SnmpPriv28!' 10.28.90.1 system` → v2c時と同じsystemサブツリーの値が返る
- [ ] **cap**で同様に `tcpdump -i eth1 -w /tmp/snmp_v3.pcap udp port 161` を取得しWiresharkで開く → PDUが暗号化されており、コミュニティ名やOID値がASCII平文では確認できない（nms以外では復号不能）
- [ ] 監視対象機器で `no snmp-server community public` を投入後、**nms**から `snmpwalk -v2c -c public 10.28.90.1 system` を実行 → 応答がなくTimeoutになる（旧v2c手順が失敗することを確認）

## 6. トラップ受信＋総合試験 (Mission 6)
Given: Mission5でSNMPv3移行が完了していること。
参考: `参考資料/ネットワーク技術/ネットワーク「動作試験」入門/_index/text/p074.md`, `p075.md`（SNMPトラップ試験）、`p109.md`（障害試験時のトラップ・監視運用の注意点）
- [ ] 監視対象機器に `snmp-server enable traps snmp linkdown linkup coldstart` と `snmp-server host 10.28.100.10 version 2c public` を投入する
- [ ] **nms**で `snmptrapd -f -Lo -c /tmp/snmptrapd.conf`（`authCommunity log public`＋`disableAuthorization yes` を設定済み。構築手順書§5参照）をフォアグラウンド起動し受信待機状態にする
- [ ] **old-sw1**で `interface Ethernet0/2` を `shutdown` → **nms**のsnmptrapd出力にold-sw1からの `linkDown` トラップが着信する
- [ ] 続けて `no shutdown` で復旧 → `linkUp` トラップが着信する
- [ ] 総合確認として、**nms**から6機器全ての監視対象IPへ `snmpwalk -v3 -l authPriv -u nmsuser -a SHA -A 'SnmpAuth28!' -x AES -X 'SnmpPriv28!' <IP> system` を実行し、全て成功する（Mission1〜6で構築した監視基盤が一通り機能していることを確認する）
