# 統合冗長（Track × HSRP × OSPF）チャレンジ

第2章の総仕上げです。
下流（LAN側）の「HSRP」と、上流（WAN側）の「OSPF」を稼働させます。
さらに、**「Track機能（IP SLA）」**を用いて「上流リンクの障害」を検知し、自動的にHSRPのActiveを切り替える（連動させる）という、エンタープライズネットワークで超頻出の構成を構築します。

---

## 🎯 トポロジ設計とIPアドレス

### 機器構成
- **R1**: インターネット側のコアルーター
- **SW1**: メインルーター（HSRP Active）
- **SW2**: サブルーター（HSRP Standby）
- **SW3**: L2スイッチ
- **PC1**: 検証用PC

### ログインIP一覧
- R1: `172.20.20.6` (telnet)
- SW1: `172.20.20.5` (telnet)
- SW2: `172.20.20.2` (telnet)
- SW3: `172.20.20.3` (telnet または ssh)
- PC1: `172.20.20.4` (telnet または ssh)
※ユーザ/パスはすべて `admin` / `admin`

### IPアドレス設計
**【WAN側（OSPFネットワーク）】**
- **R1 - SW1 間**: `10.1.1.0 /30` (R1: 10.1.1.1, SW1: 10.1.1.2)
- **R1 - SW2 間**: `10.1.2.0 /30` (R1: 10.1.2.1, SW2: 10.1.2.2)
- **R1 の Loopback0**: `8.8.8.8 /32` (インターネットの宛先として使用)

**【LAN側（HSRPネットワーク）】**
- **VLAN**: `VLAN 10`
- **ネットワーク**: `10.10.10.0 /24`
- **仮想GW (HSRP VIP)**: `10.10.10.254`
- **SW1**: `10.10.10.252`
- **SW2**: `10.10.10.253`
- **PC1**: `10.10.10.11` (デフォルトルートは 10.10.10.254 に向ける)

## 🏗️ 構築要件

### 1. LAN側（L2 / HSRP）の構築
前回のテーマと同様に、SW3からPC1へのVLANアクセス設定と、SW1/SW2でのトランク設定、そしてHSRP（グループ10、PriorityはSW1が110でActive）を設定してください。
※ 今回はTrackで切り替えるため、**必ず `standby 10 preempt` を有効にしてください。**

### 2. WAN側（OSPF）の構築
- R1, SW1, SW2 で OSPF（プロセスID `1`、エリア `0`）を稼働させます。
- 各機器の物理インターフェースにIPアドレスを設定します。
  - **R1はルーター**なので物理インターフェースにそのままIPを設定できます（`no switchport` は不要）。
  - **SW1, SW2はL3スイッチ**のため、R1と接続するインターフェースでは **`no switchport`** コマンドを入れて「ルーテッドポート」に変換してからIPを設定してください。
- `network` コマンドで接続しているネットワークをOSPFで広報してください。

### 3. Track機能の構築（今回のメイン！）
SW1 の「上流リンク（R1側）」がダウンした際、自動的にHSRPのPriorityを下げる設定を行います。

**【SW1での設定ヒント】**
1. Trackオブジェクトの作成（例: インターフェースのLine Protocolを監視）
   ```
   track 1 interface Ethernet0/1 line-protocol
   ```
2. HSRPにTrackを紐付ける（Track1がダウンしたらPriorityを20下げる）
   ```
   interface Vlan 10
   standby 10 track 1 decrement 20
   ```

---

## ✅ 最終検証
1. PC1 から `ping 8.8.8.8 repeat 100000` を実行。
2. Pingが通っている最中に、**SW1 の `Ethernet0/1`（R1側ポート）を `shutdown`** する。
3. Trackが発動し、SW1のPriorityが110から90に下がり、SW2（Priority 100）がActiveに昇格する。
4. OSPFの経路も自動的にSW2経由となり、Pingがほぼ無瞬断で復旧することを確認！
