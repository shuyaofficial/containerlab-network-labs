# 軽量検証結果 — OSSイメージarm64可用性（2026-07-04実測）

## 検証方法

clab VM（OrbStack Ubuntu 24.04 / arm64、docker 29.1.3）上で `docker manifest inspect <image>` を実行し、`linux/arm64` プラットフォームエントリの有無を確認した。manifest listが得られない・あいまいな場合のみ `docker pull` + `docker inspect --format '{{.Architecture}}'` で実イメージのarchを実測した。

## 総括（ZT6観点別の実装可能性判定）

判定基準: arm64あり = High（デプロイ不要で見込み確定）／arm64なし = 代替差し替えが必要。

1. **①ID統制（keycloak + openldap）** — **High**。両イメージともarm64ネイティブあり。IDプロバイダ+ディレクトリの組み合わせはそのまま実装可能。

2. **②デバイス統制（step-ca + osquery）** — **要代替あり**。step-caはarm64対応でHigh。osqueryはarm64非対応（amd64のみ実測確認）のため、デバイス統制（エンドポイント可視化・コンプライアンスチェック）部分は代替手段が必要。案: (a) osquery公式バイナリをソースからarm64向けにビルド、(b) Wazuh agentのosquery連携機能で代替、(c) 当面はamd64エミュレーション（QEMU経由、性能劣化を許容）で動作させる。

3. **③IAP（oauth2-proxy / pomerium / openziti）** — **High**。3つともarm64ネイティブあり。IAP層の実装可能性は確定。

4. **④SWG（mitmproxy / squid + suricata）** — **High**。mitmproxy・squid・suricataすべてarm64ネイティブあり。SWG（Secure Web Gateway）層は実装可能性が立った。

5. **⑤DLP（mitmproxy）** — **High**。mitmproxyはarm64対応済み（SWGと共用イメージ）。DLPのプロキシベース実装は見込み確定。

6. **⑥SIEM（grafana / loki / promtail + wazuh）** — **High（wazuhはタグ指定条件付き）**。grafana/loki/promtailは全てarm64ネイティブ。wazuh-managerは`latest`タグが存在しないためバージョンタグ（例: `4.14.6`）を明示指定する運用ルールが必要だが、実体はarm64対応済みでHigh判定。

### 総合結論

検証した17イメージのうち **15個はarm64ネイティブ対応を確認**（うちwazuhは`latest`タグ不在のためタグ指定必須という運用上の注意点あり）。**arm64非対応で代替が必要なのは clamav と osquery の2つ**のみ。ZT6の6観点のうち、代替が必要なのは「②デバイス統制」のosquery部分のみで、他5観点はイメージレベルでの実装可能性が確定した。
