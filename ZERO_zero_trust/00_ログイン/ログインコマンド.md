# テーマZERO｜ログインコマンド

VM への接続、Phase 0 ノードへのアクセス、Phase 1 以降の予定アクセス方法をまとめる。
Phase 1 以降の URL・認証情報は **各 Phase 実装時に確定**する（本ラボの今回スコープでは未デプロイ）。

## VM 接続

```bash
# OrbStack VM clab へ SSH
ssh clab@orb

# 接続確認
docker --version
containerlab version
```

## Phase 0 ノードへのアクセス（docker exec）

Phase 0 は `wbitt/network-multitool` を `client` / `app` / `external` に流用する。
コンテナ名は containerlab のプレフィックス規約により `clab-<lab名>-<node名>` になる（実際の名前は `docker ps` で確認）。

```bash
# 起動中コンテナ一覧
docker ps --format 'table {{.Names}}\t{{.Image}}\t{{.Status}}'

# client 端末に入る（シェル）
docker exec -it clab-zero-client bash

# app サービス側に入る
docker exec -it clab-zero-app bash

# external（外部/未管理端末）に入る
docker exec -it clab-zero-external bash
```

## エンドポイント疎通確認（Phase 0 ゲート）

```bash
# client → app への疎通（Phase 0 の合格条件）
docker exec clab-zero-client ping -c 3 app
docker exec clab-zero-client curl -sS http://app/

# ネットワーク所属の確認
docker exec clab-zero-client ip -4 addr show
```

## Phase 1 以降の予定アクセス方法（実装時に確定）

以下は目標像。**URL・ポート・認証情報は各 Phase 実装時に確定**する。現段階では設計上の見込み値。

| Phase | コンポーネント | 予定アクセス方法 | 確定時期 |
|---|---|---|---|
| 1 | Keycloak 管理画面 | `http://<VM>:8080/admin`（admin/初期PWは実装時に設定） | Phase 1 実装時に確定 |
| 1 | OIDC トークン取得 | `curl` で token endpoint に POST（realm/client 名は実装時に確定） | Phase 1 実装時に確定 |
| 2 | Pomerium / oauth2-proxy | `http://<VM>:<port>/`（保護対象 app への入口） | Phase 2 実装時に確定 |
| 3 | Grafana | `http://<VM>:3000/`（初期 admin/admin、実装時に変更） | Phase 3 実装時に確定 |
| 4 | mitmproxy Web | `http://<VM>:8081/`（mitmweb UI） | Phase 4 実装時に確定 |
| 6 | step-ca | `https://<VM>:9000/`（ACME/証明書発行） | Phase 6 実装時に確定 |

> Phase 1 以降は今回デプロイしないため、上記は設計値である。実装着手時にこの表を確定値へ更新する。
