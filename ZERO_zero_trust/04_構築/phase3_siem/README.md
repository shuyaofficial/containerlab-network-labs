# テーマZERO｜Phase 3 SIEM（Loki + Grafana）

> 今回スコープでは**未デプロイ**（設計値）。このスタブは実装着手時の作業起点。詳細は [段階ロードマップ](../../02_基本設計/段階ロードマップ.md) Phase 3。

## 目的

各観点のログを集約・可視化する基盤を早期に確保する（D-3）。Phase 4 以降で「セキュリティ機構が効いている証拠」を可視化するため、SWG/DLP より前に投入する。

## コンポーネント

| ノード | 役割 | ゾーン | 予定 IP | 待受 |
|---|---|---|---|---|
| loki | ログ保存 | 可観測 (zt-obs) | 172.30.30.41 | 3100/HTTP |
| promtail | ログ収集（各ノード→Loki） | 可観測 (zt-obs) | 172.30.30.42 | 送信のみ |
| grafana | 可視化 | 可観測 (zt-obs) | 172.30.30.43 | 3000/HTTP |

- 発展: Wazuh（統合 SIEM/EDR、2-4GB 級）。arm64 可用性・メモリ次第で Loki 一本に留める（D-4、[軽量検証計画](../../03_詳細設計/軽量検証計画.md) 代替ルール）。

## 依存

- Phase 1, 2（集約対象のログ源: Keycloak 認証ログ・Pomerium 認可ログ）。

## ゲート条件（次へ進む条件）

Phase 1/2 の認証・認可ログが Grafana のダッシュボードで表示できる（[要件定義書](../../01_要件定義/要件定義書.md) AC-3 / 試験 T-3-*）。

## 実装時の作業リスト

- [ ] `phase3_siem/` に clab.yml（loki/promtail/grafana、可観測ゾーン）と deploy.sh
- [ ] arm64 manifest 確認（loki/promtail/grafana は High 見込み）
- [ ] Promtail に各コンテナのログ収集ラベルを設定
- [ ] Loki を保存先に、片方向（生成元→Loki）で流入する構成にする
- [ ] Grafana に Loki をデータソース登録（初期 admin は実装時に変更）
- [ ] Phase 1/2 ログがクエリで見えることを確認（T-3-2）
- [ ] 未認証拒否・認証成功の件数を可視化するダッシュボードを作成（T-3-3）
- [ ] [構築ログ](../構築ログ_テンプレート.md) を複製して記録

## 学べること

ログ集約パイプライン、可観測ゾーンへの片方向流入、SIEM を効果測定の道具として使う考え方。
