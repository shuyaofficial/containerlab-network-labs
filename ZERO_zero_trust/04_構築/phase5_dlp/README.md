# テーマZERO｜Phase 5 DLP

> 今回スコープでは**未デプロイ**（設計値）。このスタブは実装着手時の作業起点。詳細は [段階ロードマップ](../../02_基本設計/段階ロードマップ.md) Phase 5。

## 目的

アップロード内容を検査し、機密キーワードを含むデータの持ち出しを検知・ブロックし、検知イベントを SIEM に記録する。DLP を OSS で代替する。

## コンポーネント

| ノード | 役割 | ゾーン | 予定 IP | 待受 |
|---|---|---|---|---|
| mitmproxy（アドオン） | DLP（内容検査・ブロック）※SWG と兼務 | DMZ（Untrust に足） | 172.30.10.31 | 8080 (proxy) |

- SWG（Phase 4）の mitmproxy に **Python アドオン**を同居させる（D-2）。同一プロキシ経路で検査と持ち出し制御を両立し、Suricata/c-icap の追加を回避する。
- 代替: c-icap + ClamAV が arm64 不可なら **mitmproxy DLP に一本化**（[軽量検証計画](../../03_詳細設計/軽量検証計画.md) 代替ルール）。

## 依存

- Phase 4（SWG の同一プロキシ経路上にアドオンを載せる）。

## ゲート条件（次へ進む条件）

キーワードを含むアップロードを検知・ブロックし、検知イベントを SIEM に記録できる（[要件定義書](../../01_要件定義/要件定義書.md) AC-5 / 試験 T-5-*）。

## 実装時の作業リスト

- [ ] Phase 4 の mitmproxy に DLP Python アドオンを追加
- [ ] 機密キーワード定義（設定は外部化。値は直書きしない）
- [ ] リクエストボディ（アップロード）を検査する処理を実装
- [ ] キーワード検出時にブロック応答を返す（T-5-2）
- [ ] 検知イベントをログ出力（Loki 集約対象）（T-5-3）
- [ ] 検知（キーワードあり）とブロックを確認（T-5-1）
- [ ] 誤検知/未検知の閾値を軽く確認（PoC 範囲）
- [ ] [構築ログ](../構築ログ_テンプレート.md) を複製して記録

## 学べること

DLP の内容検査、プロキシ経路への処理差し込み、検知と遮断の分離、共通基盤集約（SWG との兼務）の実装。
