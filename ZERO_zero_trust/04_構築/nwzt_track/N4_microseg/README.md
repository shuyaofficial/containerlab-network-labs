# テーマZERO｜N4 μセグメンテーション 構築スタブ

> 今回スコープでは**未デプロイ**（設計値）。このスタブは実装着手時の作業起点。詳細な定義・比較表は [NW-ZT トラックロードマップ](../../../02_基本設計/NW-ZT_トラックロードマップ.md) N4 を正とする。

## 目的

ゾーン内の east-west 通信を最小権限化し、侵害端末からの横移動（ラテラルムーブメント）を物理的に困難にする。ネットワーク層（VLAN/ACL）とホスト層（nftables）の二層でμセグメンテーションを実現する。

## コンポーネント

| ノード | 役割 |
|---|---|
| Cisco IOL（VLAN/ACL） | L2/L3 セグメントの分離とゾーン間制御 |
| ホスト nftables | ホスト単位の分散ファイアウォール（中央管理思想の縮図） |

## 依存

- N1（VLAN 基盤）。動的 VLAN 割当が先に成立している必要がある。
- テーマ22（既存の VLAN/ACL/HSRP 資産）を参照。

## ゲート条件（次へ進む条件）

- 同一ゾーン内でも、許可されていない端末間通信を ACL/タグで遮断できる。
- 横移動を模擬した通信が実際に遮断されることを検証で確認できる。

## 実装先

**テーマ31拡張＋テーマ22参照**。[ロードマップPHASE2](../../../../ロードマップ/PHASE2_MODERN_ENTERPRISE.md) を入口として参照する。テーマ22 は既存の VLAN/ACL 資産として直接参照する（拡張ではなく前提資産）。

## 実装時の作業リスト

- [ ] N1 の動的 VLAN 基盤が稼働していることを前提確認
- [ ] テーマ22 の既存 VLAN/ACL/HSRP 資産を棚卸し
- [ ] ゾーン内 east-west 通信の許可/拒否マトリクスを設計
- [ ] IOL 側 VLAN 間 ACL の設定投入
- [ ] ホスト側 nftables ルールの設計・投入
- [ ] 横移動を模擬した通信（許可端末間・非許可端末間）を発生させ遮断結果を確認
- [ ] ネットワーク層とホスト層、二層それぞれで遮断が効いていることを個別確認

## 学べること

TrustSec/SGT（Security Group Tag）の思想、east-west 通信制御の設計、L2/L3 とホストの二層μセグメンテーション、Illumio 的な分散ファイアウォール中央管理の考え方。
