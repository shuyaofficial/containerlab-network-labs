# テーマZERO｜試験計画書

各 Phase のゲート条件を試験 ID に落とし、合否で段階を進める。
今回スコープは **Phase 0 疎通試験** と **arm64 軽量検証（T-V*）** の実施。Phase 1-6（T-1*〜T-6*）は設計値で、実装時に実施する。
ゲート条件の出典は [段階ロードマップ](../02_基本設計/段階ロードマップ.md) と [要件定義書](../01_要件定義/要件定義書.md) 受け入れ基準。

## 試験方針

- **ゲート＝試験**: 各 Phase の「次に進む条件」をそのまま試験項目にする。合格しない限り次 Phase へ進まない。
- **証拠を残す**: 認証・拒否・検知は Phase 3 以降 Grafana で件数を可視化し、「効いている証拠」を記録する。
- **切り分けは層別で**: 不合格時は [切り分けシート](切り分けシート.md) を複製し、層別（コンテナ→到達→TLS→認証→認可→ログ）に切り分ける。
- 実施結果は `05_試験/試験結果_YYYY-MM-DD.md`（type: test-result）に記録する。

## Phase 1-6 ゲート試験（設計値・実装時に実施）

各 Phase のゲート条件を試験 ID 化した目標項目。実装着手時に手順を確定する。

| 試験ID | 項目 | 期待結果 | 対応 AC |
|---|---|---|---|
| T-1-1 | Keycloak 起動・管理画面到達 | admin にログインできる | AC-1 |
| T-1-2 | realm/client 作成 | 認可コードフロー用の client がある | AC-1 |
| T-1-3 | OIDC トークン取得 | `curl` またはブラウザでトークンが返る | AC-1 |

### Phase 2 — NWセキュリティ（IAP）

| 試験ID | 項目 | 期待結果 | 対応 AC |
|---|---|---|---|
| T-2-1 | 未認証拒否 | 未ログインで app にアクセス→拒否/リダイレクト | AC-2 |
| T-2-2 | 認証後到達 | Keycloak ログイン後は app に到達 | AC-2 |
| T-2-3 | 直通禁止 | Untrust から Trust への直通が不可 | AC-2 |

### Phase 3 — SIEM

| 試験ID | 項目 | 期待結果 | 対応 AC |

| T-3-1 | ログ収集 | Promtail が各ノードのログを Loki へ送る | AC-3 |

| T-3-3 | 拒否/許可可視化 | 未認証拒否・認証成功の件数が表示できる | AC-3 |

### Phase 4 — WEBセキュリティ（SWG）

### Phase 5 — DLP

## 合格基準（全体）

- 今回スコープ: T-0-* と T-V-* がすべて合格。
- Phase 進行: 各 Phase の T-N-* がすべて合格したら次 Phase へ進む。1つでも不合格なら [切り分けシート](切り分けシート.md) で切り分けてから再試験する。
