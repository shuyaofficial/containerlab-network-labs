- `external`（172.30.0.12）は Untrust ゾーンに同居するのみで、Phase 0 では通信試験対象外（Phase 4 SWG 検証用の外部先として起動確認のみ）。

## 4. この結果の読み方

- 「テーマZERO の全要件を完遂した」という結果ではない。**Phase 0（土台）のみ完遂**。Phase 1-6 は設計値のまま、今回はデプロイしていない。
- `zt-untrust` / `zt-trust` は containerlab の `bridge` kind（ホスト Linux bridge）で表現し、ゾーン間は関所（Phase 2 の Pomerium/oauth2-proxy）がまだ無いため、**ゲート確認限定でホスト L3 中継**（ブリッジに GW IP 付与 + `ip_forward=1` + 各ノードへの静的経路）を暫定的に張った。論理構成設計の「Phase 0 では `client`→`app` 疎通はゲート確認のための設計上の許可経路」という記述に対応する実装判断であり、Phase 2 で関所が入った時点でこの暫定経路は関所転送に置き換える前提。
- 本番の「Untrust→Trust 直通禁止」は Phase 2（IAP/関所）で実現するものであり、Phase 0 の暫定中継はその前段階の土台に過ぎない。

## 5. 残課題

- Phase 1（Keycloak）以降は未着手（設計のみ、今回スコープ外）。
- Phase 0 の暫定 L3 中継（ホストでのブリッジ間ルーティング）は、Phase 2 で関所を導入する際に見直しが必要（関所のマルチホーム経由に置き換え、ホスト中継は撤去する）。
- 撤収済み（`./deploy.sh destroy`）。VM はクリーンな状態に戻している。
