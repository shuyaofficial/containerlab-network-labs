# VRRP ラボ 構築チャレンジ要件

このラボでは、デフォルトゲートウェイの冗長化を実現するオープン標準プロトコル **VRRP (Virtual Router Redundancy Protocol)** を構築します。
HSRPとほぼ同じですが、コマンドやデフォルトの挙動（Preemptなど）がわずかに異なります。ご自身の力で設定し、障害時の無瞬断切り替えを体感してみてください！

---

## 🎯 構築チャレンジ要件

### 1. 機器構成（トポロジ）
以下の4台のノードを使用します。

- **SW1 (L3 Core/Dist)**
  - 役割: メインルーター（VRRP Master）
- **SW2 (L3 Core/Dist)**
  - 役割: サブルーター（VRRP Backup）
- **SW3 (L2 Access)**
  - 役割: PCやルーター群を収容するL2スイッチ
- **PC1 (End Host)**
  - 役割: 検証用エンドホスト（疑似PC）

### 2. IPアドレスとVLANの設計
- **使用するVLAN**: `VLAN 10`
- **ネットワークアドレス**: `10.10.10.0 /24`
- **VRRP Virtual IP (仮想GW)**: `10.10.10.254`

各機器のIP割り当ては以下の通りです。

* **SW1**: `10.10.10.252 /24` (VLAN10のSVI)
* **SW2**: `10.10.10.253 /24` (VLAN10のSVI)
* **PC1**: `10.10.10.11 /24` (VLAN1上に設定し、仮想IPへデフォルトルートを向ける)

### 3. 設定要件 (要件のみ・正解コマンドなし)

**【SW1 / SW2 の要件 (L3設定)】**
1. **IPルーティングの有効化**: スイッチ上でIPルーティングを有効にしてください。
2. **VLAN 10 の作成とトランク設定**: SW3と繋がる物理インターフェースをトランクポートとして設定し、VLAN 10 を許可してください。
3. **SVI の作成**: `interface vlan 10` を作成し、指定された物理IPアドレスを割り当ててください（`no shut` をお忘れなく！）。
4. **VRRP の設定**:
   - グループ番号は `10`、Virtual IP は `10.10.10.254` を設定してください。
   - **[SW1のみ]**: 優先度（priority）を `110` に設定し、確実に Master になるようにしてください（デフォルトは100）。
   - **[SW2のみ]**: 優先度（priority）を `90` に設定し、Backup になるようにしてください。
   - ※ヒント: HSRPの `standby` コマンドが、VRRPでは `vrrp` に変わります。また、VRRPはデフォルトでPreempt（自動切り戻し）が有効なため、Preemptのコマンド設定は不要です。

**【SW3 の要件 (L2設定)】**
1. **VLAN 10 の作成**: VLAN 10を作成してください。
2. **トランクポート設定**: SW1 および SW2 と繋がるポートをトランクポートとし、VLAN 10 を許可してください。
3. **アクセスポート設定**: PC1 と繋がるポートを VLAN 10 のアクセスポートに設定し、`spanning-tree portfast` を有効にしてください。

**【PC1 の要件】**
1. **IP設定**: 疑似PCのため、`interface vlan 1` にIPアドレスを設定してください。
2. **デフォルトゲートウェイ**: VRRP Virtual IP（`10.10.10.254`）へ向けたデフォルトルートを設定してください（※ `no ip routing` は不要です）。

---

## ✅ 最終検証のゴール
1. 各ルーターで VRRP の状態を確認し（`show vrrp` など）、**SW1がMaster、SW2がBackup**になっていること。
2. PC1から Virtual IP (`10.10.10.254`) へ連続してPingを打つ (`ping 10.10.10.254 repeat 100000`)。
3. そのPingを打ちながら **SW1 の SVI（または物理ポート）を shutdown** する。
4. Pingのロスが最小限で済み、通信が瞬時に **SW2 に切り替わって疎通し続けること**を確認する。
5. **SW1 を no shutdown して復旧**させたとき、Preemptを設定していなくても自動的にSW1が Master に戻ることを確認できれば大成功です！
