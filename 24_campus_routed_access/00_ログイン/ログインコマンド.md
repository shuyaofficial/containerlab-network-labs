# Theme 24 機器別ログインコマンド

## 1. 実行場所

以下はすべてOrbStack Linux VM `clab`で実行します。

```bash
ssh clab@orb
cd /Users/shuya/Documents/claude/Mac仮想環境構築/24_campus_routed_access
```

## 2. 稼働確認


```bash
cd 04_構築
./deploy.sh inspect
```

6台すべてが`running`であることを確認します。

## 3. Cisco IOLへのログイン

Theme 23と同じく、IOLをPID1として起動し、`docker attach`の標準入力をIOSへ直結しています。

画面がすぐ表示されない場合は、接続後にEnterを1〜2回押してください。初回はCiscoの`System Configuration Dialog`が表示されます。セットアップウィザードを使うかどうかは自分で選択します。

### core1

```bash
sudo docker attach --sig-proxy=false clab-campus-routed-access-core1
```

### core2

```bash
sudo docker attach --sig-proxy=false clab-campus-routed-access-core2
```

### access1

```bash
sudo docker attach --sig-proxy=false clab-campus-routed-access-access1
```

### access2

```bash
sudo docker attach --sig-proxy=false clab-campus-routed-access-access2
```

### コンテナを止めずに離脱

`Ctrl-p`を押し、続けて`Ctrl-q`を押します。`Ctrl-c`は離脱には使いません。

## 4. Linux端末へのログイン

### pc-a

```bash
sudo docker exec -it clab-campus-routed-access-pc-a bash
```

### pc-b

```bash
sudo docker exec -it clab-campus-routed-access-pc-b bash
```

Linuxシェルからは`exit`で抜けます。

## 5. SSHについて

containerlab公式の標準管理方法はSSHですが、このラボの旧IOLイメージでは初期SSH bootstrapが完了しません。自分でSSH管理設定を作成するまでは、次の形式を使いません。

```bash
ssh admin@clab-campus-routed-access-core1
```

## 6. ログインできない場合

### 6.1 コンテナ確認

```bash
sudo docker ps --format 'table {{.Names}}\t{{.Status}}' | grep campus-routed-access
```

### 6.2 IOLがPID1か確認

```bash
sudo docker top clab-campus-routed-access-core1
```

`/iol/iol.bin`がPID1のプロセスとして表示されることを確認します。

### 6.3 起動ログ確認

```bash
sudo docker logs --tail 50 clab-campus-routed-access-core1
```

### 6.4 iouyap再起動

```bash
cd 04_構築
./deploy.sh iouyap
```

### 6.5 ラボを作り直す場合

> 保存していないIOS設定は消えるため、必要な設定を保存してから実行します。

```bash
cd 04_構築
./deploy.sh destroy
./deploy.sh deploy
```
