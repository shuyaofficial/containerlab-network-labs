要件定義書: ゼロトラスト_ネットワーク特化

v1.0 draft 最終更新 2026-07-08 対象: N1〜N4（L7層 P0-P6 は対象外）

1. 背景・目的#

ゼロトラスト完全版は仮想企業（1拠点・約100名）の全社アーキテクチャを設計したが、統合ポイント（I-1〜I-8）・E2Eフロー（F-A〜F-E）の大半は「◐ 設計のみ（接続未実施）」に留まった。本テーマは、そのうちネットワーク層に閉じる部分（N1 NAC・N2 ZTNA・N3 NDR・N4 μセグ）を選び、単一の実在するcontainerlabトポロジ上に統合し、実デプロイまで行うことで、完全版の「接続◐」を「実際に動く」段階へ進める。L7層（Keycloak/IAP/SIEM/mTLS等 P0-P6）は本テーマのスコープ外とし、完全版の設計のままとする。

構築部材（nwzt.clab.yml・core_sw_merged.cfg・deploy-all.sh・ztna/setup_ziti.sh・ndr/suricata/local.rules）は完成済みであり、本書はそれと整合する設計書として書く。

2. 対象範囲#

対象

  • N1 NAC（802.1X）: core-sw を認証者、radius(FreeRADIUS) を認証サーバとし、認証成功端末を動的VLAN、未認証端末を隔離VLANへ振り分け
  • N4 μセグ: core-sw の inter-VLAN ACL（層1・粗い分離）と、サーバ側ホスト nftables（層2・同一VLAN内の細かい分離）の二層構成
  • N3 NDR: サーバ室ブリッジ nwzt-srv0 を Suricata が host mode で監視し、east-west の SYN スキャンを検知して Loki/Grafana に集約
  • N2 ZTNA（SDP/OpenZiti）: 保護対象サーバ srv-app1 をダークサービス化し、overlay 経由のみ到達可能にする

対象外

  • L7層（ID統制/IAP/SWG/DLP/デバイス統制、ゼロトラスト完全版の P0-P6）
  • I-4（検知→対応の自動CoA連携）の自動化実装

ID要件主要値検証ゲート

5. ゾーン設計方針#

社内LANは 172.31.0.0/16、第3オクテット＝VLAN ID とする（完全版のIPアドレス管理表を無変更で踏襲）。VLAN100（コア・認証管理）を中核に、営業(VLAN10)・開発(VLAN20)・ゲスト(VLAN30)・サーバ室(VLAN50)・隔離(VLAN99)へ分離し、監視SOC（論理VLAN90、host mode）が全体を横断監視する。詳細は03 IPアドレス管理表を参照。

6. 制約#

ID制約詳細
REQ-C01arm64制約OrbStack VM (Ubuntu 24.04 arm64) 上で実行する。IOLはx86イメージのarm64エミュレーションのため起動に約2分かかる
REQ-C02iouyap必須IOLのデータプレーンは deploy 直後に iouyap(513) を起動しないと全断する
REQ-C03config投入は expect の1 attach直列docker attach の多重起動は禁止
REQ-C04サーバ室ブリッジの事前作成nwzt-srv0(docker network, GW .254) は containerlab deploy より前に作成しておく必要がある（kind:bridge が既存ブリッジを要求するため）
REQ-C05分担方針の一時的な上書き通常運用の分担方針（構築はユーザー本人、Claudeは設計書・部材担当）を、本テーマに限りユーザー指示により上書きし、Claudeが実デプロイ検証まで行う
REQ-C06I-4は対象外検知→対応の自動CoA隔離は完全版同様に◐設計のまま。本テーマではホスト nftables の手動投入によるデモに留める

7. 受け入れ基準#

  • README・00〜05のドキュメント一式が frontmatter必須7項目・H1単一・章立て規約を満たすこと
  • 記載するIP・ノード名・ポート割当が実ファイル（nwzt.clab.yml・core_sw_merged.cfg）と一致すること
  • 05_試験/試験計画書.md の B1〜B6 ゲートが定義されていること（判定は実デプロイ後に記録するため、本テーマ作成時点では空欄／未実施表記で良い）
  • 05_試験/試験結果_*.md は本タスクでは作成しない（実デプロイ後にメインセッションが実測値で記録する）
  • node 規約/ビルド/build.mjs --check で本テーマ分のNFD/H1複数/frontmatter欠落エラーが無いこと
