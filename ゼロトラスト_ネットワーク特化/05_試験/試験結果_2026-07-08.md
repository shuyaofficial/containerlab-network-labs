## 検証エビデンス（要点）

### B2 802.1X 動的VLAN（F-A）
- pc-sales で `wpa_supplicant -i eth1 -D wired`（alice/EAP-MD5）起動 → `wpa_cli status`:
  `Supplicant PAE state=AUTHENTICATED / suppPortStatus=Authorized / EAP state=SUCCESS`。
- 認証成功後 pc-sales に VLAN10 データIP（172.31.10.101/24）を付与、SVI Vlan10（172.31.10.1）へ疎通。
- RADIUS 到達は SVI Vlan100（172.31.0.1）を source-interface として FreeRADIUS 172.31.0.10 へ。

### B3 マイクロセグメンテーション（F-E）
- 層1（inter-VLAN ACL `SEG10-OUT`、Vlan10 in）: 営業 172.31.10.0/24 → srv-app1 172.31.50.11 は
  tcp/80 許可・tcp/22 拒否・icmp 許可。実測: 80=OPEN, 22=BLOCKED, icmp=0%loss。
- 層2（ホスト nftables）: srv-app1 に `ip saddr 172.31.50.31 counter drop` を投入 →
  同一VLAN50 内の host-infected→srv-app1 東西通信が 0%loss から 100%loss に遷移。
  ＝VLAN/ACL では止まらない同一VLAN内横移動をホスト分散FWで遮断。

### B4 NDR east-west 検知（F-C の検知部）
- サーバ室 VLAN50 を docker bridge `nwzt-srv0` として実体化し、Suricata を `--network host -i nwzt-srv0` で
  東西トラフィックを SPAN 不要で DPI。
- host-infected(172.31.50.31)→srv-app1(172.31.50.11) へ `nmap -sS` → eve.json に
  `sid:1000001` alert（"ZT-NDR east-west SYN scan"）を記録。Promtail→Loki に 277行取込・LogQL 抽出可。
- ※F-C の「検知→CoA自動隔離」の自動化部分（SOAR→CoA）は完全版同様に未実装（設計 ◐）。
  本テーマでは層2 nft を手動投入して隔離相当を実演。

### B5 SDP型ZTNA ダークサービス（F-D）
- srv-app1 を OpenZiti のダークサービスとして overlay に張り出し（apptun が nwzt-srv0 に接続し dial）。
- clienttun（＝リモート社員端末、zn-ziti のみ）から: overlay 経由（localhost:8080）= HTTP 200／
  直接 172.31.50.11:80 = HTTP 000（到達不能）。
- ダーク性の担保: サーバ室ブリッジのホストGW（172.31.50.254）を除去し、docker ホストが
  zn-ziti→VLAN50 を L3 転送しないようにした（apptun は同一ブリッジ L2 で到達を維持）。

## 残課題

- B2b: 未認証端末の VLAN99 自動割当（IBNS 1.0 の no-response action を 15.2/advipservices で要精査）。
  現状は「未認証＝全通信不可（封じ込め）」までで、隔離VLANへの自動移動は未確認（T-N1-7 継承）。
- I-4（検知→CoA自動隔離）の自動化（Suricata→SOAR-lite→RADIUS CoA）は未実装。
  本テーマは検知（B4）と隔離手段（B3層2）を個別に実証し、両者の自動連結は今後の発展課題。

## 再現手順（要約）

```bash
cd ".../ゼロトラスト_ネットワーク特化/04_構築"
sudo ./deploy-all.sh deploy    # bridge作成 + clab deploy + iouyap + fix_dataplane
sudo ./deploy-all.sh config    # 統合cfg投入（expect）
sudo ./deploy-all.sh auth      # 802.1X認証 + サーバサービス起動
sudo ./deploy-all.sh ndr       # Suricata + Loki/Grafana
sudo ./deploy-all.sh ziti      # OpenZiti overlay（ダーク化含む）
sudo ./deploy-all.sh verify    # B2/B3 疎通確認
sudo ./deploy-all.sh scan; sudo ./deploy-all.sh eve   # B4 検知確認
sudo ./deploy-all.sh nft       # B3 層2 遮断
sudo ./deploy-all.sh destroy   # 全撤去
```

実行環境: OrbStack VM `clab`（Ubuntu 24.04 / arm64）、containerlab 0.77.0、docker 29.x。
IOL は L2-advipservices-2017（802.1X＋L3 SVI＋ACL を単一機に集約。probe で dot1x 受理を事前確認）。
