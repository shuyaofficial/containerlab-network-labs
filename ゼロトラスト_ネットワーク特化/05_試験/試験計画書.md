# 試験計画書: ゼロトラスト_ネットワーク特化

## 試験方針

N1(NAC)・N4(μセグ)・N3(NDR)・N2(ZTNA)を統合した単一トポロジが、[基本設計書](../02_基本設計/基本設計書.md) の設計通りに実際のデプロイで動作することを、B1〜B6の6ゲートで確認する。**2026-07-08 に実デプロイ検証を実施済み**。判定と実測エビデンスは [試験結果_2026-07-08.md](試験結果_2026-07-08.md) を正とする（B1〜B6 は合格、B2b 未認証→VLAN99 自動割当のみ ◐ 残課題）。本書は各ゲートの手順・期待結果を定義する。

| B1 | 基盤 | clab deploy + iouyap + config反映 | `deploy-all.sh deploy` → `config` を実行 | 全ノードrunning、core-swの全SVIがup/up、`write memory`完了 | 実デプロイ後に記録 |
| B2 | N1 NAC | 動的VLAN割当 | pc-salesでサプリカント起動、pc-unauthは未起動のまま放置 | pc-sales(Et0/1)がAUTHORIZED・VLAN10、pc-unauth(Et1/0)がVLAN99（no-response） | 実デプロイ後に記録 |

### B1 基盤疎通
- [ ] `core-sw` で `show ip interface brief | include Vlan` を実行し、Vlan10/20/30/50/99/100 が up/up であることを確認する。
- [ ] `show authentication sessions` が実行できること（構文エラーが出ないこと）を確認する。

### B2 N1 動的VLAN
- [ ] `core-sw` で `show authentication sessions` を実行し、pc-sales(Et0/1)がVLAN10でAUTHORIZEDと表示されることを確認する。
- [ ] `core-sw` で pc-unauth(Et1/0)がno-responseによりVLAN99へ割当されていることを確認する。

### B3 N4 μセグ（2層）
- [ ] pc-sales→srv-app1:80がHTTP 200で到達することを確認する。
- [ ] pc-sales→srv-app1:22がACL `SEG10-OUT` のdenyで遮断されることを確認する。
- [ ] nft適用前、host-infected→srv-app1が到達可能であることを対照として確認する。
- [ ] `deploy-all.sh nft` 適用後、host-infected→srv-app1が遮断されることを確認する。

### B4 N3 検知
- [ ] `deploy-all.sh scan` でhost-infected→srv-app1へSYNスキャンを実行する。
- [ ] `deploy-all.sh eve` でsid:1000001のalertが出力されることを確認する。
- [ ] Grafana(:3000)でLoki経由のログが可視化されることを確認する。

### B5 N2 overlay到達
- [ ] clienttunからoverlay経由（localhost:8080）でsrv-app1にHTTP 200で到達することを確認する。
- [ ] clienttunから直接（172.31.50.11:80）は到達不能であることを確認する。

### 総合障害試験
- [ ] core-swの `dot1x timeout tx-period` 到達後の再認証挙動を確認する（サプリカント再起動で再認証されること）。
- [ ] `deploy-all.sh nft-flush` でN4層2ルールを解除し、host-infected→srv-app1の到達が復旧することを確認する。

## 合格基準

- B1〜B5の必須項目が全て合格していること。
- B6（E2E通し）でB1〜B5の結果に矛盾がないこと。
- 総合障害試験で、再認証・nft解除後にそれぞれ期待通りの状態へ復旧すること。
- 上記が確認でき次第、`05_試験/試験結果_YYYY-MM-DD.md` を新規作成し実測値を記録する（本テーマの範囲外、実デプロイ後にメインセッションが実施）。
