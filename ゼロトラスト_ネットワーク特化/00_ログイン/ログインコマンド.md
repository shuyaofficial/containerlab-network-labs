# ゼロトラスト_ネットワーク特化 機器別ログインコマンド

## 1. VM接続

以下はすべて OrbStack Linux VM `clab` で実行する。

```bash
ssh clab@orb
cd /path/to/Mac仮想環境構築/ゼロトラスト_ネットワーク特化/04_構築
```
<!-- 上記パスはVM内での作業ディレクトリの例。本文に実際の絶対パスは書かない。 -->

## 2. 稼働確認

```bash
./deploy-all.sh inspect
```

`core-sw`・`radius`・`pc-sales`・`pc-dev`・`guest-pc`・`pc-unauth`・`srv-app1`・`host-infected` が `running` であることを確認する（`nwzt-srv0` は clab ノードではなく docker bridge のため `inspect` には現れない）。

N2/N3 は clab 管理外の docker コンテナのため別途確認する。

```bash
sudo docker ps --format 'table {{.Names}}\t{{.Status}}' | grep -E 'ziti|apptun|clienttun|suricata|loki|promtail|grafana'
```

## 3. 機器コンソール（docker attach）

IOL コアスイッチのみ対象。抜けるのは `Ctrl-p → Ctrl-q`（`Ctrl-c` は使わない）。

### core-sw
```bash
sudo docker attach --sig-proxy=false clab-nwzt-lan-core-sw
```

## 4. エンドポイント接続（docker exec）

### clab管理下のLinuxノード（radius / pc-sales / pc-dev / guest-pc / pc-unauth / srv-app1 / host-infected）
```bash
sudo docker exec -it clab-nwzt-lan-<node> sh
```
`exit` で抜ける。

### N2 ZTNA（clab管理外・docker run起動）
```bash
sudo docker exec -it ziti sh        # controller + router
sudo docker exec -it apptun sh      # srv-app1側 tunneler（zn-ziti + nwzt-srv0）
sudo docker exec -it clienttun sh   # リモート社員端末役（zn-zitiのみ）
```

### N3 NDR（clab管理外・docker run起動、`--network host`）
```bash
sudo docker exec -it suricata sh
sudo docker logs --tail 50 promtail
```
Grafana UI は VM 内 `http://localhost:3000`（匿名Admin有効、`GF_SECURITY_ADMIN_PASSWORD=admin`）、Loki API は `http://localhost:3100`。Macから見る場合はVM側でのポート転送を別途行う。

## 5. ログインできない場合

### 5.1 コンテナ確認
```bash
sudo docker ps --format 'table {{.Names}}\t{{.Status}}' | grep clab-nwzt-lan
```

### 5.2 起動ログ確認
```bash
sudo docker logs --tail 50 clab-nwzt-lan-core-sw
```

### 5.3 iouyap未起動を疑う場合
IOL はデータプレーンに `iouyap`(513) が必須。全断（コンソールは繋がるがpingが通らない）の場合はまずこれを疑う。

```bash
./deploy-all.sh iouyap
```

### 5.4 ラボを作り直す場合
> 保存していない設定は消える。
```bash
./deploy-all.sh destroy
./deploy-all.sh deploy
```
