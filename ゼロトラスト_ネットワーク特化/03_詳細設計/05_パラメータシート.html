2. core-sw#

8. N4 層2（ホストnftables）#

項目値
対象srv-app1
ルールtable inet filter → chain input {type filter hook input priority 0; policy accept;} → rule ip saddr 172.31.50.31 counter drop（host-infected を拒否）
適用/解除deploy-all.sh nft / deploy-all.sh nft-flush
◐ 残課題：同一VLAN内は層1 ACL の対象外のため層2 nftables で補うが、検知→対応（CoA自動隔離, I-4）はスコープ外。本テーマでは deploy-all.sh nft の手動投入デモに留める。
