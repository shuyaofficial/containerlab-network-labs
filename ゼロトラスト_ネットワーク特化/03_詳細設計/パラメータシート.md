# パラメータシート（詳細設計）— ゼロトラスト_ネットワーク特化

本テーマは Claude 実装テーマであり（[要件定義書](../01_要件定義/要件定義書.md) 制約）、設定値を明記する。原本は [core_sw_merged.cfg](../04_構築/core_sw_merged.cfg) 等の実ファイルであり、本書はそれを設計値として整理したもの。実際の投入コマンド列は [CLI投入コマンド.md](CLI投入コマンド.md) を参照。

## 1. 共通設計

| 項目 | 値 |
|---|---|
| 認証方式 | 802.1X（`aaa authentication dot1x default group radius` / `aaa authorization network default group radius`） |
| RADIUSサーバ | 172.31.0.10:1812（認証）/ 1813（アカウンティング）、key `radlab123` |
| ip routing | 有効化（core-sw が SVI間ルーティングを行う） |
| VLAN一覧 | 10(SALES) / 20(DEV) / 30(GUEST) / 50(SERVER) / 99(QUARANTINE) / 100(MGMT) |

## 2. core-sw

### 2.1 インターフェース

| IF | 接続先 | 種別 | VLAN/認証 | 備考 |
|---|---|---|---:|---|
| Et0/1 | pc-sales | access | dot1x → 動的VLAN10 | `authentication event no-response action authorize vlan 99` |
| Et0/2 | pc-dev | access | 20（静的） | |
| Et0/3 | guest-pc | access | 30（静的） | |
| Et1/0 | pc-unauth | access | dot1x → 隔離VLAN99 | Et0/1と同一の認証設定 |
| Et1/1 | radius | access | 100 | RADIUS到達用 |
| Et1/2 | nwzt-srv0 | access | 50（uplink） | サーバ室ブリッジへの接続 |

### 2.2 L2設計

| 項目 | 設計値 |
|---|---|
| VLAN10 | name SALES |
| VLAN20 | name DEV |
| VLAN30 | name GUEST |
| VLAN50 | name SERVER |
| VLAN99 | name QUARANTINE |
| VLAN100 | name MGMT |
| spanning-tree | 全アクセスポートで `portfast` |

### 2.3 ルーティング設計（SVI）

| SVI | IP | 備考 |
|---|---|---|
| Vlan10 | 172.31.10.1/24 | `ip access-group SEG10-OUT in` |
| Vlan20 | 172.31.20.1/24 | |
| Vlan30 | 172.31.30.1/24 | |
| Vlan50 | 172.31.50.1/24 | サーバ室（nwzt-srv0とGW非衝突: docker bridge側GWは.254） |
| Vlan99 | 172.31.99.1/24 | 隔離 |
| Vlan100 | 172.31.0.1/24 | `ip radius source-interface Vlan100` |

### 2.4 セキュリティ設計

| 項目 | 設計値 |
|---|---|
| dot1x（Et0/1・Et1/0） | `authentication port-control auto` / `dot1x pae authenticator` / `dot1x timeout tx-period 10` / `dot1x max-reauth-req 2` |
| no-response時の挙動 | `authentication event no-response action authorize vlan 99`（隔離VLANへ自動割当） |
| ACL `SEG10-OUT`（Vlan10 in適用） | `permit tcp 172.31.10.0/24 host 172.31.50.11 eq 80` → `permit icmp 172.31.10.0/24 172.31.50.0/24` → `deny tcp 172.31.10.0/24 host 172.31.50.11 eq 22` → `permit ip any any` |

## 3. radius（nac-freeradius コンテナ）

| 項目 | 値 |
|---|---|
| インターフェース | eth1: 172.31.0.10/24（clab.yml execで付与） |
| NASクライアント | `172.16.0.0/12` 全体を許可、secret `radlab123`（[clients.conf](../04_構築/nac_freeradius/clients.conf)） |
| 認可プロファイル | ユーザー `alice`／`alicepass`、`Tunnel-Private-Group-Id="10"`（[authorize](../04_構築/nac_freeradius/authorize)） |
| デフォルト | `DEFAULT Auth-Type := Reject`（未登録ユーザーは拒否） |

## 4. エンドポイント（業務端末）

|---|---|---|---|---|
| pc-sales | 172.20.40.21 | 172.31.10.101/24（認証後付与） | 802.1X動的 | wpa_supplicant: identity=`alice`, password=`alicepass`, eap=MD5（[wpa_supplicant.conf](../04_構築/nac_client/wpa_supplicant.conf)） |
| pc-dev | 172.20.40.22 | 172.31.20.101/24 | 静的（clab.yml exec） | ― |
| guest-pc | 172.20.40.23 | 172.31.30.101/24 | 静的（clab.yml exec） | ― |
| pc-unauth | 172.20.40.24 | 割当なし | サプリカント未起動→no-response | ― |

## 5. サーバ室（VLAN50 / nwzt-srv0）

| ノード | データIP | サービス/ツール | 備考 |
|---|---|---|---|
| srv-app1 | 172.31.50.11/24 | `python3 -m http.server 80`、`ncat -lk -p 22 -c 'echo NWZT-SRV-APP1-SSH-OK'` | N2ダークサービス対象、N4層2 nft遮断対象 |
| host-infected | 172.31.50.31/24 | nmap（`wbitt/network-multitool`） | 東西SYNスキャン源 |

両ノードとも `ip route replace 172.31.0.0/16 via 172.31.50.1` でVLAN間経路を確保する。

## 6. N2 ZTNA（OpenZiti）

| 項目 | 値 |
|---|---|
| コントローラ+ルータ | `ziti` コンテナ（`openziti/ziti-cli:latest`）、`edge quickstart` |
| app側tunneler | `apptun`（`zn-ziti` + `nwzt-srv0` 接続、固定IP 172.31.50.50） |
| client側tunneler | `clienttun`（`zn-ziti` のみ＝直接到達不可） |
| サービス設定 | `webapp-host`（host.v1、protocol=tcp、address=172.31.50.11、port=80） |
| サービス | `webapp`（configs=webapp-host） |
| identity | `apphost`(role: hosts) / `webclient`(role: clients) |
| ポリシー | `webapp-bind`(Bind, `@webapp`↔`#hosts`) / `webapp-dial`(Dial, `@webapp`↔`#clients`) / `allEr` / `allSerp` |
| tunneler起動 | apptun: `ziti tunnel host`／clienttun: `ziti tunnel proxy webapp:8080` |

詳細は [ztna/setup_ziti.sh](../04_構築/ztna/setup_ziti.sh)。

## 7. N3 NDR（Suricata + Loki/Grafana）

| 項目 | 値 |
|---|---|
| 監視IF | `-i nwzt-srv0`（host mode、promiscuous） |
| HOME_NET | 172.31.50.0/24 |
| ルール sid:1000001 | `flags:S; threshold:type both,track by_src,count 15,seconds 10` — 東西SYNスキャン検知 |
| ルール sid:1000002 | `flow:to_server` — srv-app1:80への直接アクセスを記録（可視化補助） |
| Loki | :3100 |
| Grafana | :3000、匿名Admin有効、`GF_SECURITY_ADMIN_PASSWORD=admin`（ラボ用途） |

詳細は [ndr/suricata/local.rules](../04_構築/ndr/suricata/local.rules)。

## 8. N4 層2（ホストnftables）

| 項目 | 値 |
|---|---|
| 対象 | srv-app1 |
| ルール | `table inet filter` → `chain input {type filter hook input priority 0; policy accept;}` → `rule ip saddr 172.31.50.31 counter drop`（host-infectedを拒否） |
| 適用/解除 | `deploy-all.sh nft` / `deploy-all.sh nft-flush` |

## 9. 期待する経路と疎通結果

| 送信元 | 宛先 | 期待結果 | 対応ゲート |
|---|---|---|---|
| pc-sales | srv-app1:80 | 到達（HTTP 200） | B3 |
| pc-sales | srv-app1:22 | 遮断（ACL deny） | B3 |
| host-infected | srv-app1（nft適用前） | 到達 | B3対照 |
| host-infected | srv-app1（nft適用後） | 遮断 | B3 |
| host-infected | srv-app1（SYNスキャン） | Suricata sid:1000001 alert | B4 |
| clienttun | localhost:8080（overlay経由） | 到達（HTTP 200） | B5 |
| clienttun | 172.31.50.11:80（直接） | 到達不能 | B5 |
