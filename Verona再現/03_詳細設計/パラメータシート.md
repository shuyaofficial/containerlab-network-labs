# パラメータシート（詳細設計）— Verona再現

> 本ラボはIOS/CLI投入ではなく `docker run` パラメータが設定値そのものであるため、
> 本書は [04_構築/deploy.sh](../04_構築/deploy.sh) の変数定義と1対1で対応させる。

## 1. ネットワーク設計

| ネットワーク名 | CIDR | bridge名 | 用途 | Verona対応ゾーン |
|---|---|---|---|---|
| vsase-cloud | 172.60.10.0/24 | vsase-br0（固定。Suricata host mode監視対象） | ZTNAコントローラ・SWG群・FWaaS監視点・発展機能 | Verona SASE（クラウド基盤） |
| vsase-branch | 172.60.20.0/24 | （docker自動割当） | branch-clientのホームLAN | 拠点（Verona Edge配下） |
| vsase-dark | 172.60.30.0/24 | （docker自動割当） | 保護対象アプリ専用。clientは非参加 | ダークサービス化された保護対象リソース |

## 2. ポート設計（host公開ポート）

| サービス | host port | container port | プロトコル | 用途 |
|---|---|---|---|---|
| Squid | 3128 | 3128 | tcp | SWG明示プロキシ（URLフィルタ） |
| Blocky | 5353 | 53 | udp/tcp | SWG DNSフィルタ（host 53番回避） |
| Blocky | 4000 | 4000 | tcp | Blocky HTTP（クエリログ/メトリクス想定） |
| Keycloak | 8081 | 8080 | tcp | IdP Admin Console（発展） |
| Headscale | 8082 | 8080 | tcp | 拠点間トンネルcoordinator（発展） |
| step-ca | 9000 | 9000 | tcp | デバイスポスチャーCA（発展） |

> host 53番ポートは意図的に避け5353を使用（VM/Macホスト側のDNS設定と衝突させないため）。
> 他テーマ（ALog再現想定のGrafana 3001/Loki 3101/MinIO 9001等）とは非重複。

## 4. 証明書/PKIパラメータ（step-ca・発展）

| 項目 | 値 |
|---|---|
| CA名 | Verona-Lab-CA（`DOCKER_STEPCA_INIT_NAME`） |
| DNS names | step-ca, localhost, 127.0.0.1（`DOCKER_STEPCA_INIT_DNS_NAMES`） |
| 初期パスワード | verona-lab-posture（`DOCKER_STEPCA_INIT_PASSWORD`。ラボ内限定の平文値、本番利用不可） |
| CA待受ポート | 9000（container/host共通） |
| クライアント証明書の想定用途 | check_posture.sh の第1引数（デフォルト `/tmp/posture/client.crt`） |

## 5. フィルタ対象ドメイン（[squid/blocklist.txt](../04_構築/squid/blocklist.txt) と共用）

| ドメイン | 分類（Verona p42相当のコメント） | 用途 |
|---|---|---|
| malware.verona-lab.test | Security層相当 | ブロック検証 |
| c2.verona-lab.test | Security層相当 | ブロック検証 |
| phishing.verona-lab.test | Security層相当 | ブロック検証 |
| crypto-mining.verona-lab.test | Enterprise層相当 | ブロック検証 |
| adult-content.verona-lab.test | Family層相当 | ブロック検証 |
| safe.verona-lab.test | ―（許可ケース） | Blocky customDNS mappingで172.60.10.13（safe-site）に解決。許可検証 |

すべて `.test`（IANA予約TLD、RFC 2606）を用い、実在ドメインとの衝突を避ける。

## 6. Suricataパラメータ（FWaaS/IDS）

| 項目 | 値 |
|---|---|
| 監視インターフェース | vsase-br0（host mode） |
| HOME_NET | 172.60.10.0/24 |
| EXTERNAL_NET | any |
| ルールファイル | [suricata/local.rules](../04_構築/suricata/local.rules) |
| sid:2000001 | recon SYN scan検知（10秒間に同一送信元から15回以上のSYN） |
| sid:2000002 | HTTP通信の可視化用ログ（classtype: not-suspicious） |
| eve.json出力先 | `04_構築/suricata/log/eve.json`（bind mount） |

## 8. 期待する到達性（コア機能）

| 送信元 | 宛先 | 経路 | 期待結果 |
|---|---|---|---|
| clienttun | protected-app:80（直接） | vsase-cloud→vsase-dark（無し） | 到達不能（unreachable） |
| clienttun | localhost:8080（overlay経由） | OpenZiti overlay→apptun→protected-app | HTTP 200 |

| branch-client | safe.verona-lab.test（Blocky DNS） | vsase-cloud→blocky（customDNS） | 172.60.10.13 |
| branch-client | malware.verona-lab.test（Blocky DNS） | vsase-cloud→blocky（denylist） | 0.0.0.0 |
