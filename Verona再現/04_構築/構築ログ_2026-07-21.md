# Verona再現 構築ログ（2026-07-21）

メインセッションが OrbStack VM `clab`（arm64）へ実機デプロイし、コア4機能（T-V1〜T-V4）＋
発展3機能の基盤動作（T-V5〜T-V7）すべての動作を確認した。

## 到達点（実証済み）

- **コア4機能（OpenZiti / Squid / Blocky / Suricata）は初回デプロイで全て成功し、修正不要
  だった。** `deploy-core` → `setup-ziti` → `test-swg` → `test-ids` の一連の手順で
  T-V1〜T-V4がそのまま合格。
- **発展3機能（step-ca / Keycloak / Headscale）のうち、step-ca・Keycloakも修正不要で動作。**
  Headscale（拠点間トンネル）のみ、バージョン固有の設定要件（entrypoint・DNS・DERP）で
  3点の調整を要した（詳細は下記「詰まりどころと解決」）。
- 調整後、Headscaleコーディネータの起動、および `headscale users create` /
  `headscale preauthkeys create` によるuser作成・事前認証キー発行までを実機で確認した。
  tailscaleクライアントの実参加（データプレーン疎通）は次段階（未実施）。

## 詰まりどころと解決（Headscale・3点）

### 1. コンテナ起動コマンドの二重指定（`unknown command headscale`）

- **事象**: `deploy-tunnel` で `"$IMG_HEADSCALE" headscale serve` としてコンテナを起動すると、
  `unknown command "headscale" for "headscale"` 相当のエラーで即終了する。
- **原因**: `headscale/headscale` イメージの `ENTRYPOINT` は既に `headscale` バイナリを
  指しているため、コマンド引数に重ねて `headscale serve` を渡すと
  `headscale headscale serve` という二重呼び出しになってしまう。
- **対処**: `04_構築/deploy.sh` の `deploy-tunnel` の起動コマンドを `"$IMG_HEADSCALE" serve`
  （`serve` 単体）に修正（メインセッションが実施）。

### 2. `dns.override_local_dns` の既定値による起動失敗

- **事象**: config.yaml の `dns` セクションが最小構成（`magic_dns` / `base_domain` のみ）だと
  Headscaleがconfig検証エラーで起動しない。
- **原因**: v0.29.2は `dns.override_local_dns` の既定値が `true` であり、`true` の場合
  `dns.nameservers.global` の明示が必須。本ラボの最小構成にはこの項目が無かった。
- **対処**: `04_構築/headscale/config.yaml` の `dns` セクションに
  `override_local_dns: false` と `nameservers.global: [1.1.1.1]` を追加
  （メインセッションが実施。`false` にしても `nameservers.global` 自体は要求されるため
  両方を明示する必要があった）。

### 3. DERPMap非空要件による起動失敗

- **事象**: `dns` 修正後もなお起動に失敗する。
- **原因**: v0.29系は DERPMap（DERPリレーサーバの一覧）が非空でなければならない。本ラボは
  外部インターネット上のTailscale公式DERPに依存させたくなかったため、`derp.urls` を空
  （`[]`）のままにしていたが、その場合の代替（埋め込みDERP）が未設定だった。
- **対処**: `04_構築/headscale/config.yaml` に `derp.server` セクションを新規追加し、
  埋め込みDERPサーバを有効化（`enabled: true`、`region_id: 999`、
  `private_key_path: /var/lib/headscale/derp_server_private.key`）。外部DERPに依存せず
  ラボ内で完結させる方針を維持したまま起動要件を満たした（メインセッションが実施）。

## 再現手順（後から自分で構築する場合）

```bash
cd Verona再現/04_構築
./deploy.sh deploy-core     # OpenZiti/Squid/Blocky/Suricata（コア4機能）
./deploy.sh setup-ziti      # ZTNAダークサービス構成＋実証（T-V1）
./deploy.sh test-swg        # URL/DNSフィルタ検証（T-V2/T-V3）
./deploy.sh test-ids        # IDS検知検証（T-V4）
./deploy.sh deploy-posture  # step-ca（T-V5）
./deploy.sh deploy-idp      # Keycloak（T-V6）
./deploy.sh deploy-tunnel   # Headscale（T-V7。上記3点の設定調整を反映済みのconfig.yamlを使用）
./deploy.sh destroy         # 片付け
```
