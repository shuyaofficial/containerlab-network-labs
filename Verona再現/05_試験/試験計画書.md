# 試験計画書: Verona再現

> **2026-07-21 実施済み。T-V1〜T-V7 全項目合格。** 実測値・エビデンスは [試験結果_2026-07-21.md](試験結果_2026-07-21.md) を参照（本計画書の各行「判定」列は結果ドキュメントに集約）。

## 試験方針

Verona再現ラボが [基本設計書](../02_基本設計/基本設計書.md) の設計通りに動作することを、
T-V1〜T-V7の7項目で確認する。T-V1〜T-V4はコア機能（[04_構築/deploy.sh](../04_構築/deploy.sh)
の `deploy-core`／`test-swg`／`test-ids` で確実に検証できる想定）、T-V5〜T-V7は発展機能
（単独起動の確認に留め、完全な連携疎通までは求めない）。**本書は本タスク（ファイル作成のみ）
の時点では実デプロイ未実施のため、判定は全て「未実施」である。** 実デプロイ・実測記録は
メインセッションが別途 `05_試験/試験結果_YYYY-MM-DD.md` として作成する。

### T-V1 ZTNAダークサービス疎通
- [ ] `./deploy.sh deploy-core` でコア9コンテナが起動することを確認する。
- [ ] `./deploy.sh setup-ziti` でサービス/ポリシー作成・enrollment・tunneler起動が成功することを確認する。
- [ ] clienttunからoverlay経由（localhost:8080）でprotected-appにHTTP 200で到達することを確認する。
- [ ] clienttunから直接（protected-app:80）は到達不能であることを確認する。

### T-V2 SWG URLフィルタ
- [ ] `./deploy.sh test-swg` を実行する。
- [ ] branch-client→squid(3128)→safe.verona-lab.test がHTTP 200で到達することを確認する。
- [ ] branch-client→squid(3128)→malware.verona-lab.test がHTTP 403でブロックされることを確認する。

### T-V3 SWG DNSフィルタ
- [ ] branch-client→blocky で safe.verona-lab.test を問い合わせ、172.60.10.13が返ることを確認する。
- [ ] branch-client→blocky で malware.verona-lab.test を問い合わせ、0.0.0.0が返ることを確認する。

### T-V4 FWaaS IDS検知
- [ ] `./deploy.sh test-ids` でbranch-client→squidへSYNスキャンを送る。
- [ ] `suricata/log/eve.json` にsid:2000001のalertイベントが記録されることを確認する。

### T-V5 デバイスポスチャー拒否
- [ ] `./deploy.sh deploy-posture` でstep-caが起動することを確認する。
- [ ] 証明書ファイルが存在しない状態で `check_posture.sh` を実行し、NG(exit 1)が返ることを確認する。
- [ ] 証明書を発行し `VERONA_EDR_VERSION` を要件未満に設定して実行し、NG(exit 1)が返ることを確認する。
- [ ] 証明書あり・`VERONA_EDR_VERSION` が要件以上の場合にOK(exit 0)が返ることを確認する。

### T-V6 IdP SSO（発展）
- [ ] `./deploy.sh deploy-idp` でKeycloakコンテナが起動することを確認する。
- [ ] `http://localhost:8081/` のAdmin Consoleにadmin/adminでログインできることを確認する。
- [ ] realm/client作成手順（[keycloak/README.md](../04_構築/keycloak/README.md)）が実行可能であることを確認する（完全なSSOフローの疎通までは求めない）。

### T-V7 拠点間トンネル（発展）
- [ ] `./deploy.sh deploy-tunnel` でHeadscaleコンテナが起動することを確認する。
- [ ] Headscaleのuser作成・preauthkey発行手順（[headscale/README.md](../04_構築/headscale/README.md)）が実行可能であることを確認する（2拠点間の実疎通までは求めない）。

## 合格基準

- T-V1〜T-V4（コア機能）の必須項目が全て合格していること。
- T-V5〜T-V7（発展機能）は、単独起動が確認できれば合格とする（完全な連携疎通は合格基準に含めない）。
- 本書作成時点（ファイル作成タスク）では実デプロイを行っていないため、上記は全て「未実施」。
  実デプロイ後、メインセッションが `05_試験/試験結果_YYYY-MM-DD.md` を新規作成し実測値を記録する。
