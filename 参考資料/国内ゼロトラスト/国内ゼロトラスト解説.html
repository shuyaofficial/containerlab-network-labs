2なぜ今「ゼロトラスト」なのか

セキュリティの世界でよく聞く「ゼロトラスト何も無条件に信頼せず、アクセスのたびに検証するというセキュリティの考え方。」という言葉。これを理解するには、まず従来の考え方だった「境界型防御社内ネットワークの内側は安全、外側は危険という前提で、境界（ファイアウォールなど）を守る従来型の防御の考え方。」がなぜ限界を迎えたのかを知る必要があります。

従来の考え方：「城と堀」の境界型防御

これまで多くの企業は、「社内ネットワークの内側にいれば安全、外側は危険」という前提でセキュリティを考えてきました。社内と社外の境界にファイアウォールなどの「壁」を築き、その壁の内側だけを守るという発想です。ちょうどお城の周りに堀を掘り、城壁の中だけを守るイメージに近いといえます。

図：境界型防御＝「城と堀」のイメージ。壁（境界）の内側は無条件に信頼される。

その前提が崩れた3つの理由

ところが近年、この「壁の中は安全」という前提そのものが通用しなくなってきました。理由は主に、テレワークの普及とクラウドサービスの利用拡大により、社員が社外から社内システムやSaaSにアクセスする機会が急増したためです。壁の「外」と「内」の境目があいまいになり、境界を守るだけでは不十分になったのです。

さらに、実際のランサムウェアファイルを勝手に暗号化し、復旧と引き換えに身代金を要求するマルウェア（悪意あるソフトウェア）。被害の多くは、次の3つの経路から侵入しています。

図：ランサムウェアの主な侵入経路3パターン（Verona資料の課題提起の図を再構成）。

ゼロトラストという発想への転換

こうした状況を踏まえて広まったのが「ゼロトラスト」という考え方です。ゼロトラストとは、社内・社外という場所で信頼を判断するのではなく、誰が・どの端末で・何にアクセスしようとしているかを、そのつど検証するという発想です。「一度社内に入れたら後は自由」ではなく、「アクセスのたびに毎回確認する」という点が、境界型防御との決定的な違いです。

この違いは、従来のVPNVirtual Private Networkの略。インターネット上に専用トンネルを作り、社外から社内ネットワークへ接続する技術。とゼロトラストを比べるとわかりやすくなります。従来のVPNは、いったん認証されると社内ネットワークに広くアクセスできてしまうため、もし攻撃者に一度侵入されると、社内のあらゆるサーバーへ被害が広がる（横展開（ラテラルムーブメント）侵入した攻撃者が、最初に侵入した端末から内部ネットワークの他のシステムへ次々と移動し、被害範囲を広げる行為。）リスクを抱えています。一方でゼロトラストの考え方では、「ユーザーAはサーバーAだけにアクセスできる」というように、アクセスできる範囲を必要最小限に絞り込みます。

図：VPN＝一度入れば広範囲にアクセス可能／ゼロトラスト＝ユーザーごとに必要な範囲だけ許可（Verona資料の図を再構成）。
💡たとえ話で理解する
従来のVPNは「会社の鍵をまるごと1本渡す」ようなもの。一度渡してしまえば、その人はどの部屋にも入れてしまいます。ゼロトラストは「必要な部屋の鍵だけをその都度渡す」イメージです。同じ社員でも、経理部屋の鍵は経理担当だけに、サーバー室の鍵は情シス担当だけに、というように必要な分だけアクセスを許可します。

製品解説① カンタンSIEM

ALog — ログを集めて異常を自動で見つける

ALogは、株式会社網屋が提供する「カンタンSIEM」というキャッチコピーのSIEMログを収集・分析してセキュリティ対策に役立てる仕組み・製品カテゴリ。製品です。専門知識がなくても始められることを重視して設計されています。

解決する課題：「証拠」がないと調査できない

ALog資料は、ログの重要性をこう説明しています。セキュリティインシデントが発生したとき、9割以上の組織で十分なログ管理ができておらず、「攻撃された証拠」を得られていません。実際、調査可能なログが存在しない組織は60%にのぼるという調査結果もあります（グローバルセキュリティエキスパート社の年間300件の被害対応相談実績より）。ログがなければ、侵入の兆候を見逃す、原因の特定ができない、内部不正を検知できない、といった問題につながります。

製品解説② フルマネージドSASE

Verona — ネットワークとセキュリティをクラウドでまとめて守る

Veronaは、株式会社網屋が提供するSASESecure Access Service Edge。ネットワークとセキュリティをクラウド上で統合するモデル。製品で、「フルマネージドSASE」を掲げています。専門要員がいなくても運用を任せられることが最大の特徴です。

解決する課題：VPN依存の限界

前章で見た通り、従来のVPNには「常時ポートが開放されている」「一度突破されると社内を横展開されるリスクがある」という弱点があります。Veronaは、このVPNVirtual Private Network。インターネット上に専用トンネルを作り社内ネットワークへ接続する技術。依存の限界を解決するために設計されています。

3つの構成要素

Veronaは、大きく3つの要素で構成されています。

Verona SASE

ネットワーク機能とセキュリティ機能を提供するクラウド基盤。全体の中心となる部分です。

Verona Edge

拠点（本社・支社・工場など）で使うSASEルータ。拠点間の安全なプライベートアクセスを提供します。

Verona Client

テレワークなどリモート端末にインストールするソフト。証明書認証で安全にアクセスします。

図：Verona SASEを中心に、拠点はVerona Edge、リモート端末はVerona Clientで接続し、SaaS/IaaSとも連携する構成。

5大機能

ZTNAゼロトラストネットワークアクセス。端末やユーザーごとにアクセスを検証し、必要な範囲だけ接続を許可する仕組み。

接続前に端末のセキュリティ状態を自動チェックする「デバイスポスチャー」機能、社外端末を常時Verona SASEに接続させる「起動強制」機能、就業時間外の新規接続を制限する「接続時間制御」機能などを備えます。

SWGSecure Web Gateway。URLフィルタリングやDNSセキュリティなど、Web通信を検査・制御する機能区分。

91カテゴリに基づくURLフィルタリングで危険サイトへのアクセスをブロックするほか、DNSレイヤーでの通信チェックである「DNSセキュリティ」により、Web以外の通信も含めて不正な通信を検知します。

FWaaSFirewall as a Service。IPS/IDS、アンチウイルス、アンチスパムなどを含むファイアウォール機能区分。

IPS/IDS（不正アクセス検知・防御）、アンチウイルス（ウイルスやスパイウェアの検知・ブロック）、アンチスパム（スパムメールの削減）を包括的に提供します。

CASBCloud Access Security Broker。クラウドサービス（アプリケーション）の利用状況を可視化・制御する仕組み。

4,800以上のアプリケーションを識別できるコントロールリストにより、業務に必要なアプリだけを許可したり、情報漏えいの危険があるアプリの使用を禁止したりできます。

SD-WAN拠点間通信を効率よく制御・最適化するネットワーク技術。

メイン回線がダウンした際にサブ回線へ自動フェイルオーバーする「WAN冗長化」や、AWS・Azure・GCPなど複数クラウドへの接続に対応する「IaaS接続」を提供します。

目玉機能：ダイナミックポートコントロール

Veronaの大きな特徴が「ダイナミックポートコントロール」です。多くのランサムウェア攻撃はVPNの脆弱性を悪用しますが、Veronaは通信の出入口となるポートを常時開放せず、証明書と紐づいた正規端末からの認証要求があったときだけポートを開放します。不正なユーザーはそもそもVeronaの存在自体を認識できないため、認証要求すらできません。

図：鍵を強くする発想 vs 存在そのものを見えなくする発想（ダイナミックポートコントロールのイメージ）。
💡たとえ話で理解する
一般的なセキュリティ強化は「家の鍵をより頑丈にする」発想です。しかしダイナミックポートコントロールは、「家の鍵を強くする」のではなく「家の場所そのものを外から見えなくする」発想に近いといえます。そこに家（＝アクセスできる入口）があることさえわからなければ、そもそも泥棒（攻撃者）は近づくことすらできません。

なぜ「フルマネージド」が重要なのか

Verona資料は、日本企業にフルマネージド（運用まるごと代行）型のSASEが向いている理由を、IT人材の所在データで説明しています。日本ではIT人材の72%がベンダー企業側に在籍している一方、ユーザー企業側は28%にとどまります。対照的に米国ではユーザー企業側に65%のIT人材が在籍しています（情報処理推進機構「IT人材白書2017」を基に作成）。

つまり、自社エンジニアでの運用を前提に作られた海外製のSASEは、IT人材の多くがベンダー側にいる日本企業にとって、自社運用が難しいという構造的なミスマッチがあります。Veronaはこの点を踏まえ、専門のセキュリティエンジニアが運用・設定管理・障害対応を代行する「フルマネージド」を標準で提供しています。

通信速度への工夫

特定のクラウドサービスへは本社を経由せず拠点から直接インターネットへ接続する「ローカルブレイクアウト特定のクラウドサービスへの通信を本社経由にせず、拠点から直接インターネットへ接続させる方式。回線負荷の軽減や速度向上に寄与する。」により、回線の混雑を回避しながら通信品質を向上させます。また、従来のPPPoE方式より最大10倍の通信速度を実現するIPoEIPv4 over IPv6。従来のPPPoE方式に比べて混雑の影響を受けにくく、高速な接続を実現する新しい接続方式。方式対応ルータにも対応しています。

73つの資料の全体像

最後に、ここまで見てきた3つの資料がどう関係しているかを整理します。ひとことで言うと、Verona（防御）とALog（検知）という2つの製品を、NATURE SERIES（人的サービス）が取り巻いている、という構造です。

図：VeronaからALogへのログ転送（Verona資料p66）と、NIST CSFフェーズ上の位置づけ。

Veronaはネットワークの入口や通信そのものを守る「防御」の役割を担い、実際にVerona資料の中でも、Veronaで取得した通信ログをALogへ転送して分析する連携が紹介されています。一方でALogは、あらゆるシステムのログを集めて異常を見つけ、万一のときの証跡を残す「検知」の役割を担います。

そして、これら2つの製品だけでは対応しきれない「導入前の計画」「診断」「教育」「有事の対応」「復旧」といった人手が必要な領域を、NATURE SERIESのコンサルティング・診断・教育・インシデント対応サービスが取り巻いてカバーする、という構図になっています。NIST CSFの6フェーズで言えば、Veronaは「防御」、ALogは「検知」に位置づけられ、残りの「統治」「特定」「対応」「復旧」はNATURE SERIESの各種サービスが担当します。

8用語集

ここまでに登場した専門用語を辞書形式でまとめました。

SIEM（シーム）
Security Information and Event Managementの略。さまざまなシステムのログを収集・分析し、セキュリティ対策に活用する仕組み・製品カテゴリ。ALogはこの分野の国産製品。
SASE（サシー）
Secure Access Service Edgeの略。ネットワーク機能とセキュリティ機能をクラウド上で統合して提供するモデル。Veronaはこの分野のフルマネージド製品。
ZTNA
ゼロトラストネットワークアクセス。ユーザーや端末ごとにアクセスを検証し、必要なリソースだけへの接続を許可する仕組み。
SWG（Secure Web Gateway）
URLフィルタリングやDNSセキュリティなど、Web通信を検査・制御する機能区分。
FWaaS（Firewall as a Service）
IPS/IDS、アンチウイルス、アンチスパムなどをクラウドで提供するファイアウォール機能。
CASB（Cloud Access Security Broker）
クラウドサービス（アプリケーション）の利用状況を可視化・制御する仕組み。
SD-WAN
拠点間の通信を効率よく制御・最適化するネットワーク技術。回線の冗長化やクラウド接続の最適化を含む。
EDR（Endpoint Detection and Response）
PCやサーバーなど端末上の不審な挙動を検知・記録し、対応する仕組み。SentinelOneはこの分野の製品。
MDR（Managed Detection and Response）
監視から分析・報告・改善までのセキュリティ運用を専門チームが代行するサービス形態。ALog MDRサービスなど。
SOC（Security Operation Center）
ログやアラートを監視・分析する専門拠点・チーム。
CSIRT（Computer Security Incident Response Team）
組織内でセキュリティインシデントが発生した際に対応する専門チーム。
IDaaS（Identity as a Service）
クラウド上でID管理・認証を提供するサービス（Entra ID、Oktaなど）。Veronaは証明書認証とIDaaS認証を組み合わせた2要素認証に対応。
ローカルブレイクアウト
特定のクラウドサービスへの通信を本社経由にせず、拠点から直接インターネットへ接続させる方式。回線負荷の軽減や速度向上に寄与する。
C&Cサーバ（Command & Control）
マルウェアが外部の攻撃者と通信するために使うサーバ。感染端末はC&Cサーバからの指令を受けて動作する。
DNSトンネリング
DNSクエリ（名前解決の問い合わせ）を悪用して、C&Cサーバとの通信を確立する攻撃手法。
ランサムウェア
ファイルを勝手に暗号化し、復旧と引き換えに身代金を要求するマルウェア。
フィッシング
偽のメールやWebサイトを使って、パスワードなどの情報を盗み取る攻撃手法。
横展開（ラテラルムーブメント）
侵入した攻撃者が、最初に侵入した端末から内部ネットワークの他のシステムへ次々と移動し、被害範囲を広げる行為。
境界型防御
社内ネットワークの内側は安全、外側は危険という前提で、ファイアウォールなどの境界を守る従来型の防御の考え方。
フォレンジック
インシデント発生後に端末やネットワーク機器、サーバーのログを詳細解析し、原因・被害範囲・証拠を明らかにする調査手法。
ペネトレーションテスト
攻撃者目線で実際にシステムへの侵入を試み、現行の対策の有効性やリスクの有無を明らかにするテスト。
アタックサーフェス
攻撃者から狙われうるIT資産・接点の範囲。ドメインやIPアドレスをもとに調査するサービスがある。
NIST CSF 2.0
米国国立標準技術研究所（NIST）が策定した世界標準のサイバーセキュリティフレームワーク。統治・特定・防御・検知・対応・復旧の6フェーズで対策を整理する。
ダイナミックポートコントロール
通信の出入口となるポートを常時開放せず、正規端末からの認証要求時のみ動的に開放する仕組み。Veronaの目玉機能。
デバイスポスチャー機能
接続前に端末のセキュリティ状態（ドメイン参加有無、セキュリティソフトの稼働状況など）を自動チェックする機能。
ISMS / ISO/IEC 27001
情報セキュリティマネジメントシステムの国際規格・認証制度。組織の情報資産を適切に管理する体制を評価する。
IPS/IDS
通信パケットを監視し、ネットワークへの不正アクセスを検知・防御する仕組み（侵入防止システム／侵入検知システム）。
