OSS Reproduction & Verification Report

商用ゼロトラスト製品をOSSで再現できるか？

このページは、株式会社網屋（AMIYA）の商用製品ALog（SIEMSecurity Information and Event Managementの略。ログを収集・分析してセキュリティ対策に活用する仕組み・製品カテゴリ。）とVerona（SASESecure Access Service Edgeの略。ネットワークとセキュリティの機能をクラウド上でまとめて提供するモデル。）を、OSS（オープンソースソフトウェア）だけでどこまで再現できるかを検証したレポートです。仮想環境（arm64 / OrbStack / containerlab）の中に「ALog再現」「Verona再現」という2つのラボを新たに構築し、実機デプロイして機能を実際に動かした結果をまとめています。

セキュリティ製品やSIEM・SASEといった専門知識がゼロの読者でも読めるように、専門用語にはホバー説明この点線下線の付いた語にカーソルを合わせる（スマホでは軽くタップする）と、簡単な説明が吹き出しで表示されます。を付けています。

この資料の使い方： 「2. なぜ再現するのか」で背景を押さえたあと、「3. ALog再現」「4. Verona再現」でそれぞれの再現スタックと実機検証結果（✅合格した項目の実測値）を確認し、「5. まとめ」で2製品を通じた再現度の全体像をつかむ、という順に読み進めると理解しやすい構成です。

2前提: なぜ再現するのか

ALogやVeronaのような商用ゼロトラスト製品は、月額数十万円クラスのライセンス費用がかかる上、内部の実装は基本的にブラックボックスです。「なぜそのアラートが発報されるのか」「なぜこのポートは外から見えないのか」を、実際に自分の手でOSSを組み合わせて動かしてみることで、商用製品が解決している課題の本質を学ぶ。それが今回2つのラボを新設した狙いです。

💡たとえ話で理解する
完成品の自動車を眺めているだけでは、エンジンがどう動いているかは分かりません。同じ部品（OSS）をひとつずつ組み立てて動かしてみると、「アクセルを踏むと何が起きるか」が体で理解できます。今回のラボは、ALogとVeronaという2台の「完成車」を、市販部品（OSS）だけでどこまで分解・再構築できるかに挑戦したものです。

既存ラボを製品視点で統合・拡張

今回の2ラボは、ゼロから作ったわけではありません。このリポジトリには、以前から個別の技術要素を検証した既存ラボがあり、それらを「商用製品のどの機能に対応するか」という製品視点で束ね直し、足りない機能を追加する形で構築しています。

再現ラボ① SIEM

ALog再現 — ログを集めて異常を自動検知する国産SIEM

ALogは、ログを集めて異常を自動で見つけるSIEMSecurity Information and Event Management。ログを収集・分析してセキュリティ対策に活用する仕組み・製品カテゴリ。製品です。本ラボでは、その中核機能（多様なログソース収集・翻訳変換・3観点AI検知・アラート通知・S3長期保管）を、OSSで組み上げたスタックとして再現し、OrbStack VM上で実機デプロイして動作検証しました。

再現スタック構成

syslog（AD/サーバ/NW機器を模擬）とSuricataオープンソースのネットワーク侵入検知/防御エンジン。パケットを解析し不正な通信パターンを検知する。の検知ログという2系統のログ源を、それぞれVectorCNCFのログ・メトリクス集約ツール。VRL（Vector Remap Language）というミニ言語でログの変換・正規化ができる。とPromtailGrafana Loki用のログ収集エージェント。ログファイルを監視しLokiへ転送する。経由でLokiGrafana社が開発する軽量ログ集約システム。ログにラベルを付けて保管し、LogQLというクエリ言語で検索・集計できる。へ集約し、Grafanaオープンソースの可視化・アラーティングツール。ダッシュボード表示や閾値ベースのアラート発報を行う。で3観点検知とアラート通知、MinIOS3互換のオープンソースオブジェクトストレージ。AWS S3と同じAPIでセルフホストできる。で長期保管を担う構成です。

図：ALog再現の再現スタック。syslog系とSuricata系の2系統のログをLokiに集約し、MinIOへの長期保管とGrafanaでの3観点検知・Webhook通知に分岐する。

再現ラボ② SASE

Verona再現 — ネットワークとセキュリティをクラウドでまとめて守る国産SASE

Veronaは、ネットワーク機能とセキュリティ機能をクラウド上でまとめて提供するSASESecure Access Service Edge。ネットワークとセキュリティの機能をクラウド上で統合するモデル。製品です。本ラボでは、その5大機能（ZTNAゼロトラストネットワークアクセス。ユーザーや端末ごとにアクセスを検証し、必要なリソースだけへの接続を許可する仕組み。 / SWGSecure Web Gatewayの略。URLフィルタリングやDNSセキュリティなど、Web通信を検査・制御する仕組み。 / FWaaSFirewall as a Serviceの略。IPS/IDSなどをクラウドで提供するファイアウォール機能。 / CASB / SD-WAN）を個別OSSで組み合わせて再現し、OrbStack VM上で実機デプロイして動作検証しました。

再現スタック構成

「SASEクラウド」（OpenZitiオープンソースのZTNA（ゼロトラストネットワークアクセス）プラットフォーム。証明書ベースの認証で、必要な相手だけに動的に通信経路を開く。＋Squid＋Blocky＋Suricata）、「拠点」（branch-client）、保護対象アプリ（ダークサービス公開ポートを一切持たず、認証済みのオーバーレイ経由でのみ到達可能なサービス。外部からは存在自体が見えない。）、そして発展機能（step-ca / Keycloak / Headscale）という4つのゾーンで構成されています。

図：Verona再現の再現スタック。branch-clientはSquid/Blockyでフィルタ検証し、OpenZiti経由の認証済みオーバーレイのみがダークサービスprotected-appに到達できる（直接アクセスは到達不能）。

5まとめ: 再現できたこと・できなかったこと

2つのラボを通じて分かったのは、「中核データパイプライン」と「ゼロトラストの核心となる仕組み」はOSSで実機再現できる、という点です。一方で、商用製品ならではの運用の作り込みや大規模化・完全自動化の部分は、再現の対象外としています。誇張せず、両方をそのまま並べて示します。

6用語集

ここまでに登場した専門用語を辞書形式でまとめました。

SIEM（シーム）
Security Information and Event Managementの略。さまざまなシステムのログを収集・分析し、セキュリティ対策に活用する仕組み・製品カテゴリ。ALogはこの分野の国産製品。
SASE（サシー）
Secure Access Service Edgeの略。ネットワーク機能とセキュリティ機能をクラウド上で統合して提供するモデル。Veronaはこの分野のフルマネージド製品。
ZTNA
ゼロトラストネットワークアクセス。ユーザーや端末ごとにアクセスを検証し、必要なリソースだけへの接続を許可する仕組み。本ラボではOpenZitiで再現。
SWG（Secure Web Gateway）
URLフィルタリング（危険なWebサイトへのアクセス遮断）やDNSセキュリティ（DNSレイヤーでの通信チェック）など、Web通信を検査・制御する機能区分。本ラボではSquid（URL）とBlocky（DNS）の二層で再現。
FWaaS（Firewall as a Service）
IPS/IDS、アンチウイルス、アンチスパムなどをクラウドで提供するファイアウォール機能。本ラボはSuricataによるIDS（検知のみ）で再現。
IDS / IPS
通信パケットを監視し、ネットワークへの不正アクセスを検知する仕組みがIDS（侵入検知システム）、検知した通信を自動遮断するのがIPS（侵入防止システム）。本ラボはSuricataでIDSモードのみを再現し、IPSのインライン遮断は対象外。
デバイスポスチャー機能
接続前に端末のセキュリティ状態（証明書の有無、OS種別、EDRの稼働状況など）を自動チェックする機能。本ラボはstep-ca発行の証明書＋擬似EDRチェックのモックで再現。
IdP（Identity Provider）
ユーザー認証・IDを一元管理し、他のサービスへ認証結果を提供する仕組み。本ラボではKeycloakを利用。
OIDC（OpenID Connect）
OAuth 2.0を拡張した認証プロトコル。IdPがユーザー認証結果をトークンとして発行する標準規格。本ラボではKeycloakのOIDC discoveryエンドポイントの応答を確認。
SD-WAN
拠点間の通信を効率よく制御・最適化するネットワーク技術。回線の冗長化やクラウド接続の最適化を含む。本ラボはHeadscale（WireGuardメッシュの制御プレーン）で拠点管理の基盤動作を再現。
ダークサービス
公開ポートを一切持たず、認証済みのオーバーレイ経由でのみ到達可能なサービス。外部から見ると存在自体が認識できない。Veronaの「ダイナミックポートコントロール」の考え方の核心。
ローカルブレイクアウト
特定のクラウドサービスへの通信を本社経由にせず、拠点から直接インターネットへ接続させる方式。回線負荷の軽減や速度向上に寄与する（本ラボでは対象外）。
CASB（Cloud Access Security Broker）
クラウドサービス（アプリケーション）の利用状況を可視化・制御する仕組み。本ラボでは対応するOSSの実装コストが高く、完全に対象外とした。
LogQL
Grafana Lokiのログクエリ言語。count_over_time（時間窓内の件数集計）やunless（集合差演算）などを使い、3観点検知のルールを表現する。
VRL（Vector Remap Language）
Vectorが提供するログ変換用のミニ言語。生ログから構造化フィールドを抽出し、人間可読なメッセージを付与する変換（本ラボでの「翻訳変換」の再現）に使用。
S3互換ストレージ
Amazon S3と同じAPI仕様で操作できるオブジェクトストレージ。本ラボではMinIOをセルフホストし、Lokiのログchunkのオフロード先として使用。
WireGuard
高速・軽量な暗号化VPNプロトコル。Headscaleはこのプロトコルのメッシュネットワークを制御するオープンソースのコーディネータ。
preauthキー
WireGuardメッシュ（Headscale/Tailscale）に端末を事前認証で参加させるための使い捨てトークン。拠点の初期登録に使う。
オーバーレイネットワーク
物理ネットワークの上に論理的に構築される仮想的な通信経路。OpenZitiやWireGuardメッシュは、認証済みの相手同士だけがオーバーレイ経由で通信できる。
DNSシンクホール
危険な/禁止されたドメインへのDNS問い合わせに対し、実在しないアドレス（0.0.0.0など）を返して通信を無効化する手法。本ラボではBlockyのzeroIp設定で再現。
ルールベース近似
AIの学習・適応能力の代わりに、固定の閾値・時間窓を使ったクエリ（LogQL等）で異常検知の「考え方」だけを再現する手法。本ラボの3観点検知はこれにあたる。
