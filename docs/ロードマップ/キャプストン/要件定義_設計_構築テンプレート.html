0. 題材の前提（自由に調整可） #

本キャプストンは 2段構え：まず STEP1=基本形 で全機能を統合し、次に STEP2=最終形 で大規模化する。

  • STEP1 基本形（まずここを完成させる）
    • 想定企業: 社員 約120名（本社100／支社20）の中堅企業
    • 拠点: 本社(HQ) 2フロア、支社(BR) 1フロア
    • 接続: 本社はデュアルISP（冗長）、本社⇔支社は IPsec VPN（インターネットVPN）
    • 方針: 「止まらない・守られる・運用できる」社内LAN
  • STEP2 最終形（大規模社内LAN）（STEP1の後に拡張）
    • 本社を 3階層（コア/ディストリ/アクセス）のキャンパスLAN へ拡張（複数棟・多数フロア）
    • 規模拡大への対応: マルチエリアOSPF・経路集約・MST・階層的IP設計
    • ※ DC（VXLAN/EVPNファブリック）は対象外。詳細は末尾「付録: 大規模社内LANへのスケール設計」
    • 仮想環境では縮小モデルで原理確認 → 設計書で大規模を表現の二段構えで現実的に進める

付録: 大規模社内LANへのスケール設計（STEP2 最終形） #

STEP1（2拠点）が完成したら、本社を3階層キャンパスLANへ拡張し「大規模でも破綻しない設計」を学ぶ。

構成イメージ #

              [ Core SW x2 ]                 ← L3中枢（ルーティング専念・全棟を束ねる）
             /      |       \
     [Distri-A]  [Distri-B]  [Distri-C]      ← 棟/フロア集約（L3境界・経路集約・STP(MST) root）
        /|\         ...          ...
  [ Access SW 群（各フロアでPC/AP/IP電話収容）]  ← VLAN・ポートセキュリティ

設計記入欄（STEP1からの差分） #

論点 手法 記入
階層分割 Core=ルーティングのみ / Distri=L2-L3境界 / Access=端末収容 □
ルーティング OSPF マルチエリア（Area0=コア、棟ごとにArea） □ エリア割当
経路集約 ABR（ディストリ）で棟内サブネットを集約 □ 集約アドレス
IPアドレス 棟→フロア→VLAN で集約しやすい階層設計 □ 体系
STP MST（VLAN群→インスタンス）、Root=ディストリ □ インスタンス設計
冗長 各階層二重化・上下リンク冗長（EtherChannel） □
拠点横展開 支社テンプレを標準化して複製 □ 標準仕様

仮想環境での進め方 #

  • ノードは原理確認に必要な最小数（例: コア2＋ディストリ2＋アクセス2＋PC）に絞る。
  • 「縮小モデルで挙動確認（マルチエリア境界の集約・MSTのRoot）→ 設計書では本来規模を表現」。
