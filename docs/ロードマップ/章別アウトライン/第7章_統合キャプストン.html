第7章: 統合キャプストン — 2拠点 →（最終形）大規模社内LAN（⬜ ゴール）

これまでの全章を1つに束ねる総合演習。「企業の社内ネットワークを、 要件定義から設計・構築・試験・ドキュメント化まで自力で一気通貫」 がゴール。 2段構えで進める：まず 基本形＝本社＋支社2拠点 で全機能を統合し、 次に 最終形＝大規模社内LAN（3階層キャンパス） へスケールアップする。 （DC＝VXLAN/EVPNファブリックは目標外。第6章の任意枠で“仕組みを知る”に留める） 詳細な進め方・記入テンプレは ../キャプストン/要件定義_設計_構築テンプレート.md。

         [ISP-A]                                   [ISP-B]
            │   eBGP(デュアルホーム)                  │
       ┌──[ R-Edge1 ]══iBGP/OSPF══[ R-Edge2 ]──┐     （第1章: BGP/BFD）
       │        │  FW/DMZ・NAT・ACL  │            │     （第4章: ACL/NAT）
       │   ┌─[ CoreSW1 ]══EtherChannel══[ CoreSW2 ]─┐

       │   │   VLAN10 営業 / VLAN20 開発 / VLAN30 サーバ
       │   └─ DHCP/DNS/NTP（第3章）, SNMP/Syslog（第5章）
       │
       │═══════ 拠点間 IPsec VPN（第4章: rfc4301/4303）═══════
       │
   支社BR [ R-Branch ]─[ L3SW ]─ VLAN40 支社LAN（第2章/第3章）

  • 支社(BR): L3SW + ルータ、支社LAN、本社へIPsec VPN接続
  • 自動化（第5章 Ansible）で設定投入・取得を効率化、（任意）IPv6デュアルスタック（第6章）

🏗️ 最終形：大規模社内LANへのスケール #

基本形（2拠点）で全機能を一度通したら、**「設計が大規模でも破綻しない」**ことを学ぶために 本社を 3階層（コア／ディストリビューション／アクセス）のキャンパスLAN へ拡張する。

            [ Core SW x2 ]              ← 高速・冗長の中枢（L3、ルーティングのみ）
           /     |      \
   [Distri-A] [Distri-B] [Distri-C]     ← 棟/フロア単位の集約（L3境界・経路集約・STP root）
      / | \      ...        ...
 [Access SW 群（各フロア）]              ← 末端ポート収容（VLAN・ポートセキュリティ）
スケール時の論点 大規模で効く設計手法
経路数の爆発 OSPF マルチエリア化＋経路集約（エリア境界で束ねる）
STPの複雑化 MST（VLAN群をインスタンスに束ねる）、ディストリでL2/L3境界を切る
アドレス枯渇/管理 階層的IP設計（棟→フロア→VLANで集約しやすく）
単一障害点 各階層を冗長化（コア/ディストリ二重化、上下リンク冗長）
拠点増 支社テンプレを横展開（同じ設計を複製できる標準化）

仮想環境（arm64/IOL）はノードが重いので、「縮小モデルで原理を確認 → 設計書では大規模を表現」 という二段構えが現実的。ノードは原理確認に必要な最小数（例: コア2＋ディストリ2＋アクセス2）に絞る。
