第5章: 監視・運用・自動化（⬜）

「作って終わり」ではなく「運用できる」状態へ。監視（見える化）と自動化（人手削減）。 この章は既存の clab 資産（ansible-inventory.yml / nornir-simple-inventory.yml）が活きる。

🎯 章の狙い #

  • SNMP/Syslog/NetFlow で機器とトラフィックを「見える化」できる
  • NETCONF/RESTCONF + YANG で、構造化データによる設定投入/取得ができる
  • Ansible/nornir で複数機器へ設定を一括・冪等に流せる

前提 #

  • 第1〜4章（監視/自動化の対象となる構成が一通り組めること）
  • 機器へのSSH有効化（crypto key generate rsa 等。作業ログ §3 のSSH未設定メモ参照）

🔑 つまずきポイント先回り #

  • SNMP: v2c(community) と v3(認証/暗号) の違い、OID/MIBの読み方
  • NETCONF: candidate/running/startup データストア、commit/lock
  • Ansible: 冪等性、ios_config と純正モジュール、インベントリ（既存ファイルを流用）

🔗 既存資産の活用 #

  • 各 clab-*/ansible-inventory.yml・nornir-simple-inventory.yml をそのまま教材に。
  • まず「全機器の show version を一括取得」から始めると自動化の効果を体感できる。
