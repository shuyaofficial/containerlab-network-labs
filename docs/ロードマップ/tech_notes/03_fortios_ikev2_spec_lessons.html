2. 教訓と次回以降のアクション（FortiOS環境） #

FortiOSでは「使用されないVPNトンネル（＝ポリシーがないトンネル）」はネゴシエーション自体が意図的に破棄される仕様がある。 そのため、ネットワークエンジニアの基本である「レイヤーの低いところから順番に設定して疎通確認していく」というセオリー通りに進めると、設定が正しいのにVPNが上がらないというドツボにハマってしまう。

【次回のアクション】 FortiGateが関わるVPN環境を構築する際は、以下の順番を必ず守る（設計書通りに一気に流し込む）。

  1. VPNインターフェース（Phase 1 / Phase 2）の作成
  2. スタティックルートの作成
  3. ファイアウォールポリシー（VPNトンネルの入出力許可）の作成 ← 重要（テスト用でも良いので all to all を入れる）
  4. 最後にVPNのステータス確認およびPingテストを実施する

「VPNが上がってからポリシーを入れる」のではなく、「ポリシーを入れないとVPNが上がらない」という依存関係を意識した構成管理を行うこと。
