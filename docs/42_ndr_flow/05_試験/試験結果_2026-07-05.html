NW-ZT トラック N3。NDR の本質「同一サブネットの端末同士（east-west）の通信を、経路に割り込まず観測し、横方向の偵察・スキャンを検知してフローとして可視化・集約する」を実機で実証。

  • east-west の定義: attacker と victim は同一サブネット 172.40.0.0/24（L2 隣接）。両者間の通信は docker bridge ndr-br0 を横方向に流れる。Suricata はその bridge を promiscuous 監視し、north-south（境界越え）ではなく east-west を捉える。

総括 #

NDR の核心を OSS（Suricata + Loki/Grafana, arm64）で east-west にフォーカスして実証完了。 docker bridge を L2 セグメントに見立て、Suricata を host mode で bridge に張り付けることで、SPAN 配線なしに east-west フレームを直接監視できた。SYN スキャン（横方向の偵察）が DPI（sid:1000001 alert）とフロー（5-tuple flow 300 件超）の両面で捉えられ、それが Loki/Grafana に集約されて LogQL で絞り込める。これは Darktrace（DPI 振る舞い）と Cisco Secure Network Analytics/Stealthwatch（NetFlow ベース挙動分析）の 2 系統を 1 センサで追体験する構成。

north-south の IDS（境界の関所で外向きを見る）と違い、N3 は内部端末同士の横方向を見る点が本質。「侵害は起きる前提、起きた後に横移動に気づく」という NDR の思想を実機で確認した。NetFlow の完全パイプライン（softflowd→goflow2）は arm64 softflowd の制限で Suricata flow に代替したが、フロー統計としての情報（誰が誰とどれだけ）は等価に取得できている。詳細な詰まりどころは 構築ログ。
