要件定義書: QoS/ACL/route-map応用

1. 背景と目的 #

社内ネットワークの基本ルーティング基盤（OSPF/BGP）が構築された後、実際の業務環境では「通信の品質担保」と「セキュリティの確保」が求められる。 本ラボでは、音声通話（VoIP）やビデオ会議など遅延にシビアなトラフィックの通信品質を保証するQoS（Quality of Service）と、不要な通信や外部からの攻撃を防ぐためのアクセス制御（ACL）、およびトラフィックを柔軟にコントロールするポリシーベースルーティング（PBR）を実装する。

2. セキュリティ（ACL）要件 #

2.1 ゾーンベースの通信制御 #

  • Trustゾーン（社内）:
    • インターネット（Untrust）やDMZへのアクセスは原則として許可する。
    • DMZのサーバに対しては、HTTP(80)/HTTPS(443)のみを許可し、不要なポート（SSH等）へのアクセスは禁止する。
  • DMZゾーン:
    • インターネットからのアクセスは、公開サービスであるHTTP(80)およびHTTPS(443)のみを許可する。
    • DMZからTrustゾーン（社内）への通信開始は全て禁止する（万が一DMZが乗っ取られた場合の被害拡大防止）。
  • Untrustゾーン（インターネット）:
    • 社内（Trust）からの戻り通信（established）のみ許可する（※ルータのステートフルファイアウォール機能またはACLのestablished句で代用）。

3. 通信品質（QoS）要件 #

3.1 トラフィックの識別と優先度（マーキング） #

  • 音声トラフィック（VoIP）は遅延とジッタに極めて敏感であるため、最高優先度とする。
    • hq-voip端末（10.27.50.0/24）からの通信は、DSCP値 EF (Expedited Forwarding) でマーキングする。
  • 業務データトラフィックは中優先度とする。
    • 社内PCからの特定の業務アプリケーショントラフィックを AF21 等でマーキングする。
  • その他のトラフィックは Best Effort (BE) とする。

3.2 帯域制御とキューイング #

  • WAN回線（hq-edgeのインターネット出口など）において帯域幅に上限（例: 10Mbps）を設ける。
  • 混雑時に音声トラフィックが遅延しないよう、LLQ（Low Latency Queuing）を用いて音声トラフィック用に一定の帯域を絶対保証する。

4. 経路制御（Policy-Based Routing / BGP制御）要件 #

  • PBR (Policy-Based Routing):
    • 通常の宛先ベースのルーティングではなく、特定の「送信元IP」や「プロトコル」に基づいた経路選択を行う。例えば、音声トラフィックだけを遅延の少ない専用回線（別ネクストホップ）に回すなど。
  • BGPルートフィルタリング:
    • インターネット（ISP）からのフルルート受信を防ぎ、必要な経路（デフォルトルート等）のみを受信・インストールする。
    • 自社のプレフィックスのみをISPへ広報し、意図しない他拠点の経路をインターネットに漏らさない。
