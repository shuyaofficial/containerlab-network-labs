🚀 フェーズ2：構築ミッション #

以下のミッションを、あなたがこれまで培った知識で、自分の好きな順番で 構築してください。 （わからなくなった時は、過去のフォルダのREADMEやコマンド履歴を見返して構いません！）

ミッション①：本社の冗長化LAN構築（L2/L3・OSPF・HSRP） #

  1. HQ-Core1 と HQ-Core2 の間に VLAN 10 を作成し、PCを接続する。
  2. コアスイッチ間のリンクを EtherChannel（トランク）で冗長化する。
  3. HQ-Core1 を STP のルートブリッジ、かつ HSRP の Active（仮想IP: 10.1.10.254）に設定する。
  4. HQ-Edge1, HQ-Edge2, HQ-Core1, HQ-Core2 の間で OSPF (Area 0) を構成し、社内のルーティングを確立する。

ミッション②：インターネット境界とBGPデュアルホーム #

  1. HQ-Edge1 と HQ-Edge2 でインターネットに出るための NAT (PAT) を設定する。（内部の10.x.x.xを、外部インターフェースのIPに変換）
  2. ISPとの間で BGP を確立する。（ISPのAS番号を 65000、本社のAS番号を 65001 とする）
  3. BGPの Local Preference 等を操作し、通常時は HQ-Edge1（メイン回線）を通ってインターネットに出るようにする。

ミッション③：拠点間 IPsec VPN #

  1. 本社（HQ-Edge1）と 支社（BR-Edge）の間にインターネット越し（200.x.x.x）の IPsec VPN を構築する。
  2. 対象となる通信は「支社(10.2.40.0/24) ⇔ 本社営業部(10.1.10.0/24)」のみとする。

🎯 フェーズ3：最終試験（テスト要件） #

すべての構築が終わったら、以下のテストを実施し、すべてクリアできれば**合格（卒業）**です！

✅ テスト1：インターネット疎通 HQ-PC-Sales および BR-PC から、ISP（200.0.1.254 など）に Ping が届くこと。（NATが成功している証拠）

✅ テスト2：拠点間通信の暗号化 BR-PC から HQ-PC-Sales (10.1.10.100) に Ping が届くこと。かつ、その通信がインターネット上（ISPのルーター）では暗号化されていて中身が見えないこと。

✅ テスト3：究極の耐障害性テスト HQ-PC-Sales から ISP へ Ping を打ち続けている状態で、HQ-Edge1 のコンソールから e0/1 を shutdown する。 数秒の断のあと、自動的にサブ回線（HQ-Edge2）へ経路が切り替わり、Pingが復活すること。
