パラメータシート（セキュリティ・NAT編）

本ドキュメントは、プライベートIPアドレスをインターネットへ変換するための「NAT設定」と、インターネット越しに本社と支社を安全に繋ぐ「IPsec VPN設定」のパラメータを定義します。

1. NAT 設計 (Network Address Translation) #

社内のプライベートIPアドレス（10.0.0.0/8）を、各エッジルーターが持つパブリックIPアドレスに変換し、インターネット通信を可能にします。 方式としては、1つのパブリックIPで複数PCの通信を賄う NAPT（PAT / IP Masquerade） を採用します。

NAT/PAT パラメータ #

機器名 内部側 (ip nat inside) 外部側 (ip nat outside) 変換対象の送信元IP (ACL) 変換後のパブリックIP
HQ-Edge1 e0/2, e0/3 e0/1 10.0.0.0 0.255.255.255 e0/1 のIP (200.0.1.1) にオーバーロード
HQ-Edge2 e0/2, e0/3 e0/1 10.0.0.0 0.255.255.255 e0/1 のIP (200.0.2.1) にオーバーロード
BR-Edge e0/2 e0/1 10.0.0.0 0.255.255.255 e0/1 のIP (200.0.3.1) にオーバーロード

※ 注意: IPsec VPN トンネルを通る通信（本社〜支社間）は、NATの変換対象から除外（NAT Exempt）する必要があります。

2. IPsec VPN 設計 (拠点間暗号化通信) #

インターネット（WAN）を経由して、本社の HQ-Edge1 と支社の BR-Edge 間で暗号化トンネル（VPN）を構築します。 Cisco機器で一般的な GRE over IPsec または VTI (Virtual Tunnel Interface) を想定し、IKEv2 を用いて鍵交換を行います。

IKE Phase 1 (ISAKMP / IKEv2 Proposal) パラメータ #

トンネルを張るための「事前の安全な通信路」を作るフェーズです。

項目 設定値 備考
暗号化アルゴリズム AES-256 高度なセキュリティ
ハッシュアルゴリズム SHA-256 データの改ざん検知
DHグループ Group 14 鍵交換の強度
認証方式 Pre-Shared Key (事前共有鍵) パスワード認証
事前共有鍵 (PSK) CiscoSecretKey123! エッジルーター間で一致させる

IKE Phase 2 (IPsec Transform Set) パラメータ #

実際のデータを暗号化してカプセル化するフェーズです。

項目 設定値 備考
プロトコル ESP (Encapsulating Security Payload) 暗号化＋認証
暗号化アルゴリズム AES-256
ハッシュアルゴリズム SHA-256
IPsec モード Tunnel モード IPヘッダごと暗号化してカプセル化

トンネルインターフェース (Tunnel 0) パラメータ #

仮想的なトンネルインターフェースを作成し、OSPFなどのルーティングを通せるようにします。

機器名 トンネルIPアドレス トンネル送信元 (Source) トンネル宛先 (Destination)
HQ-Edge1 192.168.100.1 /30 e0/1 (200.0.1.1) 200.0.3.1 (BR-Edge)
BR-Edge 192.168.100.2 /30 e0/1 (200.0.3.1) 200.0.1.1 (HQ-Edge1)

※この Tunnel 0 に対して OSPF（プロセス1、エリア0）を有効化し、支社のLAN（10.2.40.0/24）と本社LAN間で動的に経路を交換します。
