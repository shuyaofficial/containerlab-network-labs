3. ルーティング設計 #

  • 内部ルーティング (IGP): OSPF を採用。本社内のエッジルーターとコアスイッチ間を Area 0 で接続し、動的に経路情報を同期する。
  • 外部ルーティング (EGP): BGP を採用。本社とISP間でピアを確立し、デュアルホーム環境（2回線）でのインターネット通信を制御する。
  • 拠点間ルーティング: 支社と本社の間はインターネット越しの接続となるため、IPsecトンネルインタフェースを構築し、スタティックルートで経路を制御する。

4. 冗長化設計（高可用性） #

単一障害点（SPOF）を排除するため、各レイヤーで以下の冗長化技術を組み合わせる。

  • L2レイヤー (ループ防止・リンク冗長): コアスイッチ間を EtherChannel（複数物理リンクの束ね）で接続し、帯域増強と耐障害性を確保する。また STP (RSTP) によりL2ループを未然に防ぐ。
  • L3レイヤー (デフォルトゲートウェイ冗長): HSRP を導入。HQ-Core1をActive、HQ-Core2をStandbyとし、端末から見たゲートウェイIP（10.1.x.254）を仮想化する。
  • WANレイヤー (上流回線連動): Track 機能を活用し、メイン回線（HQ-Edge1）のダウンを検知した場合、自動的にHSRPのActive権をHQ-Core2に譲り、サブ回線（HQ-Edge2）へトラフィックを迂回させる。

5. セキュリティ設計 #

  • インターネットアクセス: 本社および支社のエッジルーターにて NAT (PAT) を設定し、プライベートIPを隠蔽する。
  • 拠点間通信: IPsec VPN を用いて、支社〜本社間の通信ペイロードを暗号化（ESP）する。
  • アクセス制御: 将来的に、支社から本社開発部への通信を遮断する ACL (Access Control List) を導入する準備をしておく。
