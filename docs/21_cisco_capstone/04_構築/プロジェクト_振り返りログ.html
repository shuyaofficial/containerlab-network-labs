構築・トラブルシューティング履歴 (2026-06-08)

本ドキュメントは、Capstone プロジェクトにおける構築中の躓き（トラブル）と、その根本原因、解決策の履歴をまとめたものです。今後の運用や同様の環境構築時のナレッジベースとして活用します。

1. L2スイッチの初期IP割当問題 #

  • 事象: hq-core1 および hq-core2 が起動直後、管理用ネットワーク（clab-mgmt）のIPアドレスを持っておらず、SSHでのログインができない状態となった。
  • 原因: 対象機器がL2スイッチイメージ (cisco_iol:L2-15.2) であったため、管理用インターフェース (Ethernet0/0) がデフォルトで「スイッチポートモード (L2モード)」として認識され、IPがアサインされなかった。
  • 解決策: コンソール（expectスクリプト）経由でログインし、Ethernet0/0 に対して no switchport を発行。その後 vrf clab-mgmt の再紐付けとIPアドレス再設定を行うことでSSH疎通が可能になった。

2. Cisco IOL メモリ不足 (MALLOCFAIL) 問題 #

  • 事象: NATやIPsecの設定を投入していくと、ルーター（特に hq-edge1, br-edge）上で %SYS-2-MALLOCFAIL や %FIB_NAT-2-NOMEM などのエラーが発生し、コマンドが受け付けられなくなったり、機器がクラッシュする事象が発生した。
  • 原因: Containerlab上のCisco IOLのデフォルトメモリが小さく（約76MB）、NAT変換テーブルや暗号化の機能に必要なメモリ領域が枯渇した。
  • 解決策: capstone.clab.yml ファイル内で各ルーターノードの環境変数 IOL_MEMORY: "1024" を明示的に指定して再デプロイし、十分なメモリ（1GB）を確保することで根本解決した。

3. IPsec VPN (VTI) 設定漏れ・タイポ #

  • 事象: IPsecトンネルのステータスが上がらず、OSPFのネイバー関係も構築されない。
  • 原因: 以下の複数の設定不備・入力ミスが重なっていた。
    1. HQ-Edge1: pre-shared-key にタイポがあった（小文字の k が混入: CiscoSecretkey123!）。
    2. BR-Edge: peer の設定内で address 200.0.1.1 の指定が漏れており、IKEv2プロファイルが対向IPと結びつかなかった。
    3. BR-Edge: NATの設定（Standard ACL）が緩すぎたため、IPsecトンネルを通るべき通信（本社宛）までISP側にNATされてしまっていた。
  • 解決策:
    • NATはExtended ACL (access-list 100) を用い、「本社宛以外の通信だけNATする (NAT Exempt)」よう修正。
    • 両端の keyring 設定を正しく修正し、Pingを打ってトンネルを起動した。

4. 支社LANの経路広報漏れ #

  • 事象: トンネル（Tunnel0）は開通したものの、支社PC（10.2.40.100）から本社PC（10.1.10.100）へのPingが届かない。
  • 原因: BR-Edge において、OSPFプロセス (router ospf 1) 内で、支社側ローカルLAN (10.2.40.0/24) を network コマンドでアドバタイズ（広報）し忘れていたため、本社側が支社LANへの戻り経路を知らなかった。
  • 解決策: BR-Edge にて network 10.2.40.0 0.0.0.255 area 0 を追加し、通信が開通した。

5. BGP・OSPF連携によるフェイルオーバーテスト（ECMP） #

  • 事象: HQ-Edge1 の e0/1 をシャットダウンしてフェイルオーバーの動作を確認しようとしたが、Pingが全く途切れることなく（パケットロス0%）成功し続けた。
  • 原因: HQ-Edge1 と HQ-Edge2 の両方が、デフォルト設定のまま OSPF に対して default-information originate を発行していたため、Coreスイッチ側からは「両方とも等コスト（ECMP）でインターネットに出られる」と見なされていた。そのためトラフィックが負荷分散されており、たまたまHQ-Edge2を通っていたPingは断の影響を受けなかった。
  • 解決策: HQ-Edge1 を完全なメイン回線（Active）として扱うため、HQ-Edge2 の設定に metric 20 を付与し、平時はすべてのトラフィックがHQ-Edge1を経由するように修正した。これにより意図した「Active/Standby」構成となった。
