⚠ 旧世代IOLのライセンス問題（重要） #

modern vrnetlab(v0.21.0)のIOLビルダーは iourcライセンス生成を持っていない（iol.binを直接起動するだけ）。 旧環境の15.x世代IOLは iourc必須で、消えた旧VMでは自作entrypoint（キー自動生成 + expectで初期ダイアログ自動応答）で解決していた。 → 同世代の.binを使う場合、このカスタムentrypointの復元が必要。.binを1つ用意してくれれば、実バイナリのhostid出力に合わせて復元・検証する（盲目で当てずに、実物で確実に通す）。

代替案（より入手容易・堅牢）: iourc不要の**新しめIOL（例: 17.12.x のL2/L3）**を入手 → modern定義のままビルド可。 その場合は campus.clab.yml の image: タグを入手版に合わせて差し替える（タグは単なるラベルなので問題なし）。

解決した問題 #

問題 原因 解決策
IOLルータ(15.7.3M2) ARM64でSystem Config Dialogハング IOLルータバイナリがARM64 QEMU上でダイアログ表示時にCPU 96%で無限ループ L2-advipservices-2017イメージに切替（L3ルーティング完備・ダイアログ非表示）
docker attachでコンソール接続不可（全IOLノード） entrypoint.shで&バックグラウンド起動→IOLのfd 0が/dev/nullに execでIOLを直接起動（PID 1=iol.bin、fd 0=/dev/pts/0）
FortiGate QEMU crash ARM64でcpu=host不可 env: QEMU_CPU: qemu64 をclab.ymlに設定
