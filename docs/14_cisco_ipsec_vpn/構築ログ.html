テーマ14: IPsec サイト間VPN 構築ログと教訓

本ドキュメントは、IPsec VPN（サイト間トンネル）構築時のトラブルシューティングノウハウ、および実務上の教訓を記録する。

1. IKE フェーズ1（ISAKMP）のパラメータ完全一致の原則 #

  • 概念: ルーター同士が暗号化のネゴシエーション（通信）を行う際、暗号化アルゴリズム（AES等）、ハッシュ関数（SHA等）、認証方式（Pre-Share）、DHグループの4つの設定項目が両端で完全に一致していなければならない。
  • 教訓: 実務で「VPNが繋がらない」場合、真っ先に疑うべきはフェーズ1のパラメータ不一致である。show crypto isakmp sa で状態が ACTIVE にならない場合、設定の打ち間違いがないか確認する。

2. Crypto Map（クリプトマップ）における「鏡合わせ」の法則 #

  • 概念: Crypto Mapを適用する際、アクセスリスト（ACL）と対向先IP（Peer）は、両拠点から見てお互いを指し示す「鏡合わせ」にする必要がある。
  • 具体例:
    • 東京側ACL: permit ip 192.168.10.0 ... 192.168.20.0 ... / Peer: 200.2.2.1
    • 大阪側ACL: permit ip 192.168.20.0 ... 192.168.10.0 ... / Peer: 200.1.1.1
  • 教訓: 実務での設定コピーミス（コピペして宛先を直し忘れる）が多発するポイントである。

3. VPNとルーティング（ip default-gateway と ip route の違い） #

  • 概念: ルーター（ip routing稼働中）はパケットを中継・処理するために自身のルーティングテーブルを使用する。そのためインターネット側（ISP）へVPNトラフィックを届けるには ip route 0.0.0.0 0.0.0.0 の設定が必須である。ホスト端末用の ip default-gateway は効かない。
  • 教訓: 「機器がL2ホストとして動いているか、L3ルーターとして動いているか」を常に意識し、適切なゲートウェイ/ルーティング設定を使い分けることが不可欠。
