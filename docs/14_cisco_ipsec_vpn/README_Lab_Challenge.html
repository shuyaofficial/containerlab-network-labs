🏗️ 設定パラメータ要件 #

要件1：IPアドレスとルーティングの基礎 #

各機器に上記のIPアドレスを設定し、以下のルーティング要件を満たしてください。

  • PC類: L2ホストとして動作させ（no ip routing）、適切なデフォルトゲートウェイを設定すること。
  • ルーター類 (Tokyo-R1 / Osaka-R1): 外部（ISP: 200.x.x.254）へ向かうデフォルトルート（ip route）を設定すること。

✅ 【検証1】 Tokyo-R1 から Osaka-R1（ping 200.2.2.1）へPingが飛ぶこと。

要件2：IKEフェーズ1（ISAKMP ポリシー）パラメータ #

東京と大阪で、全く同じ以下の鍵交換ポリシー（ルール番号 10）を作成してください。

項目 指定パラメータ
暗号化アルゴリズム (Encryption) AES
ハッシュ関数 (Hash) SHA
認証方式 (Authentication) Pre-Share (事前共有キー)
DHグループ (Group) 14
事前共有キー (Key) cisco123

※注意：事前共有キーのピアアドレス（対向先IP）は、お互いのルーターのインターネット側IPを指定すること。

要件3：IKEフェーズ2（IPsec トランスフォームセット）パラメータ #

実際にデータをカプセル化する際の「変換ルール」を作成してください。

項目 指定パラメータ
セット名 MYSET (任意だが両端で合わせるのが推奨)
カプセル化と暗号化 esp-aes
認証・改ざん防止 esp-sha-hmac

要件4：暗号化対象トラフィックの定義（ACL） #

「どの通信をVPNトンネルに通すか」を定義するアクセスリスト（番号 100）を作成してください。

  • Tokyo-R1側: 192.168.10.0/24 から 192.168.20.0/24 へ向かうIP通信を許可
  • Osaka-R1側: 192.168.20.0/24 から 192.168.10.0/24 へ向かうIP通信を許可（※東京の鏡合わせになります）

要件5：Crypto Map（クリプトマップ）の作成と適用 #

上記のフェーズ1、フェーズ2、ACLをすべて束ねる指示書を作成し、外側のインターフェースに適用してください。

項目 指定パラメータ
マップ名 / 番号 MYMAP / 10
プロトコル ipsec-isakmp
対象トラフィック (Match) 作成したアクセスリスト 100
変換ルール (Transform-set) 作成した MYSET
対向先IP (Peer) 相手拠点のインターネット側IP
適用先インターフェース Ethernet 0/1 (インターネット側)

🏁 最終検証（トンネルの開通） #

すべて設定できたら、ついにトンネルに通信を通します！

✅ 【検証2】

  1. Tokyo-PC から Osaka-PC（ping 192.168.20.10）へPingを打つ。（最初はトンネル生成のため1〜2回タイムアウトしますが、通るはずです）
  2. show crypto isakmp sa で状態が ACTIVE（または QM_IDLE）になっていることを確認。
  3. show crypto ipsec sa で #pkts encaps と #pkts encrypt のカウンターが増加し、パケットが暗号化されていることを確認。
