📖 Cisco IOL BGP基礎 手動構築ガイド

本ファイルは、メンターである私（AI）との対話を通じて、完全に真っ白なCisco IOLルータ2台を手動でBGP（eBGP）接続していくステップバイステップの進行記録ガイドです。

🗺️ トポロジーと設計要件 #

ルータ2台を直結したシンプルなeBGP接続ネットワークを設計します。

           [ AS 65001 ]                     [ AS 65002 ]
         +--------------+                 +--------------+
         |      R1      |                 |      R2      |
         |  Loopback0:  |                 |  Loopback0:  |
         |   1.1.1.1    |                 |   2.2.2.2    |
         +------+-------+                 +------+-------+
                | Ethernet0/1                    | Ethernet0/1
                | .1                             | .2
                +--------------------------------+
                        192.168.12.0/24

インターフェース＆BGP設計 #

デバイス インターフェース IPアドレス / サブネット BGP AS番号
R1 Ethernet0/1 192.168.12.1/24 AS 65001
R1 Loopback0 1.1.1.1/32
R2 Ethernet0/1 192.168.12.2/24 AS 65002
R2 Loopback0 2.2.2.2/32

🏁 STEP 1: 物理トポロジーの起動とログイン #

1. 機器の CLI へのログイン方法 #

Cisco IOLイメージには初期IPやSSH設定が入っていないため、**コンソール接続（docker attach）**でログインします。

  • R1 に接続:
    sudo docker attach clab-cisco-bgp-basics-r1
  • R2 に接続:
    sudo docker attach clab-cisco-bgp-basics-r2

[!CAUTION] コンソールからの切断方法: アタッチしたコンソールから抜ける（ホストに戻る）ときは、Ctrl + P の後に Ctrl + Q を順番に押してください。

🛠️ ミッション一覧 #

💻 Mission 1: IPアドレス設定と生存確認 (STEP 1) #

R1、R2にお互いIPアドレスを設定し、物理的な隣人同士でPingが通るようにします。

  • R1設定目標:
    • Loopback0: 1.1.1.1 255.255.255.255
    • Ethernet0/1: 192.168.12.1 255.255.255.0 (インターフェースの有効化 no shutdown を忘れずに！)
  • R2設定目標:
    • Loopback0: 2.2.2.2 255.255.255.255
    • Ethernet0/1: 192.168.12.2 255.255.255.0 (有効化も忘れずに！)
  • 検証:
    • R1から ping 192.168.12.2 が成功することを確認。

💻 Mission 2: BGPネイバーの構築とTCP 179の観測 (STEP 2) #

異なるASのルータ同士でBGPネイバー（ピア）を確立します。 BGPはOSPFのように自動で相手を探さないため、「相手のIPアドレス」と「相手のAS番号」を明示的に指定する必要があります。

  • BGPの基本設定コマンド（R1の例）:
    router bgp 65001
 neighbor 192.168.12.2 remote-as 65002
  • 検証コマンド:
    • BGP全体の要約を確認： show ip bgp summary
    • ネイバーの状態が Active（相手を探している）や OpenSent などの遷移を経て、最終的に 数字（受信した経路数。現状は0） または Established になることを確認します。

💻 Mission 3: OSPFと決定的に異なる「network」広告の検証 (STEP 3) #

BGPの network コマンドは、OSPFの「この範囲のインターフェースでプロトコルを起動する」という意味とは全く異なり、**「このIPアドレスのルートを、他のルータに広告（注入）する」**という意味を持ちます。

さらに、BGPには**「自分のルーティングテーブル（show ip route）に載っていない経路は、絶対に広告してはならない」**という厳しいルールがあります。

1. 破壊実験（無効な広告）

  • R1で、まだ自分が持っていない架空のIP 9.9.9.9 を広報するように設定します。
    router bgp 65001
 network 9.9.9.9 mask 255.255.255.255
  • 対向のR2で show ip bgp や show ip route を実行し、何も届いていないことを観測します。

2. 正しい広告

  • R1で、すでにルーティングテーブル（直接接続 C）に載っている Loopback0（1.1.1.1）を広報するように設定します。
    router bgp 65001
 network 1.1.1.1 mask 255.255.255.255
  • 対向のR2で show ip bgp を実行し、1.1.1.1 が届いていることを確認します。
  • その際、経路情報の横に 65001 i （AS 65001 を経由してきたことを示すAS-Path）がくっついているのを観察します！
