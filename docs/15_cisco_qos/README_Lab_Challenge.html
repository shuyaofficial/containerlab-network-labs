QoS / DiffServ チャレンジ：パラメータシート形式

このドキュメントは、実務の「要件定義（パラメータシート）」のみを記載したチャレンジ用資料です。 今回はルーター（R1）に「Telnet通信（遠隔操作）を最優先で処理せよ！」というVIP設定を施します。

ユーザー/パス: admin / admin

  • PC1: telnet 172.20.20.2
  • R1: telnet 172.20.20.4
  • R2: telnet 172.20.20.3
  • PC2: telnet 172.20.20.5

インターフェース設計 #

【送信元拠点】

  • PC1 (e0/1): 192.168.10.10 /24 （GW: 192.168.10.254）
  • R1 (e0/2): 192.168.10.254 /24 （PC1側）
  • R1 (e0/1): 10.1.1.1 /24 （R2側）

【宛先拠点】

  • R2 (e0/1): 10.1.1.2 /24 （R1側）
  • R2 (e0/2): 192.168.20.254 /24 （PC2側）
  • PC2 (e0/1): 192.168.20.10 /24 （GW: 192.168.20.254）

🏗️ 設定パラメータ要件 #

要件1：IPアドレスとルーティングの基礎 #

各機器に上記のIPを設定し、R1とR2で互いのLAN（192.168.10.0 と 192.168.20.0）へ到達できるようにスタティックルート（ip route）を設定してください。（※PC1とPC2の no ip routing も忘れずに！）

✅ 【検証1】 PC1 から PC2（ping 192.168.20.10）へPingが通ることを確認。

要件2：QoSの「分類（Class-Map）」ルールの作成（R1にて） #

QoSの第一歩は「誰を特別扱いするか」を決めることです。 まず、Telnet通信を識別するためのアクセスリスト（番号 100）を作成し、それをクラスマップに紐づけます。

項目 指定パラメータ
ACL番号 100
ACL条件 送信元 any から 宛先 any への tcp 23番（telnet）
クラスマップ名 CLASS-TELNET
マッチ条件 作成したアクセスリスト 100

💡 【構造のヒント】

! ① ACLで荷物を特定する
R1(config)# access-list 100 permit tcp any any eq 23

! ② クラスマップ（特定の荷物のグループ）を作る
R1(config)# class-map CLASS-TELNET
R1(config-cmap)# match access-group 100

要件3：QoSの「アクション（Policy-Map）」の定義（R1にて） #

次に、「先ほど分類した荷物に対して、どういう優先処理を行うか」をポリシーマップで定義します。 今回はTelnetパケットに「これは最優先（EF）だ！」というVIPシール（DSCP）を貼り付け、優先レーン（Priority）を通すように指示します。

項目 指定パラメータ
ポリシーマップ名 POLICY-QOS
適用するクラス CLASS-TELNET (要件2で作ったもの)
シール貼り (Marking) DSCP値を ef (Expedited Forwarding) に設定
優先レーン (Queuing) priority コマンドで優先帯域を 512 kbps割り当て

💡 【構造のヒント】

R1(config)# policy-map POLICY-QOS
R1(config-pmap)# class CLASS-TELNET
R1(config-pmap-c)# set dscp ef
R1(config-pmap-c)# priority 512

要件4：インターフェースへの適用（Service-Policy）（R1にて） #

最後に、このポリシー（警備員マニュアル）をR1がパケットを「送り出す」ドアに適用します。

項目 指定パラメータ
適用先インターフェース Ethernet 0/1 (R2へ出ていく側)
適用方向 out (出ていくパケットに対して適用)

💡 【構造のヒント】

R1(config)# interface e0/1
R1(config-if)# service-policy out POLICY-QOS

🏁 最終検証（優先制御の確認） #

すべて設定できたら、実際にパケットを流してQoSが効いているか確認します。

✅ 【検証2】

  1. まず PC1 からただのPingを打ちます（ping 192.168.20.10）。
  2. 次に PC1 からTelnetを打ちます（telnet 192.168.20.10）。※パスワードが無いので弾かれますが、パケットはR1を通過します。
  3. R1 で show policy-map interface e0/1 を実行してください。
    • Class CLASS-TELNET の欄に pkts matched というカウンターが増えていれば、**「TelnetパケットがVIPレーンを通って優先処理された」**という大成功の証拠です！！
