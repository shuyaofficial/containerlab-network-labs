Theme 25: Multi-Campus 構築・学習振り返り

1. 作業概要 #

  • 実施内容: Theme 25 (Multi-Campus) のラボ環境構築、およびCisco IOL（ルーター）へのコンフィグ投入。
  • 学習テーマ:
    • 複数キャンパス間（OSPF 10 と OSPF 20）をeBGPで接続。
    • prefix-list と route-map を用いた経路集約（/16）と再配信制御。
    • tag を用いた経路のルーティングループ防止策。

2. 作業中の疑問点と解決内容 #

コンフィグ投入にあたり、BGPとOSPFの再配信（redistribute）や経路制御について生じた疑問と、その解決・理解内容をまとめます。

① route-map は ACL の in と同じようなもの？ #

  • 疑問: route-map は物理インターフェースに適用するACL（パケットフィルタ）のようなものか？
  • 解決:
    • 違う。ACLの in は「データプレーン（ユーザーのパケット）」を制御する。
    • route-map は「コントロールプレーン（経路情報）」を制御する。OSPFとBGPの間で経路をやり取りする連絡通路に立ち、経路を通すか捨てるか、スタンプ（タグ）を押すかを決める役割。

② tag によるループブロックの仕組みとは？ #

  • 疑問: tagを付けると、どのようにして自分の出したパケット（経路）が戻ってきた時にブロックされるのか？
  • 解決:
    • BGPからOSPFに経路を入れる際に、set tag 65060 のように「この経路はBranch(AS 65060)から来た」という目印を付ける。
    • その経路がOSPF内を一周して再びBGPへ出ようとした際、出口の route-map で match tag 65060 → deny と設定しておくことで、「元々Branchから来た経路をBranchに送り返す」という無限ループを未然に防ぐことができる。

③ エラー: set tag not supported #

  • 事象: redistribute ospf 20 route-map OSPF-TO-BGP を適用しようとした際、以下のエラーが発生した。 % "OSPF-TO-BGP" used as redistribute ospf into bgp route-map, set tag not supported
  • 原因と解決:
    • 原因: route-map OSPF-TO-BGP の設定内で、match tag 65050（タグを見て捨てる条件）と書くべきところを、誤って set tag 65050（タグを付ける処理）と書いてしまっていた。
    • 理由: BGPというプロトコルにはOSPFのような「タグ」を付ける機能（属性）が無いため、BGPへ向かう再配信で set tag を行おうとするとIOSがエラーを出す。
    • 解決: no set tag 65050 で削除し、match tag 65050 に修正することで正常に適用できた。

④ show route-map のカウントが増えない #

  • 事象: show route-map BGP-to-OSPF を確認したが、Policy routing matches: 0 packets, 0 bytes となっておりカウントされていなかった。
  • 原因と解決:
    • 原因: Cisco IOSの仕様。このカウントは「ポリシーベースルーティング（PBR）」としてユーザーのパケットを処理した回数を示すものであり、今回のような「経路の再配信」で処理した回数はここにはカウントされない。
    • 解決: 再配信が成功したかを確認するには、show ip ospf database external や show ip route を使い、実際にタグ付きの経路がルーティングテーブル等に入っているかを直接確認するのが正しいアプローチである。

3. 重要なコマンドまとめ #

  • 経路が正しくBGPで隣に送られているか確認
    show ip bgp neighbors 198.51.100.2 advertised-routes
  • OSPFデータベース内で外部経路（とタグ）を確認
    show ip ospf database external
  • 再配信時の重要オプション（BGP→OSPF）
    redistribute bgp 65050 subnets route-map BGP-to-OSPF
    ※ subnets をつけないとクラスフルネットワークしか再配信されないため必須。
