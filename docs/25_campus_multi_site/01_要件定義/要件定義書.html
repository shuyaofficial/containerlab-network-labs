要件定義書 — Theme 25: Multi-Campus

1. プロジェクトの目的 #

株式会社XXX（架空）は、本社キャンパス（Main Campus）と支社キャンパス（Branch Campus）を持つ。各拠点は独立したIGP（OSPF）ドメインで運用し、拠点間はeBGPで接続する。

目的は、拠点内の詳細経路を相手拠点へ漏らさない集約と、経路ループを作らない制御付き再配信の設計を学ぶことである。IPsecによるWAN暗号化は追加Missionとして分離する。

2. 対象範囲 #

2.1 対象 #

  • Main Campus: 内部ルーター1台 + 境界ルーター1台
  • Branch Campus: 境界ルーター1台 + 内部ルーター1台
  • 各キャンパスの端末1台ずつ
  • キャンパス内OSPF（別プロセス・別ドメイン）
  • 拠点間eBGP
  • 経路集約（/16単位）
  • 制御付き双方向再配信（OSPF↔BGP）
  • prefix-list、route-map、tag

2.2 対象外 #

  • STP、VLAN Trunk、HSRP
  • インターネット接続、NAT、Firewall
  • VXLAN/EVPN Overlay
  • 実機Catalyst固有の機能・性能評価
  • IPsec（追加Missionとして別扱い）

3. 業務・通信要件 #

4. 機能要件 #

4.1 物理・論理構成 #

  • Main Campusはmain-coreとmain-edgeの2台で構成する。
  • Branch Campusはbranch-edgeとbranch-coreの2台で構成する。
  • Edge間はWANリンク（198.51.100.0/30）で接続する。
  • 各キャンパス内は/30のPoint-to-Pointリンクを使用する。
  • 利用者セグメントは各キャンパスに1つずつ設ける。

4.2 IGP（OSPF） #

  • Main CampusはOSPF process 10、Branch CampusはOSPF process 20を使用する。
  • 各キャンパスの内部リンク、利用者サブネット、LoopbackをOSPF area 0で広告する。
  • WANリンクはOSPFへ参加させない。
  • Router IDは各ノードのLoopback0を使用する。

4.3 EGP（BGP） #

  • main-edgeをAS 65050、branch-edgeをAS 65060とする。
  • WANリンク上でeBGPネイバーを確立する。
  • BGP Router IDをLoopbackの値へ固定する。

4.4 集約・再配信制御 #

  • OSPFからBGPへ渡す経路をprefix-listとroute-mapで自拠点プレフィックスだけに制限する。
  • BGPでは/16集約だけを相手拠点へ広告し、配下の/24を抑止する。
  • BGPからOSPFへは相手拠点の/16だけを再配信する。
  • 再配信経路へtagを付け、再注入や経路ループを判別できるようにする。

4.5 障害対応 #

  • BGPセッション停止時に相手拠点経路がRIBから消えること。
  • BGP復旧後にeBGPと相手拠点経路が復元すること。

5. 非機能要件 #

5.1 可用性 #

  • 単一WAN接続のため、WANリンク障害で拠点間通信は停止する。
  • BGP停止・復旧の影響範囲とRIB変化を試験結果へ記録できること。

5.2 運用性 #

  • BGP表、OSPF表、RIBを区別して障害を切り分けられること。
  • 機器名、管理IP、データプレーンIP、AS番号が文書間で一致していること。
  • 第三者がREADMEから要件、設計、構築、試験の順に追えること。

5.3 学習上の要件 #

  • IOSコンフィグは自動投入しない。
  • startup-configや模範解答を同梱しない。
  • 学習者自身が設計値をIOS/Linuxコマンドへ変換する。
  • PC（Linux端末）のIPと経路のみ環境側で自動設定する。

7. 設計上の禁止事項 #

  • prefix-listなしの相互redistribute。
  • 10.0.0.0/8のような実際の拠点範囲より広すぎる集約。
  • 基礎ルーティングが未完成のままIPsecを追加すること。
  • BGP表、OSPF表、RIBを区別せず「pingが通るから完成」と判定すること。

8. 完了条件 #

  • 6ノードが起動し、5リンクが生成されること。
  • docker attachによる初期コンソールアクセスが可能であること。
  • 学習者がデータプレーン設定を投入する前は、OSPF・BGP・Loopback・インフラIPが未設定であること。
  • 試験計画書の正常系・障害系を実施できる土台があること。
