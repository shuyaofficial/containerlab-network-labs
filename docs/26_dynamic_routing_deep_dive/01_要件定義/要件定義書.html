要件定義書: 動的ルーティング深堀り

1. 背景と目的 #

中規模企業ネットワーク（本社・支社）において、ネットワークセグメントの増加や外部（ISP）との接続に伴い、静的ルーティングだけでは運用負荷が高く、障害時の自動切り替えが困難になっている。本ラボでは、要件に応じて適切な動的ルーティングプロトコル（OSPF, EIGRP, BGP）を選択・設計し、高い可用性と拡張性を持つネットワーク基盤を構築する。

2. ネットワークゾーン要件 #

2.1 本社（HQ） #

  • Trustゾーン（社内LAN）: 社内ユーザーのPCや社内システムが収容される。可用性が高く、障害時に迅速に経路が切り替わるルーティング基盤が必要。
  • DMZゾーン: 外部向け公開サーバが収容される。Trustゾーンとはルーティング的に分離しつつ、必要な通信のみを許可する基盤とする。
  • Untrustゾーン（エッジ）: ISPとの接続点。社内（プライベートIP）からインターネットへのアクセスを提供するため、NAT機能が必須。

2.2 支社（Branch） #

  • 最小限の機器構成でTrustゾーンを提供。本社やDMZゾーンへの安全なアクセスを確保する。

3. ルーティング要件 #

3.1 内部ルーティング（IGP） #

  • 本社のコアネットワークはマルチベンダー互換性を考慮し標準プロトコル（OSPF）を使用する。
  • 本社の一部フロア（ディストリビューション）はCisco独自のEIGRPで構成されており、コアのOSPFと相互接続（再配信）する必要がある。
  • 支社は設定の簡略化のためOSPFを使用する。

3.2 外部ルーティング（EGP） #

  • インターネット接続およびWAN（本社-支社間）接続にはBGPを使用する。
  • BGPを利用して、自社の経路（プレフィックス）だけを外部に広告し、ISPからはデフォルトルートや必要な経路のみを受信するよう制御する（ルートフィルタリング）。

3.3 可用性・負荷分散 #

  • 複数経路が存在する場合、等コストマルチパス（ECMP）を利用してトラフィックを分散させる。
  • リンクダウン等の障害時には、数秒以内で迂回経路に切り替わること。

4. IPアドレス要件 #

  • 内部ネットワークは 10.26.0.0/16 空間を使用。
  • 本社: 10.26.1.0/24〜10.26.10.0/24
  • 支社: 10.26.20.0/24
  • DMZ: 10.26.30.0/24
  • インターネット/WAN接続用はパブリックIPを模擬した 198.51.100.0/24 を使用する。
