なぜ「順番」で並べ替えたか #

ノード別に値だけ並んでいると、投入順が見えず次の事故が起きやすい：

  • Router ID 事故: Loopback を作る前に OSPF/BGP/EIGRP を有効化すると、意図しない Router ID が選ばれ、直すのに手戻りが発生する。
  • 再配信のトラブル: 再配信を設定する前にルートマップなどのポリシーオブジェクト（ACL、prefix-list等）を作っておかないと、不要な経路がルーティングドメインに流れ込んでしまい、ルーティングループやメモリ枯渇の原因になる。

これを避けるため、次の3原則で並べています：

  1. Router ID はプロセス有効化の前に確定 — Loopback を先に作る。
  2. ルーティングプロセスと隣接の確立 — 物理IFのIPを設定し、OSPF/EIGRP/BGPの隣接関係を成立させる。
  3. ポリシーの作成 → 適用 — route-mapやprefix-listを作成してから再配信（redistribute）やNATを適用する。
