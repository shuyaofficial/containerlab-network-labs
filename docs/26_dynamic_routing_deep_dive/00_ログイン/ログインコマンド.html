Theme 26 機器別ログインコマンド

1. 実行場所 #

以下はすべてOrbStack Linux VM clabで実行します。

ssh clab@orb
cd /Users/shuya/Documents/claude/Mac仮想環境構築/26_dynamic_routing_deep_dive

2. 稼働確認 #

cd 04_構築
./deploy.sh inspect

すべてのノード（IOLおよびLinuxコンテナ）がrunningであることを確認します。

3. Cisco IOLへのログイン #

IOLをPID1として起動し、docker attachの標準入力をIOSへ直結しています。 画面がすぐ表示されない場合は、接続後にEnterを1〜2回押してください。初回はCiscoのSystem Configuration Dialogが表示されます（noで抜ける）。

hq-core (OSPF Core) #

sudo docker attach --sig-proxy=false clab-dynamic-routing-lab-hq-core

hq-dist (EIGRP Distribution) #

sudo docker attach --sig-proxy=false clab-dynamic-routing-lab-hq-dist

hq-edge (OSPF/BGP Edge, NAT) #

sudo docker attach --sig-proxy=false clab-dynamic-routing-lab-hq-edge

hq-dmz (DMZ Switch) #

sudo docker attach --sig-proxy=false clab-dynamic-routing-lab-hq-dmz

br-core (Branch OSPF Core) #

sudo docker attach --sig-proxy=false clab-dynamic-routing-lab-br-core

br-edge (Branch BGP Edge) #

sudo docker attach --sig-proxy=false clab-dynamic-routing-lab-br-edge

isp (ISP Router) #

sudo docker attach --sig-proxy=false clab-dynamic-routing-lab-isp

コンテナを止めずに離脱 #

Ctrl-pを押し、続けてCtrl-qを押します。Ctrl-cは離脱には使いません。

4. Linux端末へのログイン #

hq-pc #

sudo docker exec -it clab-dynamic-routing-lab-hq-pc bash

dmz-srv #

sudo docker exec -it clab-dynamic-routing-lab-dmz-srv bash

br-pc #

sudo docker exec -it clab-dynamic-routing-lab-br-pc bash

Linuxシェルからはexitで抜けます。

5. ログインできない場合 #

5.1 コンテナ確認 #

sudo docker ps --format 'table {{.Names}}\t{{.Status}}' | grep dynamic-routing-lab

5.2 起動ログ確認 #

sudo docker logs --tail 50 clab-dynamic-routing-lab-hq-core

5.3 ラボを作り直す場合 #

⚠️ 保存していないIOS設定は消えるため、必要な設定を保存してから実行します。

cd 04_構築
./deploy.sh destroy
./deploy.sh deploy
