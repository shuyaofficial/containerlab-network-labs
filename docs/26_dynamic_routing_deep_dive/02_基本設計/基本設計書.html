基本設計書: 動的ルーティング深堀り

1. ネットワークゾーン設計 #

本システムは『ネットワーク・デザインパターン』のコンセプトに基づき、セキュリティレベルや用途の異なる4つのゾーンで構成する。

ゾーン 対象ノード セキュリティレベル 用途・役割
Trust hq-core, hq-dist, hq-pc, br-core, br-pc 高 社内ユーザー、社内システム。最も保護すべき領域。
DMZ hq-dmz, dmz-srv 中 外部へ公開するサーバ群。Untrustからのアクセスを受ける。
Untrust hq-edge, isp 低 インターネット（外部ネットワーク）。
WAN br-edge 低〜中 本社と支社を結ぶ拠点間ネットワーク。

2. ルーティング設計の方針 #

2.1 内部ルーティング（IGP）の使い分け #

  • OSPF（Open Shortest Path First）:
    • 本社コア（Area 0）および支社（Area 1）で採用。
    • 選定理由: リンクステート型でトポロジ全体を把握でき、ルーティングループが発生しにくい。マルチベンダー環境でのデファクトスタンダード。
  • EIGRP（Enhanced Interior Gateway Routing Protocol）:
    • 本社の一部ディストリビューション層で採用（AS 100）。
    • 選定理由: Cisco独自プロトコルだが、設定が容易で収束が極めて速い。既存Cisco環境の統合を模擬する。

2.2 外部ルーティング（EGP）と経路制御 #

  • BGP（Border Gateway Protocol）:
    • HQ・ISP・Branch間でeBGPを採用。
    • 選定理由: インターネットやWANにおいて、AS（Autonomous System）間の経路制御（ルートフィルタリング、属性による優先度調整）を柔軟に行うため。
  • 再配信（Redistribution）:
    • OSPFとEIGRPの境界（hq-core）で相互再配信を行う。
    • route-map を用いて、デフォルトルート等不要な経路が再配信されるのを防ぐ。

2.3 アドレス変換（NAT/PAT） #

  • 選定理由: Trust/DMZのプライベートIPをインターネット（Untrust）で通信可能にするため。
  • hq-edgeにて、内部（inside）から外部（outside）へ抜けるトラフィックを、インターフェースのパブリックIPへ変換（PAT）する。

3. 可用性設計 #

  • OSPFの ECMP（Equal-Cost Multi-Path） を活用し、等コストの経路がある場合はトラフィックを分散させる。
  • 万が一リンクが切断された場合、OSPFの高速コンバージェンスおよびEIGRPのFeasible Successorにより、数秒以内に迂回経路へ切り替わる設計とする。
