Phase 0 ノードへのアクセス（docker exec） #

Phase 0 は wbitt/network-multitool を client / app / external に流用する。 コンテナ名は containerlab のプレフィックス規約により clab-<lab名>-<node名> になる（実際の名前は docker ps で確認）。

# 起動中コンテナ一覧
docker ps --format 'table {{.Names}}\t{{.Image}}\t{{.Status}}'

# client 端末に入る（シェル）
docker exec -it clab-zero-client bash

# app サービス側に入る
docker exec -it clab-zero-app bash

# external（外部/未管理端末）に入る
docker exec -it clab-zero-external bash

エンドポイント疎通確認（Phase 0 ゲート） #

# client → app への疎通（Phase 0 の合格条件）
docker exec clab-zero-client ping -c 3 app
docker exec clab-zero-client curl -sS http://app/

# ネットワーク所属の確認
docker exec clab-zero-client ip -4 addr show

Phase コンポーネント 予定アクセス方法 確定時期
