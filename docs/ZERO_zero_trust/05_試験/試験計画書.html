試験方針 #

  • ゲート＝試験: 各 Phase の「次に進む条件」をそのまま試験項目にする。合格しない限り次 Phase へ進まない。
  • 証拠を残す: 認証・拒否・検知は Phase 3 以降 Grafana で件数を可視化し、「効いている証拠」を記録する。
  • 切り分けは層別で: 不合格時は 切り分けシート を複製し、層別（コンテナ→到達→TLS→認証→認可→ログ）に切り分ける。
  • 実施結果は 05_試験/試験結果_YYYY-MM-DD.md（type: test-result）に記録する。
