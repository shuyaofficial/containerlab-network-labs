NW-ZT トラック N2。SDP の本質「アプリは内向きポートを一切開けないのに、認可されたクライアントだけがオーバーレイ経由で到達できる（ダークサービス）」を実機で実証。

試験環境（docker） #

コンテナ ネットワーク 役割
ziti zn-ziti controller + router 一体（quickstart, arm64）
darkweb zn-app のみ nginx。ホスト公開ポートなし。client の居る zn-ziti には現れない
apptun zn-ziti + zn-app host tunneler。webapp を bind し darkweb:80 を dial
clienttun zn-ziti のみ proxy tunneler。webapp を localhost:8080 に射影

総括 #

SDP 型 ZTNA の核心を OSS（OpenZiti, arm64）で実証完了。 Zscaler ZPA / Cisco Secure Access が「内向きを開けず、外向きの張り出しでのみ到達させる」仕組みを、controller(=broker) / router / tunneler で 1:1 に再現できた。IAP 型（ZERO Phase2 の Pomerium＝アプリ前のリバースプロキシ関所）との違いは、アプリ側が着信ポートを持たない＝攻撃面が消える点にある。詳細な詰まりどころは 構築ログ。
