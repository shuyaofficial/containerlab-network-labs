3. L2設計（STP・アクセス層） #

3.1 STP（Spanning Tree Protocol） #

VLAN単位でSTP（PVST+）を稼働させる。VLANごとにRoot Bridgeを分散させることで、全アップリンクの帯域を有効活用する。

VLAN 用途 Root（Primary） Root（Secondary） ブロッキング想定位置
10 営業部 CoreDist-1 CoreDist-2 各AccessのCoreDist-2側リンク
20 開発部 CoreDist-2 CoreDist-1 各AccessのCoreDist-1側リンク

⚠️ 設計ゲート: STPプライオリティは必ずHSRPのActive側とRoot Primaryを一致させること。不一致が発生すると、L2のフォワーディングパスとL3のゲートウェイ到達パスが乖離し、非効率なトラフィックフローや不慮のループ・ドロップを引き起こす。

3.2 アクセス層 #

  • 各アクセススイッチから CoreDist-1 と CoreDist-2 に対して1本ずつアップリンク（トランク）を接続する。
  • 端末ポートはアクセスポートとして設定する。

4. L3設計（OSPF） #

4.1 エリア設計 #

全機器を OSPF Area 0（バックボーン） に収容する。

項目 設計
ルーターID 各機器のLoopback0（CoreDist-1: 10.255.0.11, CoreDist-2: 10.255.0.12）
P2Pリンク CoreDist1 - CoreDist2 間の物理ポート（e0/1）は no switchport のL3ポートとし、OSPFネットワークタイプを point-to-point とする。
利用者VLANのSVI OSPFで広報するが、passive-interface に設定し、端末セグメントへHelloパケットを流さない。

5. 冗長化設計（高可用性） #

対象 方式 設計
デフォルトGW（VLAN10） HSRP VIP=.254。CoreDist-1=Active (Priority 110/preempt), CoreDist-2=Standby (Priority 100)
デフォルトGW（VLAN20） HSRP VIP=.254。CoreDist-2=Active (Priority 110/preempt), CoreDist-1=Standby (Priority 100)
L2ループ防止・パス冗長 PVST+ §3.1 のとおり
コア間 OSPF L3ポートでのルーティング冗長。Accessからのアップリンクが片方死んだ場合でも、CoreDist間を通って目的のHSRP/L3機能へ到達可能

6. 試験方針 #

試験項目の正本は ../05_試験/試験計画書.md。基本設計として特に確認したい観点:

  1. 負荷分散の確認: VLAN10のトラフィックはCoreDist-1を、VLAN20はCoreDist-2を通過していること。
  2. HSRPとSTPの連携: 障害発生時（リンクダウンやノードダウン）に、HSRPとSTPが連動して数秒以内にフェイルオーバーすること。
  3. OSPFの経路冗長: CoreDist間のリンクが切れた場合、またはCoreDistが1台落ちた場合の迂回ルートの確認。
