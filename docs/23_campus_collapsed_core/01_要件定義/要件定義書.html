要件定義書（Requirements Document）— テーマ23: Collapsed Core Campus LAN

1. プロジェクトの目的 #

株式会社XXX（架空・社員約100名）は、中規模オフィス向けのネットワークリプレースを計画している。 これまでの伝統的な3階層モデル（コア、ディストリビューション、アクセス）から、コア層とディストリビューション層を統合した**「Collapsed Core（2階層モデル）」**を採用し、機器コストの削減と運用のシンプル化を図りつつ、高い可用性を維持することが本プロジェクトの目的である。

テーマ23の論点:

  • コア・ディストリビューション層の統合（Collapsed Core）: 1組の強力なL3スイッチにルーティング・HSRP・STPルートを集約する。
  • デフォルトGWの冗長化: HSRPによるゲートウェイ冗長化とSTPルートブリッジの配置を連携させる（VLANごとの負荷分散）。

3. 非機能要件 #

3.1 可用性（SLA） #

  • L3スイッチの冗長化: CoreDist層は2台構成とし、単一の機器故障で業務通信が停止しないこと。
  • デフォルトゲートウェイ: 端末から見たGWを仮想化（HSRP）し、障害時に数秒以内に自動切替されること。
  • L2冗長化とループ防止: アクセススイッチから2台のCoreDistに対して冗長リンクを構成し、STPによってループを防止しつつ障害時の自動切替を実現すること。

3.2 負荷分散（トラフィックの最適化） #

  • STPとHSRPの連携:
    • VLAN 10の通信は通常 CoreDist-1 を経由するように設計する（HSRP Active / STP Root Primary）。
    • VLAN 20の通信は通常 CoreDist-2 を経由するように設計する（HSRP Active / STP Root Primary）。
  • これにより、2台のCoreDistスイッチのリソースを有効活用（アクティブ/アクティブに近い負荷分散）する。

3.3 運用・保守性 #

  • 全てのL3スイッチにおいて、ダイナミックルーティング（OSPF）を用いて経路を自動交換し、片系障害時にもう一方の経路へ迂回できるようにすること。
  • 設計ドキュメント（基本設計書・パラメータシート）を用いて第三者が同一環境を再構築できること。

4. 制約事項（ラボ環境） #

  • 仮想環境（containerlab on OrbStack / arm64）のため、縮小モデルで構築する。
    • CoreDist層：Cisco IOL (15.7.3M2) × 2台
    • アクセス層：Cisco IOL (L2-advipservices-2017) × 2台
    • エンドポイント：Linuxコンテナ (wbitt/network-multitool) × 3台
