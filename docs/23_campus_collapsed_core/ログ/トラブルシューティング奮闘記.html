トラブルシューティング奮闘記：Collapsed Core Campus LAN

テーマ23「Collapsed Core Campus LAN」の構築と検証において、L2通信（STP/HSRP）が一切通らないという深刻なトラブルに直面しました。このドキュメントでは、その原因究明から解決、そして感動的なフェイルオーバー試験の成功までの軌跡をまとめます。

1. 遭遇したトラブル（事象） #

機器に完璧なコンフィグ（VLAN、Trunk、STP、HSRP）を流し込んだにも関わらず、以下の異常な状態が続きました。

  • HSRPがActive/Unknownになる: CoreDist間でHelloパケットが届かず、Standbyルーターを認識できない。
  • STPのSplit Brain（分裂）: CoreDist1とCoreDist2の双方が This bridge is the root を主張し、ネットワークが分断されている。
  • 物理層のステータス: Ciscoのコンソール上では Line protocol is up と表示され、一見すると正常に見える。

2. 原因究明のプロセス #

① L3イメージからL2イメージへの変更 #

当初、CoreDistにL3ルーターのイメージ（15.7.3M2）を使用していましたが、これでは switchport mode trunk 等のL2機能が使えないことが判明しました。そこで、アクセススイッチと同じ L2マルチレイヤスイッチ（L2-advipservices-2017） のイメージに変更しました。

② コンソールのバグ対応が招いた悲劇 #

元のイメージのままでは docker attach でコンソールに入ると画面がフリーズする（入力不能になる）問題がありました。 これを解決するため、コンテナの起動スクリプト（entrypoint.sh）を書き換え、「IOLプロセスをバックグラウンドからフォアグラウンドに引きずり出す（execを外す）」というハックを行いました。これによりコンソールの問題は解決しました。

③ 真の犯人：iouyap のクラッシュとロックファイル #

しかし、このハックが 最悪の副作用 を引き起こしました。 Linuxコンテナの仮想NIC（veth）と、IOL内部のインターフェースをブリッジする中継プログラムである iouyap が死んでいたのです。

  • 通常、iol.bin が起動すると、/tmp/netio0/3.lck のようなロックファイルを作成します。
  • 私が iol.bin をフォアグラウンド（PID 38等）で起動させたため、後から起動した iouyap がこのロックファイルを見て 「すでに別プロセス（PID 38）が使っているから起動できない！」と勘違いし、自爆（クラッシュ） していました。
  • 結果として、Cisco上では「UP」に見えていても、裏側のシステムレベルでは物理ケーブルがすべて抜かれているのと同じ状態（パケットがブラックホールに消える状態） になっていました。

3. 解決策（ハッキング） #

原因が判明したため、稼働中のコンテナにバックグラウンドで入り込み、以下の荒療治を行いました。

  1. 邪魔をしているロックファイルを強制削除する（rm -f /tmp/netio0/*.lck）
  2. 即座に iouyap をバックグラウンドで強制起動する（/usr/bin/iouyap 3 &）

これを全4台のスイッチで実行した瞬間、ブラックホールが解消され、仮想ケーブルにパケットが流れ始めました！

4. 感動のフェイルオーバー試験（T-401 & T-403） #

ケーブルが繋がった直後、STPの再計算が走り、HSRPの unknown が相手のIPアドレスに切り替わるという劇的な復旧を遂げました。 その後実施した障害試験では、設計の正しさと冗長化プロトコルの美しさを完全に証明できました。

CoreDist1 障害時の動作（T-401） #

  • e0/2 と e0/3 を手動でダウンさせたところ、約10秒でCoreDist2がHSRPのActiveに昇格。
  • 同時にSTPの再計算が走り、今までブロックしていた Et0/3 を開放（LIS → LRN → FWD）し、見事に通信の迂回路を作り出しました。

アップリンク単一障害の L2 迂回（T-403） #

  • Access1 の CoreDist1 向けリンク（e0/1）のみを切断した場合、Access1は即座にCoreDist2向けのポートをRootポートに変更。
  • CoreDist2も自身のブロックポート（Access2向け）を開放し、「Access1 → CoreDist2 → Access2 → CoreDist1」という見事なL2の迂回ルートを形成し、通信を維持しました。
