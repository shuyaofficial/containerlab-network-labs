# テーマ35 Faucet SDN｜配線・インフラ実機検証結果（Phase A/B）

> 本書は **配線（プラミング）・インフラの実機検証**の記録であり、学習課題（Mission=`faucet.yaml` での VLAN 分離・ACL 宣言）の答えではない。検証にはスモークテスト用の最小 `faucet.yaml`（全ポート単一VLANのベースライン＝Mission未実施の状態）を用い、**検証後に削除して学習用の空状態へ戻した**。Mission の試験は学習者が実施し別途 `試験結果_*.md` に記録する。

## 実施サマリ

- 実施日: 2026-07-07 / 環境: OrbStack VM `clab`（Ubuntu 24.04 arm64、containerlab 0.77.0）
- 範囲: Phase A（OVS↔Faucet 配線・netdev データパス・L2転送）＋ Phase B（NDR連携フィード：ミラー/SPAN・gauge統計）
- 結果: **全項目合格**。検証用トポロジ・スモークテスト config は撤去済み（ランタイム残骸なし）。

## 検証中に是正した"実機でしか出ない"事項（部材の範囲）

1. `privileged:` は containerlab の無効フィールド → 削除（containerlab は linux ノードを既定で privileged 起動）。
2. OVS のポート収容を entrypoint で行うと、containerlab の veth 配線前に走って取りこぼす → インターフェース出現待ちループ＋ログdir作成を追加。

## 設計前提の訂正

- 事前想定「OrbStack は openvswitch カーネルモジュール非対応」は、本VMでは**実測でロード済み**（`datapath_type=system` も選択可能）。設計は可搬性優先で `netdev`（userspace）を継続採用し、疎通を確認済み。README・要件定義書・基本設計書に反映済み。

## 残課題

- Mission（`faucet.yaml` での VLAN 分離・ACL・任意のL3）は学習者が実施し、別途記録する。
- Phase B の消費側（Zeek/Suricata・goflow2・Grafana）への実接続は NDR 基盤本体（[42_ndr_flow](../../42_ndr_flow/README_Lab_Challenge.md)）側で行う。本検証は「SDN側フィード（ミラー・gauge統計）が出ること」までを確認。
