route-map は ON。 EIGRP から戻る HQ 10.26.10.0/24 は関所を通過し、外から来た default 0.0.0.0/0 は関所で遮断されます。だから輪が閉じません。

3つの理由

行き（OSPF→EIGRP）に route-map が要らない理由

「行きに無い＝手抜き」ではありません。危険が無い向きなので、置いていないだけです。

01

輪は1回切れば止まる

相互再配信は OSPF→EIGRP→OSPF の一周の輪。ループはこの輪が閉じたときに起きます。

関所を 戻り道（EIGRP→OSPF）に1つ置けば輪は切れる。両方向に置くのは二重で、片方は無駄。

片側だけで十分
02

EIGRP 側は袋小路

hq-core の EIGRP 隣は hq-dist 1台だけ。その先は HQ内勤 LAN のみで、別ドメインへの出口がありません。

だから DMZ も default も全部そのまま渡してよい。むしろ hq-dist は default が欲しい（外に出るため）。絞ると逆に困る。

全部欲しい・漏れない
03

危ないのは「逆流」

OSPF 側には hq-edge が BGP から入れた 0.0.0.0/0 がいる。行きで EIGRP に入るのは OK。

でも戻り道で OSPF へ逆流すると本物の default と競合して事故。だから戻り道に「10.26.0.0/16 だけ」の関所を置く。

戻り道でだけ止める
