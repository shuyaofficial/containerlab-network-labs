# 試験計画書: 動的ルーティング深堀り

本試験は、設定した各種ルーティングプロトコル（OSPF, EIGRP, BGP）が要件通りに動作していること、および再配信やNATが正しく機能していることを確認するためのものです。

## 1. OSPF隣接関係と経路の確認 (Mission 1)
- [ ] **hq-core** で `show ip ospf neighbor` を実行し、`hq-edge` および `hq-dmz` とFULL状態になっているか確認する。
- [ ] **hq-core** のルーティングテーブルに `10.26.30.0/24` (DMZ) の経路が `O` で存在することを確認する。

## 2. EIGRP隣接関係と再配信の確認 (Mission 2)
- [ ] **hq-dist** で `show ip eigrp neighbors` を実行し、`hq-core` とネイバー関係が確立されているか確認する。
- [ ] **hq-core** で `show ip route` を実行し、`10.26.10.0/24` (hq-pc) が `D` で見えているか確認する。
- [ ] **hq-edge** で `show ip route` を実行し、`10.26.10.0/24` が `O E2` で見えているか確認する。

## 3. BGP隣接関係とNATの確認 (Mission 3)
- [ ] **hq-edge** で `show ip bgp summary` を実行し、`isp` (AS 65100) とピアが確立しているか確認する。
- [ ] **hq-edge** が `isp` からデフォルトルート `0.0.0.0/0` をBGP経由で学習しているか確認する。
- [ ] **hq-pc** または **dmz-srv** から `isp` (198.51.100.2 または 10.26.255.100) への ping が通るか確認する。
- [ ] ping 実行中に **hq-edge** で `show ip nat translations` を実行し、PATが機能しているか確認する。

## 4. 支社との通信と経路制御の確認 (Mission 4)
- [ ] **br-edge** で `show ip bgp summary` を実行し、`isp` (AS 65100) とピアが確立しているか確認する。
- [ ] **isp** 側で `show ip bgp` を実行し、`br-edge` からは `10.26.20.0/24` の経路（または集約経路）のみを受信しているか確認する（他の10.26.x.xを受信していないこと）。
- [ ] **br-pc** から **hq-pc** (10.26.10.10) および **dmz-srv** (10.26.30.10) へ ping が通るか確認する。

## 5. 負荷分散の確認 (Mission 5)
- [ ] （※等コスト経路がある場合）対象ルータで `show ip route` を実行し、特定の宛先に対して2つのネクストホップが登録されているか確認する。
- [ ] `traceroute` コマンドでトラフィックが複数経路に分散される様子（またはロードバランス状態）を確認する。

## 6. 総合障害試験 (Mission 6)
- [ ] 通信中に `hq-core` と `hq-edge` 間のリンクを `shutdown` する。
- [ ] OSPFの経路が代替経路に切り替わり、通信が復旧するまでの時間を計測する。
- [ ] 復旧後、再配信によるルーティングループが発生していないか `show ip route` や `traceroute` で確認する。
