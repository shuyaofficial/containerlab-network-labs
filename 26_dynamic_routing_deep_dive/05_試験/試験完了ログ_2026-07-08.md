# Theme 26: Dynamic Routing Deep Dive 完了ログ

**日付**: 2026-07-08
**状態**: 完了 (Pingエンドツーエンド疎通確認済み)

## 1. 構築と試験のサマリー
HQ（本社）から Branch（支社）まで、OSPF、EIGRP、BGP、NATを組み合わせた複雑なルーティング構成を構築し、疎通を確立しました。

### 達成事項
* **IGPの構築**: 
  * OSPF (Area 0, Area 1) および EIGRP (AS 100) の構築。
  * `hq-core` における、route-mapを用いた OSPF ↔ EIGRP 間の安全な相互再配信（Mutual Redistribution）の確立。
* **EGPの構築**:
  * ISPを介した eBGPピアリング（AS 65000, 65001, 65100）。
  * プレフィックスリストによる経路広告の制御と、`default-originate` を用いたISPからのデフォルトルートの配信。
* **NATの構築**:
  * `hq-edge` において、`ip nat inside/outside` および PAT（overload）による社内IPの隠蔽。

## 2. 発生した問題とトラブルシューティング
今回のラボ構築において、非常に実践的かつ深いトラブルシューティングが行われました。

1. **Cisco IOL NATメモリ枯渇問題 (MALLOCFAIL)**
   * **事象**: `ip nat inside` 投入時に `%SYS-2-MALLOCFAIL` と `CFT initialization failed!` が発生。
   * **解決**: 独自の起動スクリプト `entrypoint_attach.sh` 内で、`iol.bin` の起動オプションに `-m 1024` (RAM 1GB化) を明示的に付与することで解決。
   * **記録**: 詳細は `04_構築/トラブルシューティング_IOL_NAT_MALLOCFAIL.md` に格納済み。

2. **Linuxコンテナのデフォルトゲートウェイのズレ**
   * **事象**: `hq-pc` からのPingが外に出ない。
   * **解決**: Dockerが自動付与する管理用NW（eth0）へのデフォルトルートを削除し、内部ルータ（eth1）へ向けるよう `ip route` コマンドで修正。

3. **ルーティングリレーの断絶（ホスト到達不能）**
   * **事象**: Pingが `hq-edge` (10.26.1.6) まで届くが、そこで `Destination Host Unreachable` になる。
   * **解決**: `br-core` のOSPF設定漏れにより支社NW（10.26.20.0/24）がBGPに載っていなかった問題を特定。OSPFを起動し、さらに BGP / OSPF 間の `default-information originate` を適切に伝播させることで全拠点間の道を開通させた。

## 3. 結論
NAT越え、再配信、BGP広告がすべて正常に動作していることを `show ip nat translations` および `br-pc` 宛のPing（0% packet loss）にて確認。Theme 26のラボは完全に成功し、クローズとする。
