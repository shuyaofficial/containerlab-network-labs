# パラメータシート（詳細設計）— ALog再現

本テーマはClaude実装テーマであり（[要件定義書](../01_要件定義/要件定義書.md) 分担方針）、設定値を明記する。原本は[04_構築/deploy.sh](../04_構築/deploy.sh)等の実ファイルであり、本書はそれを設計値として整理したものである。

## 1. ネットワーク設計（他ラボと衝突回避のため固定）

| 項目 | 値 |
|---|---|
| ネットワーク名 | alog-lan |
| bridge名 | alog-br0 |
| subnet | 172.41.0.0/24 |
| bridge gateway | 172.41.0.1（docker自動割当） |
| logsrc | 172.41.0.11 |
| scanner | 172.41.0.12 |

## 2. ポート設計（他ラボと衝突回避のため固定）

| サービス | ホスト公開ポート | 備考 |
|---|---|---|
| Grafana | 3001 | `GF_SERVER_HTTP_PORT=3001`、host mode |
| Loki | 3101 | `server.http_listen_port: 3101`、host mode |
| Loki gRPC | 9097 | 内部用。42_ndr_flowの9096と衝突回避のため変更 |
| MinIO API | 9001 | `--address :9001` |
| MinIO Console | 9002 | `--console-address :9002` |
| Vector syslog受信 | 5514(TCP/UDP) | `sources.syslog_tcp`/`syslog_udp` |
| Vector metrics | 9598 | `prometheus_exporter`シンク |
| Promtail HTTP | 9081 | 内部用。42_ndr_flowの9080と衝突回避のため変更 |
| Webhook受信器 | 9099 | `socat -u TCP-LISTEN:9099,fork,reuseaddr OPEN:...,creat,append` |

| grafana | grafana/grafana:latest | host | 可視化・アラート評価 |
| webhook | wbitt/network-multitool:latest | host | アラート受信確認用socatリスナー |

## 4. Vector（正規化パラメータ）

| 項目 | 値 |
|---|---|
| syslogソース | TCP/UDP 0.0.0.0:5514（`type: syslog`） |
| Loki push先 | http://localhost:3101/loki/api/v1/push（`endpoint: http://localhost:3101`） |
| ラベル（低カーディナリティのみ） | job=alog, event={{event}}, source_type={{source_type}} |
| 正規化パターン1 | SSH認証失敗 → `event=auth_fail`, source_type=server |
| 正規化パターン2 | sudo実行 → `event=priv_exec`, source_type=server |
| 正規化パターン3 | AD風ログオン失敗 → `event=ad_logon_fail`, source_type=ad |
| 正規化パターン4 | NW機器インターフェース変化 → `event=nw_if_down`, source_type=network |
| 非マッチ時 | `event=raw`, source_type=unknown（生ログをそのまま保管） |
| メトリクス公開 | `internal_metrics` → `prometheus_exporter`（0.0.0.0:9598） |

詳細実装: [04_構築/vector/vector.yaml](../04_構築/vector/vector.yaml)。

## 8. Grafana Alerting（3観点検知）

| ルールuid | 観点 | LogQL条件 | 評価間隔/for |
|---|---|---|---|
| alog-count-change-auth-fail | 件数の変化 | `sum(count_over_time({job="alog",event="auth_fail"} \| json [5m])) > 10` | 1m / for 1m |
| alog-new-appearance-src-ip | 新規出現 | `sum by(src_ip)(count_over_time(...[5m])) unless sum by(src_ip)(count_over_time(...[1h] offset 5m))`、`> 0` | 1m / for 0s |
| alog-value-change-event-volume | 値の変化 | `sum(count_over_time({job="alog"} \| json [5m])) > 50` | 1m / for 1m |

| 項目 | 値 |
|---|---|
| Contact point | alog-webhook（type: webhook, url: http://localhost:9099） |
| 通知ポリシー | axis(count_change/new_appearance/value_change) 全てをalog-webhookへルーティング |

詳細実装: [04_構築/grafana/provisioning/alerting/](../04_構築/grafana/provisioning/alerting/)。

## 9. deploy.sh 攻撃トリガの内訳（attack サブコマンド）

| トリガ | 内容 | 対応する観点 |
|---|---|---|
| (a) 認証失敗連打 | 198.51.100.20（logs実行で既知化済み）から18回のFailed password | 件数の変化 |
| (b) 新規IP出現 | 203.0.113.200（未知）から1回のFailed password | 新規出現 |
| (c) イベント件数急増 | sudo実行ログを35件追加注入 | 値の変化（(a)と合算し5分間で50件超を狙う） |
| 参考: SYNスキャン | scanner(172.41.0.12)→logsrc(172.41.0.11)、`nmap -sS -p 1-200` | Suricata sid:2000001（多様なログソースの補完） |
