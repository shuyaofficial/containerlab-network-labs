# Theme 25 機器別ログインコマンド

## 1. 実行場所

以下はすべてOrbStack Linux VM `clab`で実行します。

```bash
ssh clab@orb
cd /Users/shuya/Documents/claude/Mac仮想環境構築/25_campus_multi_site
```

## 2. 稼働確認

```bash
cd 04_構築
./deploy.sh inspect
```

6台すべてが`running`であることを確認します。

## 3. Cisco IOLへのログイン

IOLをPID1として起動し、`docker attach`の標準入力をIOSへ直結しています。

画面がすぐ表示されない場合は、接続後にEnterを1〜2回押してください。初回はCiscoの`System Configuration Dialog`が表示されます。セットアップウィザードを使うかどうかは自分で選択します。

### main-core

```bash
sudo docker attach --sig-proxy=false clab-campus-multi-site-main-core
```

### main-edge

```bash
sudo docker attach --sig-proxy=false clab-campus-multi-site-main-edge
```

### branch-edge

```bash
sudo docker attach --sig-proxy=false clab-campus-multi-site-branch-edge
```

### branch-core

```bash
sudo docker attach --sig-proxy=false clab-campus-multi-site-branch-core
```

### コンテナを止めずに離脱

`Ctrl-p`を押し、続けて`Ctrl-q`を押します。`Ctrl-c`は離脱には使いません。

## 4. Linux端末へのログイン

### main-pc

```bash
sudo docker exec -it clab-campus-multi-site-main-pc bash
```

### branch-pc

```bash
sudo docker exec -it clab-campus-multi-site-branch-pc bash
```

Linuxシェルからは`exit`で抜けます。

## 5. SSHについて

containerlab公式の標準管理方法はSSHですが、このラボの旧IOLイメージでは初期SSH bootstrapが完了しません。自分でSSH管理設定を作成するまでは、次の形式を使いません。

```bash
ssh admin@clab-campus-multi-site-main-edge
```

## 6. ログインできない場合

### 6.1 コンテナ確認

```bash
sudo docker ps --format 'table {{.Names}}\t{{.Status}}' | grep campus-multi-site
```

### 6.2 IOLがPID1か確認

```bash
sudo docker top clab-campus-multi-site-main-core
```

`/iol/iol.bin`がPID1のプロセスとして表示されることを確認します。

### 6.3 起動ログ確認

```bash
sudo docker logs --tail 50 clab-campus-multi-site-main-core
```

### 6.4 iouyap再起動

```bash
cd 04_構築
./deploy.sh iouyap
```

### 6.5 ラボを作り直す場合

> 保存していないIOS設定は消えるため、必要な設定を保存してから実行します。

```bash
cd 04_構築
./deploy.sh destroy
./deploy.sh deploy
```

## 7. 初期状態の境界

環境側が用意するのは、IOL起動、仮想配線、attach可能なentrypoint、`iouyap`ブリッジです。学習用コンフィグは投入しません。

次の項目は未設定です。

- Ethernet0/1以降のデータプレーンIP
- Loopback0
- OSPF（process 10 / 20）
- BGP（AS 65050 / 65060）
- prefix-list / route-map
- 集約経路 / 再配信
