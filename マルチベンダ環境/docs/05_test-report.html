冗長試験（デュアルホーム切替）

hq-edge は isp-a / isp-b の2本でインターネット側へデュアルホームしている。isp-a向けリンク（ether2）を落とし、経路が isp-b 側へ切り替わって疎通が維持されることを確認する（試験計画書 §3 の手動手順。verify.sh の自動28項目には含まない参考検証）。198.51.100.1 は isp-b が eBGPで再広報しているため、ether2 断でも到達は維持される。

3-1 障害前 OK

基準の確認

hq-pc1 → 198.51.100.1 が到達。

docker exec clab-mvlab-hq-pc1 ping -c2 198.51.100.1
3-2 リンク断

isp-a向け遮断

ether2 を disable。to-isp-a BGPセッションが断。

ssh admin@172.20.50.11 "/interface disable ether2"
3-3 切替後 OK

疎通維持

hq-pc1 → 198.51.100.1 が到達を維持（isp-b→isp-a 経由へ迂回）。

docker exec clab-mvlab-hq-pc1 ping -c4 198.51.100.1
3-4 迂回 OK

経路の迂回確認

to-isp-b のみ表示（to-isp-a は消える）。

ssh admin@172.20.50.11 "/routing/bgp/session print"
3-5 復旧 OK

切戻し

ether2 enable。to-isp-a BGP再確立、両セッションestablished。

ssh admin@172.20.50.11 "/interface enable ether2"

※ ダウンタイムの秒数は本試験の測定対象外（verify.sh は到達可否を判定し、切替所要時間の実測値は取得していない）。切替・復旧ともに疎通維持を確認済み。 REQ-MV02
