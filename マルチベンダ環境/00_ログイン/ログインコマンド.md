# マルチベンダ環境 機器別ログインコマンド

## 1. VM接続

以下はすべてOrbStack Linux VM `clab`（`ssh clab@orb`）で実行します。Macホストから`containerlab`や
各種`curl`を直接実行しない（Dockerソケット不一致・mgmtネットワーク非公開のため失敗する）。

```bash
ssh clab@orb
cd "/Users/shuya/Documents/claude/Mac仮想環境構築/マルチベンダ環境"
```

## 2. 稼働確認

```bash
cd 04_構築
./deploy.sh inspect
```

全13ノード（IOL×2、RouterOS CHR、SR Linux×2、FRR×2、OpenWrt、端末×5）が`running`であることを確認します。

## 3. Cisco IOL（console／docker attach、SSH不可）

GNS3世代IOLはstartup-configからのRSA鍵生成ができずSSH不可のため、`docker attach`のみで運用する。

### hq-core（HQキャンパスコア、OSPF/eBGP境界）
```bash
sudo docker attach --sig-proxy=false clab-mvlab-hq-core
```

### hq-sw（HQキャンパススイッチ、VLAN10/20 SVI）
```bash
sudo docker attach --sig-proxy=false clab-mvlab-hq-sw
```

### コンテナを止めずに離脱
`Ctrl-p` を押し、続けて `Ctrl-q` を押します。`Ctrl-c` は離脱には使いません（コンテナごと終了する）。

## 4. MikroTik RouterOS（hq-edge、SSH + REST）

| 項目 | 値 |
|---|---|
| SSH | `ssh admin@172.20.50.11`（パスワード `admin`） |
| REST | `https://172.20.50.11/rest/...`（自己署名証明書、`-k`必須） |

```bash
# SSH（対話CLI）
ssh admin@172.20.50.11

# REST API（例: /system/resource取得。付録2実測パターンと同一）
curl -sk -u admin:admin https://172.20.50.11/rest/system/resource
```

> RouterOS 7.21.4は`/rest`が平文httpでも応答する（7.5はwww-ssl経由のhttpsのみ）。
> 出典: [環境/イメージ可用性検証_2026-07-17.md](../../環境/イメージ可用性検証_2026-07-17.md) 付録2。
> 本ラボはhttps（自己署名証明書、`-k`で検証スキップ）を標準手順とする。

## 5. Nokia SR Linux（dc-leaf1/dc-leaf2、SSH + JSON-RPC）

| 項目 | 値 |
|---|---|
| SSH | `ssh admin@172.20.50.13`（dc-leaf1）/ `ssh admin@172.20.50.14`（dc-leaf2） |
| パスワード | `NokiaSrl1!` |
| JSON-RPC | `https://172.20.50.1{3,4}/jsonrpc` |

```bash
# SSH（sr_cli）
ssh admin@172.20.50.13

# JSON-RPC API（例: interfaceのstate取得）
curl -sk -u admin:'NokiaSrl1!' -X POST https://172.20.50.13/jsonrpc \
  -H 'Content-Type: application/json' \
  -d '{"jsonrpc":"2.0","id":1,"method":"get","params":{"commands":[{"path":"/interface","datastore":"state"}]}}'
```

## 6. FRR（isp-a/isp-b、docker exec + vtysh）

```bash
docker exec -it clab-mvlab-isp-a vtysh
docker exec -it clab-mvlab-isp-b vtysh
```

非対話で1コマンドだけ実行する場合:
```bash
docker exec clab-mvlab-isp-a vtysh -c "show ip bgp summary"
```

## 7. OpenWrt / 端末（docker exec + sh）

### br-edge（ブランチCPE）
```bash
docker exec -it clab-mvlab-br-edge sh
```

### 端末（hq-pc1 / hq-pc2 / srv-web / srv-db / br-pc）
```bash
docker exec -it clab-mvlab-hq-pc1 sh
docker exec -it clab-mvlab-hq-pc2 sh
docker exec -it clab-mvlab-srv-web sh
docker exec -it clab-mvlab-srv-db sh
docker exec -it clab-mvlab-br-pc sh
```

Linuxシェルからは `exit` で抜けます。

## 8. Macから直接アクセスする場合の到達性について

mgmtネットワーク（`mvlab-mgmt` / 172.20.50.0/24）はOrbStack Linux VM内のDocker管理ブリッジであり、
Macホストからの直接到達は保証されない。本書のSSH／REST／JSON-RPCのコマンド例は**すべてVM内
（`ssh clab@orb`でログインした後のシェル）から実行する**こと。Macのブラウザ等からどうしても
直接アクセスしたい場合は、VM側で`ssh -L`のポートフォワードを別途張る（本書の対象外）。

## 9. ログインできない場合

### 9.1 コンテナ確認
```bash
sudo docker ps --format 'table {{.Names}}\t{{.Status}}' | grep mvlab
```

### 9.2 起動ログ確認
```bash
sudo docker logs --tail 50 clab-mvlab-hq-edge
```

### 9.3 ラボを作り直す場合
> 保存していない設定は消えるため、必要な設定を保存してから実行します。
```bash
cd 04_構築
./deploy.sh destroy
./deploy.sh deploy
```
