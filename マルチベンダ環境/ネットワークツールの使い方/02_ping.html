02. Ping実行・確認

:8602 主力 最終更新 2026-07-17

複数ターゲットの死活をカードで並べて監視し続けるダッシュボード。RTT・loss%がリアルタイムに流れ、断イベントを自動記録し、障害試験レポートとCSVを出力できる。

cd /Users/shuya/Documents/claude/ネットワークツール/02_ping && bash start.sh

このラボでの位置づけ#

主力 MacのICMPで13ノードの到達性を直接監視できる、切り分けの起点になるツール。ただしMac側のルート追加（setup_mac.sh）が前提。ルートを追加していない状態では全ターゲットが応答なしになる。

セットアップ（登録する値）#

まず前提として、Macからmgmtサブネット（172.20.50.0/24）へ到達させる。

cd "/Users/shuya/Documents/claude/Mac仮想環境構築/マルチベンダ環境/ネットワークツールの使い方"
./setup_mac.sh

使い方ステップ#

  1. setup_mac.shを実行し、疎通確認（ping -c 2 172.20.50.21相当）が通ることを確認する。
  2. bash start.shで起動し、http://127.0.0.1:8602 を開く。
  3. topbarの＋ターゲット追加で上表の13ノードを登録する。名前・ホスト・グループ（HQ / DC / Branch / ISP）を入力する。
  4. 登録が終わったら全て開始を押し、13枚のカードすべてで状態ドットが緑（ok）になることを確認する。
  5. 障害試験シナリオ: 対象ノード（例: isp-a）のカードにチェックを入れ、topbarの試験開始を押す。
  6. VM側で障害を注入する（例: docker exec clab-mvlab-isp-a でインターフェースをdownさせる、または一時的にコンテナを停止する）。
  7. 断が観測できたら停止＋Tracerouteを押す。対象ホストへのtracerouteを実行してからレポートが生成される（最大45秒/ホスト）。
  8. topbarのレポートから一覧・閲覧・ダウンロードする。個別カードのCSVでRTT履歴（ts / rtt_ms / ok）も保存できる。

制約と注意#

ルート追加が必須: setup_mac.shを実行していないと、Macから172.20.50.0/24へのICMPが届かず全ターゲットが応答なしになる。ルートはMac再起動で消えるため、再起動後は再実行が必要。
保持期間: RTT履歴は直近3600サンプル（1秒間隔で約1時間分）まで。それより古い区間はCSVにも残らないため、記録したい場合は早めに書き出す。
