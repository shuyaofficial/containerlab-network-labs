01. マクロ作成

:8601 補助 最終更新 2026-07-17

Cisco IOSへ投入するコマンド列を、テンプレートと変数フォームから機械的に生成するアプリ。VLAN・インターフェース・OSPF・BGP・ACLの5テンプレートを収録し、コピー／.txt／TeraTerm用.ttlのいずれでも取り出せる。機器へは直接投入せず、生成したコマンドを手元でコピーして使う。

cd /Users/shuya/Documents/claude/ネットワークツール/01_macro && bash start.sh

このラボでの位置づけ#

補助 機器接続なし。生成はローカルで完結し、投入は人間が手作業でconsoleに貼り付ける。mvlabで唯一のCisco系機器であるhq-core（コア）・hq-sw（L2スイッチ）向けのコマンド下書き作成に使う。

セットアップ（登録する値）#

機器への接続設定は不要。生成したコマンドの貼付先は以下の2台（いずれもconsole専用）。

ノード役割貼付方法
hq-coreHQキャンパスコア（OSPF/eBGP境界）ssh clab@orb → docker attach --sig-proxy=false clab-mvlab-hq-core
hq-swHQキャンパススイッチ（VLAN10/20 SVI）ssh clab@orb → docker attach --sig-proxy=false clab-mvlab-hq-sw
 
ssh clab@orb
docker attach --sig-proxy=false clab-mvlab-hq-sw

離脱は Ctrl-P → Ctrl-Q。Ctrl-Cはコンテナごと終了するため使わない。

使い方ステップ#

  1. 上記コマンドで起動し、http://127.0.0.1:8601 を開く。
  2. サイドバーからテンプレートを選ぶ（例: hq-swのVLAN10/20を作り直す場合はL2 → VLAN作成）。
  3. 変数フォームに値を入力する（例: 「VLAN一覧」に 10 SALES と 20 DEV を1行ずつ）。入力するたびに右側のプレビューが自動更新される。
  4. インターフェース設定テンプレートでhq-swのSVIを作る場合は、IPアドレスに 10.50.10.1/24（VLAN10）や 10.50.20.1/24（VLAN20）を指定する（実データはindex.html#nodes参照）。
  5. プレビューを確認しコピー（または.txtダウンロード）で取り出す。
  6. ssh clab@orb → docker attach --sig-proxy=false clab-mvlab-hq-sw で接続し、コピーしたコマンドを貼り付けて投入する。
  7. 投入後は Ctrl-P → Ctrl-Q で離脱（コンテナは起動したまま）。

制約と注意#

既知の制限: トランクの許可VLAN（switchport trunk allowed vlan）は単一VLANのみ指定できる（カンマ区切りの複数VLANには非対応）。拡張ACLのエントリ行は構文チェックをせずそのまま投入されるため、貼り付け前に必ず目視確認すること。
投入機能はない: このツールは生成専用で、機器への直接投入（SSH/API経由の自動投入）は行わない。貼付作業は必ず人間が行う。
