05. コンフィグ管理

:8605 主力 最終更新 2026-07-17

機器のrunning-configをスナップショットとして版管理する。任意の2版を並べてside-by-side / unifiedで差分を見比べ、全文検索にも対応するミニ構成管理ツール。/snapshots/importで機器に接続せずコンフィグ本文を貼り付けて取り込める。

cd /Users/shuya/Documents/claude/ネットワークツール/05_config && bash start.sh

このラボでの位置づけ#

主力 SSH自動取得（取得ボタン）はnetmiko/paramikoのdevice_type文字列に依存し、Cisco系ネイティブでの動作を前提にした作りのため、mvlabの各ベンダー（RouterOS / SR Linux / FRR / OpenWrt / IOL）での動作は保証されない（IOLはSSH不可のため確実に失敗）。本ラボでは「ファイルから投入」による全ベンダー貼付インポート運用を正とする。

セットアップ（登録する値）#

貼付インポート運用では認証情報の登録は不要（デバイス名・host・device_typeは記録用のラベルとして入力する）。

名前hostdevice_type（ラベル）
hq-edge172.20.50.11mikrotik_routeros
hq-core172.20.50.12cisco_ios（IOL）
dc-leaf1172.20.50.13nokia_srlinux
dc-leaf2172.20.50.14nokia_srlinux

使い方ステップ#

  1. bash start.shで起動し、http://127.0.0.1:8605 を開く。
  2. サイドバーの＋デバイス追加でhq-edgeを登録する（名前・host・device_typeはラベルとして入力、パスワードは空でよい）。
  3. RouterOSへログインして設定を取得する。
ssh mv-hq-edge
/export
  1. 出力全体をコピーし、05_configで対象デバイス（hq-edge）を選択してファイルから投入→貼り付け→投入。1件目のスナップショットとして記録される。
  2. RouterOS側で変更を加える（例: ファイアウォールルールを1本追加）。
  3. 再度 /export を実行し、出力をファイルから投入で2件目として取り込む。
  4. スナップショット一覧で2件にチェックを入れると差分パネルが自動で開く。side-by-side / unifiedを切り替えて変更点を確認する。

制約と注意#

SSH自動取得はCisco向け固定: 「取得」ボタンはnetmikoのdevice_type（既定cisco_ios）を前提にしたshow running-config実行のため、RouterOS/SR Linux/FRR/OpenWrtでの動作は未検証。IOL（hq-core/hq-sw）はSSH不可のため確実に失敗する。本ラボでは全ベンダーとも「ファイルから投入」を使う。
パスワードの扱い: 貼付インポートのみで使う場合、デバイス登録時のパスワード欄は空のままでよい（取得機能を使わないため）。
