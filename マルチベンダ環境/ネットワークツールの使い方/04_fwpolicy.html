04. FWポリシー管理

:8604 補助 最終更新 2026-07-17

ベンダー中立のFWポリシー表を管理し、「この通信はどのルールに当たるか」に即答するツール。到達不能（shadow）・冗長（redundant）・重複（duplicate）ルールを検出し、Cisco ASA / FortiGate構文への書き出しにも対応する。

cd /Users/shuya/Documents/claude/ネットワークツール/04_fwpolicy && bash start.sh

このラボでの位置づけ#

補助 機器接続なし。mvlabの実機（RouterOS / IOL / SR Linux）へポリシーを直接投入する機能はない。ゾーン間（HQ・DC・Branch・Internet）の通信ポリシーを設計・可視化するモデリング用途として使う。

セットアップ（登録する値）#

機器への接続設定は不要。ポリシー設計のインプットとして使うmvlabのセグメント一覧（index.html#nodes参照）。

ゾーンセグメント代表ホスト
HQ10.50.10.0/24, 10.50.20.0/24hq-pc1 (10.50.10.101), hq-pc2 (10.50.20.102)
DC10.50.30.0/24, 10.50.31.0/24srv-web (10.50.30.103), srv-db (10.50.31.104)
Branch192.168.40.0/24br-pc (192.168.40.100、NAT裏)
Internet198.51.100.0/24isp-a (.1), isp-b (.2)

使い方ステップ#

  1. bash start.shで起動し、http://127.0.0.1:8604 を開く。
  2. ルール追加で、HQ→DCの通信ポリシーをモデリングする（例: 名前 HQ-to-DC-web、送信元 10.50.10.0/24、宛先 10.50.30.103/32、プロトコル tcp、ポート 80、アクション permit）。
  3. 同様にDC→HQのdeny、Branch→Internetのpermitなど、ラボのゾーン設計に沿ったルールを積み上げる。
  4. 上部の送信元IP・宛先IP・プロトコル・ポートを入力して検索し、「この通信はどのルールに当たるか」をパケットマッチ検索で即答させる。
  5. 右上の分析バッジ（「問題n件」）を開き、shadowed/redundant/duplicate/overlapの指摘を確認して整理する。
  6. 設計が固まったらエクスポート ▾からCisco ASA構文またはFortiGate構文を書き出し、設計ドキュメントとして保存する。

制約と注意#

実機への直接投入は不可: 生成されるASA/FortiGate構文はいずれもmvlabの実機（RouterOS / IOL / SR Linux）とは構文体系が異なり、そのまま投入することはできない。あくまで設計・可視化・レビュー用のドキュメントとして使う。
src/dstの制約: 単一CIDRまたはanyのみ対応（IPv4のみ）。複数CIDRのグループ化はできない。
