03. ログ取得・差分

:8603 限定的 最終更新 2026-07-17

SSHで登録済み機器へ接続し、show系コマンドを一括実行してbefore/after差分を取るツール。デバイス登録画面のdevice_typeがドロップダウンでcisco_ios / juniper_junos / arista_eosに固定されている。

cd /Users/shuya/Documents/claude/ネットワークツール/03_log && bash start.sh

このラボでの位置づけ#

限定的 mvlabの6ベンダー構成のうち、device_typeドロップダウンが対応するのはCisco系のみ。しかしmvlabのCisco機器（hq-core / hq-sw）はいずれもIOLでSSH不可（console専用）のため、このラボでは実質的に使用できない。本ページは参考記載として、対応ベンダーが増えた場合の使い方を示す。

セットアップ（登録する値）#

該当なし。mvlabの現行構成では登録しても取得が成功する機器がない（IOLはSSH不可、RouterOS/SR Linux/FRR/OpenWrtはdevice_typeドロップダウンに存在しない）。show系一括取得が必要な場合は06_terminalのスニペット機能、または05_configの貼付インポートを使う。

使い方ステップ（参考）#

  1. bash start.shで起動し、http://127.0.0.1:8603 を開く。
  2. デバイス追加で名前・host・port・device_type（cisco_ios / juniper_junos / arista_eos）・usernameを登録する。
  3. 取得したいshowコマンド集合を選び、before側のスナップショットを取得する。
  4. 設定変更を実施したあと、再度取得してafter側のスナップショットを作る。
  5. 2つのスナップショットを比較し、差分を確認する。

mvlab上でこの手順を実行できるノードは現状ない（下記の制約参照）。

制約と注意#

本ラボでは使用不可: hq-core / hq-sw（IOL）はGNS3世代のイメージでSSH鍵生成ができず、console専用（docker attachのみ）。device_typeドロップダウンがCisco/Juniper/Arista固定のため、RouterOS（hq-edge）・SR Linux（dc-leaf1/2）・FRR（isp-a/b）・OpenWrt（br-edge）も対象外。
代替手段: show系一括取得は06_terminalのスニペット（マクロ）機能で代替できる。before/after差分が主目的なら05_configの貼付インポート＋diffの方が本ラボには適している。
