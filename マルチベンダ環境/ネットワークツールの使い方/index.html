ネットワークツールの使い方

v1.0 公開済み 最終更新 2026-07-17 対象ラボ: mvlab（マルチベンダ環境）

自作のFastAPI製ネットワークツール7本（01_macro〜07_triage、いずれもポート8601〜8607）を、このマルチベンダcontainerlabラボ（RouterOS / Cisco IOL / Nokia SR Linux / FRR / OpenWrt の6ベンダー構成）でどう使うかをまとめたガイド。各ツールはbash start.shで個別に起動し、すべて127.0.0.1固定・CDN不使用で完結する。ベンダーによって接続方式（SSH／console専用／REST／JSON-RPC）が異なるため、ツールごとに「このラボでの位置づけ」を明記した。

mgmtサブネットは 172.20.50.0/24。全13ノードの接続方法と認証情報。各ツールページではこの表の該当部分だけを抜粋して掲載する。

1. ツール一覧

全ツール共通: 各フォルダで bash start.sh（初回はvenv構築で要ネット接続）。127.0.0.1固定。NT_NO_OPEN=1でブラウザ自動オープンを抑止できる。

補助:8601

01. マクロ作成

Cisco IOS投入コマンドの変数展開生成（.txt / TeraTerm .ttl出力）

機器接続なし。生成→IOLのconsoleへ貼付
主力:8602

02. Ping実行・確認

マルチターゲット死活監視・断イベント記録・障害試験レポート・CSV

MacのICMP。要ルート追加（setup_mac.sh）
限定的:8603

03. ログ取得・差分

SSHでshow系一括取得（before/after差分）。device_type固定

本ラボでは限定的（IOLはSSH不可）。参考記載
補助:8604

04. FWポリシー管理

FWポリシー表管理・shadow/重複検出・パケットマッチ検索

機器接続なし。ゾーンポリシー設計のモデリング
主力:8605

05. コンフィグ管理

running-config版管理・diff・全文検索。貼付インポート対応

全ベンダー貼付インポート運用が正
主力:8606

06. ターミナル

Webターミナル（タブ/SSH/telnet/シリアル/自動ログ/マクロ送信）

mv-*エイリアスで直接SSH。~/.ssh/configが有効
補助:8607

07. 障害トリアージAI

症状+show出力貼付→AI原因仮説+読み取り専用確認コマンド

要ANTHROPIC_API_KEY。貼付のみ・全ベンダー対応

2. セットアップ状況

2-1. 到達性の前提

項目状態内容
VM側 iptables（DOCKER-USER）設定済み172.20.50.0/24 のmgmt通信を許可済み
Mac側 ~/.ssh/config（mv-*）設定済みmv-hq-edge / mv-dc-leaf1 / mv-dc-leaf2 のProxyJumpエイリアス登録済み
Mac側 ルート追加（02_pingや直接curl用）ユーザー実行が必要setup_mac.shを実行（sudo 1回）。ルート先VM IPは自動検出（192.168.139.215はDHCPのため変動可能性あり）
ルート追加なしで使えるツール: 06_terminal（ProxyJump経由）・05_config（貼付インポート）・07_triage（貼付のみ）・01_macro（機器接続なし）・04_fwpolicy（機器接続なし）。02_pingの直接ICMP監視や、Macから直接curlする場合のみ下記のルート追加が必要。

2-2. setup_mac.sh の実行

02_pingの死活監視や、Macから直接curlする場合に、mgmtサブネットへのルートを追加するワンショットスクリプト。sudoで1回だけ実行する（Mac再起動で消えるため、必要時に再実行する）。

cd "/Users/shuya/Documents/claude/Mac仮想環境構築/マルチベンダ環境/ネットワークツールの使い方"
./setup_mac.sh

2-3. 参考: API実例（VM内から実行）

いずれもssh clab@orbでログインしたVM内のシェルから実行する。Mac側から直接curlする場合は2-2のルート追加が前提。

RouterOS REST（hq-edge）

curl -sk -u admin:admin https://172.20.50.11/rest/system/resource

Nokia SR Linux JSON-RPC（dc-leaf1）

curl -s -u admin:'NokiaSrl1!' http://172.20.50.13/jsonrpc \
  -H 'Content-Type: application/json' \
  -d '{"jsonrpc":"2.0","id":0,"method":"get","params":{"commands":[{"path":"/system/information/version","datastore":"state"}]}}'

FRR（isp-a、vtysh非対話実行）

ssh clab@orb "docker exec clab-mvlab-isp-a vtysh -c 'show bgp summary'"

3. 推奨ワークフロー

障害調査から復旧・記録までの推奨手順。矢印は実施順、ひし形は判断を表す。

実行ステップ 判断（ひし形） 版管理で完了

「特定できない」の場合は06_terminalへ戻り、07_triageが提示した確認コマンドの結果を追加採取して再度AIに貼る（相棒ループ）。01_macro・04_fwpolicyはこのループの外側で、投入コマンドの事前生成やポリシー設計に随時使う。
