読み方

上段が外部接続、中央がShowNet AS290相当のバックボーン、下段がMedia over IPの遠隔3拠点です。実物の800G/1.6TやIOWN APNは物理回線なので、ラボでは経路冗長と観測ポイントに置き換えています。

既存のTheme 22は `campus` / `172.20.20.0/24`。このラボは `interop-shownet` / `172.20.30.0/24` なので、同じContainerlab環境でも管理面を分離しています。

物理構成イメージ

External
外部IX/ISP `ext-bbix` と `ext-jpix`。ShowNet AS290の対外ピアを模した外部AS。
#N-1
#N-2
対外接続ルータ `edge-n1` / `edge-n2`。写真の対外接続回線ラックを、2系統のeBGP入口として再現。
#N-3
コアネットワーク `core-n3a` / `core-n3b`。ShowNet内部の冗長コア。OSPFで内部経路を作る。
#N-4
大容量トランスポート `transport-n4`。IOWN APN/長距離伝送の抽象化ノード。Media拠点の収容点。
#S-4
#S-5
セキュリティ系 `sec-s4` と `remote-s5`。リモートアクセスを管理系へ限定し、Media系への直接到達をACLで止める。

使っているネットワーク技術

技術 このラボでの場所 何を学べるか
eBGP `ext-bbix` - `edge-n1`, `ext-jpix` - `edge-n2` ShowNet AS290の対外接続。外部ASからデフォルトルート、AS290から `10.90.0.0/16` を広告する。
iBGP `edge-n1` - `edge-n2` 外部経路をAS290内部のedge間で共有する。Loopback間neighborと `next-hop-self` を確認する。
OSPF edge/core/transport/sec/remote の内部リンク ShowNet内部バックボーンの到達性。冗長リンク、迂回、障害観測を試せる。
経路集約 AS290から外部へ `10.90.0.0/16` 細かいMedia/監視/管理セグメントを外部に全部見せず、ShowNet全体を1つのprefixとして見せる考え方。
ACL `sec-s4` の `REMOTE-S5-IN` リモート入口から監視/管理系だけ許可し、Media系へ直接入れない境界制御。
Media over IP風UDP `media-x`, `yokohama-foh`, `hamamatsu-mocap` ST 2110/PTP本格実装の前段として、遠隔拠点間のUDP疎通と経路観測を練習する。
監視/AI運用の抽象化 `monitor-ai` ping/traceroute/UDP確認で、障害時にどこで切れているかを見る観測点。
IOWN APN/SRv6の抽象化 `transport-n4` と設計文書 Cisco IOL制約のため、実SRv6ではなく「長距離・大容量・経路分離」の設計思想として扱う。
