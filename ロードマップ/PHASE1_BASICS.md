## 🎯 フェーズ1のゴール

> **企業の社内ネットワークを、要件定義 → 設計（論理／物理）→ 構築 → 試験 → ドキュメント化 まで、自力で一気通貫でやり切る。**

キャプストン（第7章）は2段構えでスケールさせます：
- **基本形（テーマ21）**: 「本社＋支社の2拠点」中堅企業ネットワーク（全機能を統合する練習台）
- **最終形（テーマ22）**: そこから **大規模な社内LAN**（3階層＝コア/ディストリ/アクセス、マルチエリアOSPF、キャンパス/複数棟、経路集約、MST）へ**スケールアップ**する。

その後、ポスト・キャプストンとしてキャンパス設計を3パターンで比較します：

- **テーマ23 / Pattern A**: Collapsed Core
- **テーマ24 / Pattern B**: Routed Access
- **テーマ25 / Pattern C**: Multi-Campus

さらに、デザインパターン本を参考にした応用テーマで深堀りします：

- **テーマ26**: 動的ルーティング深堀り（OSPF/EIGRP/BGP使い分け・再配信・NAT）

ゴールの具体像とテンプレは `キャプストン/要件定義_設計_構築テンプレート.md` を参照。

## 🔁 1テーマの進め方（学習サイクルの型）

各テーマで以下を1周します：

1. **設計を読む/書く** … `challenge_guide.md`（トポロジ図・IP設計表・Mission）を用意
2. **環境を起こす** … `sudo containerlab deploy -t <topo>.clab.yml`
3. **手で構築する** … `docker attach` でCLIに入り、自力でコマンド投入
4. **障害試験で壊す** … リンクを `shutdown` して、フェイルオーバ/収束を観測
5. **振り返る** … `振り返りシート/振り返りシート.md` に「教訓＋トラブルシュート」を残す

> 鉄則: ①**復路（帰り道）**を必ず確認 ②新環境はまず `no shutdown` ③障害試験までやって初めて「できた」。
