# 今後の環境構築に向けた教訓：VPN構築のオペレーション順序

## 1. 事象の振り返り
FortiGateとCisco間のVPN構築において、以下の順序で作業を行ったところ、思わぬ罠に引っかかり長時間のトラブルシューティングが発生した。

1. VPNインターフェース（Phase 1 / Phase 2）の作成
2. スタティックルートの作成
3. 下回りのPingテスト（開通確認）
4. （VPNがIN-NEGでスタックして進まない問題が発生）
5. ファイアウォールポリシーの投入（予定だったがエラーのため保留していた）

## 2. 教訓と次回以降のアクション（FortiOS環境）

FortiOSでは「使用されないVPNトンネル（＝ポリシーがないトンネル）」はネゴシエーション自体が意図的に破棄される仕様がある。
そのため、ネットワークエンジニアの基本である「レイヤーの低いところから順番に設定して疎通確認していく」というセオリー通りに進めると、**設定が正しいのにVPNが上がらない**というドツボにハマってしまう。

**【次回のアクション】**
FortiGateが関わるVPN環境を構築する際は、以下の順番を必ず守る（設計書通りに一気に流し込む）。

1. VPNインターフェース（Phase 1 / Phase 2）の作成
2. スタティックルートの作成
3. **ファイアウォールポリシー（VPNトンネルの入出力許可）の作成** ← 重要（テスト用でも良いので `all` to `all` を入れる）
4. 最後にVPNのステータス確認およびPingテストを実施する

「VPNが上がってからポリシーを入れる」のではなく、「ポリシーを入れないとVPNが上がらない」という依存関係を意識した構成管理を行うこと。
