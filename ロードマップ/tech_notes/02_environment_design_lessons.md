# 今後の環境構築・設計に向けた改善ポイント

## 1. エッジルーターおよびISPルーターのイメージ選定
現状、エッジ（br-edge）やISP（isp）のルーターとして、L2スイッチ用のイメージ（`vrnetlab/cisco_iol:L2-advipservices-2017`）を使用している。
このイメージはデフォルトで `ip routing` が無効になっており、純粋なルーティング処理において余計なトラブル（ルーティングされない等）を招く原因となる。

**【次回のアクション】**
ルーターの役割を持つノードには、必ず適切なL3ルーターイメージ（例: `L3-adventerprisek9` など）を選定し、デフォルトでルーティングが有効な状態のものを利用する。

## 2. FortiGate等のアプライアンスにおける管理ポートのルーティング管理
FortiGateなどのアプライアンス製品をラボ環境で展開する際、管理ポート（port1等）がDHCP経由でデフォルトルートを取得してしまうケースが多発する。
これがWANポートのルーティングと競合し、通信がブラックホール化する原因となる。

**【次回のアクション】**
ラボ環境を初期構築する際、管理ポートのDHCP設定において以下のような「デフォルトゲートウェイを取得しない」設定を明示的に行うか、WAN向けのスタティックルートのディスタンス（優先度）を強制的に下げる（例: `distance 1`）設計をあらかじめ組み込む。

```text
config system interface
    edit "port1"
        set defaultgw disable
    next
end
```
