## 🎯 フェーズ2のゴール

**Cisco IOSによる伝統的な構成から一歩進み、実際の現場で使われる「マルチベンダー製品」「ゼロトラストセキュリティ」「クラウド管理」をラボ環境に導入・検証する。**

### 主なテーマ
1. **境界防御のモダナイズ**: FortiGate / Cisco ASAv を用いた次世代ファイアウォール（NGFW）の構築。
2. **ゼロトラストとリモートアクセス**: SSL-VPN、デバイスポスチャーチェック（802.1X/NAC）、SSE（Secure Access）。
3. **実務要件の統合**: ロードバランサ（LB）、NDR（フロー可視化・分析）、認証統合（RADIUS/AD）。
4. **クラウド・SDN管理**: Meraki, Aruba, Faucet SDNによるモダンな集中管理。

## ⚠️ 前提事項：仮想基盤とホストアーキテクチャ

このMacは **Apple Silicon（arm64）** のため、仮想化する製品のアプローチを分ける必要があります。

| 方針 | 内容 | 主な対象製品 |
|---|---|---|
| ✅ **Mac上のコンテナ/OSS** | arm64でそのまま無料で動くもの | DNS/RADIUS(FreeRADIUS)/踏み台/Faucet SDN/HAProxy(LB) |
| ⚠️ **x86専用アプライアンス** | 別途 x86ホスト（VMware/Proxmox）が必要 | FortiGate VM / Cisco ISE / Aruba ClearPass / ASAv / F5 BIG-IP |
| ☁️ **クラウド/サンドボックス** | DevNetやベンダーの無料トライアルを活用 | Cisco Secure Access / Meraki / ACI / Zscaler |

### フェーズA — コアサービス & 認証基盤（ネットワーク連携のための「箱」作り）
> **💡 ネットワークエンジニアとしての立ち位置**
> 実務において、DNSやAD/RADIUS、踏み台サーバの「構築」はサーバーチームやセキュリティチームが担うことが多く、NWエンジニアがゼロから構築する可能性は低いです。
> したがって、ここでの主眼は「サーバーの構築スキルを極めること」**ではなく**、「ネットワーク機器（スイッチやFW）から認証サーバ等へ連携する際の設定（例：802.1Xの設定やRADIUS連携）を検証するために、**必要最小限のダミーサーバを立てること**」に置きます。

- **テーマ28**: 業務用DNS連携（NW機器からの名前解決やFWのFQDN制御の動作確認用）
- **テーマ29**: RADIUS / 認証サーバ連携（NW機器からのRADIUS問い合わせ先の用意）
- **テーマ30**: 踏み台サーバ（NW機器への管理アクセス経路のテスト用）
- **テーマ31**: NAC / 802.1X 認証（スイッチ側の認証設定を検証するための対向サーバとして）

### フェーズB/C — 境界・リモートアクセス・WAN（次世代FW）
- **テーマ32**: Fortinet / FortiGate VM 導入（1台で複数機能をハブ化）※x86基盤要
- **テーマ33**: SSL-VPN ＋ 端末ポスチャ / アクセスチェック（FortiGate等を利用）
- **テーマ34**: SD-WAN コントローラ導入検証
- **テーマ36**: Cisco Secure Access / SSE・ZTNA トライアル（クラウド側ポリシー管理）

### フェーズD — SDN / ロードバランサ / 可視化（実務特化）
- **テーマ35**: Faucet SDN（Open vSwitch + Faucetで宣言的ネットワーク制御）
- **テーマ42**: フロー可視化＆NDR入門（NetFlow + ntopng/ElastiFlow等）✅実務案件由来
- **テーマ43**: ロードバランサ（L4/L7分散、VIP、HAProxy等）✅実務案件由来
- **テーマ44**: アプリ連携（MQ / メッセージキューの送受信概念）※任意

### フェーズE/F — クラウド管理型 & エンドポイント
- **テーマ37**: Cisco Meraki（DevNetでDashboard / API体験）
- **テーマ38**: HPE Aruba（Aruba Central + ClearPass）
- **テーマ39**: Ubiquiti UniFi / TP-Link Omada（自宅ラボ向け・コントローラ仮想化）
