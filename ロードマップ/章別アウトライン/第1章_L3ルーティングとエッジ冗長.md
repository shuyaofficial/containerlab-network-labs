## 🎯 章の狙い
- 動的ルーティング（OSPF / BGP）で「最適経路の自動計算」と「ポリシー制御」を使い分けられる
- マルチホーム環境で **往路（Local Preference）／復路（AS Path Prepend）** を制御できる
- 障害を**秒→ミリ秒**で検知（BFD）し、無瞬断に近いフェイルオーバを実現できる

## 前提
- TCP/IP基礎（IPv4 `rfc791` / TCP `rfc793`）、サブネット、スタティックルートが分かる

## 🧠 この章で押さえた「鉄則」（06振り返りより）
1. **復路（帰り道）**が対向に無ければ ping は通らない。往復セットで確認。
2. 新環境はまず **`no shutdown`**（`show ip int brief` で全UP確認）。
3. `default-information originate`（alwaysなし）は**自分がデフォルトルートを持つ**ことが広報トリガー。
4. **OSPFコスト＝社内の往路（ヘアピン防止）／ BGP LP＝境界の往路ポリシー統一**。LPは復路に効かない。
