## 🎯 ゴール像（題材）

```
            ┌──────────── インターネット ────────────┐
         [ISP-A]                                   [ISP-B]
            │   eBGP(デュアルホーム)                  │
       ┌──[ R-Edge1 ]══iBGP/OSPF══[ R-Edge2 ]──┐     （第1章: BGP/BFD）
       │        │  FW/DMZ・NAT・ACL  │            │     （第4章: ACL/NAT）
       │   ┌─[ CoreSW1 ]══EtherChannel══[ CoreSW2 ]─┐
本社HQ │   │   HSRP仮想GW / STP Root      （第2章）   │
       │   │   VLAN10 営業 / VLAN20 開発 / VLAN30 サーバ
       │   └─ DHCP/DNS/NTP（第3章）, SNMP/Syslog（第5章）
       │
       │═══════ 拠点間 IPsec VPN（第4章: rfc4301/4303）═══════
       │
   支社BR [ R-Branch ]─[ L3SW ]─ VLAN40 支社LAN（第2章/第3章）
```

- **本社(HQ)**: デュアルISP境界・冗長コア・部署別VLAN・GW冗長・DMZ/FW・各種サービス・監視
- **支社(BR)**: L3SW + ルータ、支社LAN、本社へIPsec VPN接続
- 自動化（第5章 Ansible）で設定投入・取得を効率化、（任意）IPv6デュアルスタック（第6章）

## 🏗️ 最終形：大規模社内LANへのスケール

基本形（2拠点）で全機能を一度通したら、**「設計が大規模でも破綻しない」**ことを学ぶために
本社を **3階層（コア／ディストリビューション／アクセス）のキャンパスLAN** へ拡張する。

```
            [ Core SW x2 ]              ← 高速・冗長の中枢（L3、ルーティングのみ）
           /     |      \
   [Distri-A] [Distri-B] [Distri-C]     ← 棟/フロア単位の集約（L3境界・経路集約・STP root）
      / | \      ...        ...
 [Access SW 群（各フロア）]              ← 末端ポート収容（VLAN・ポートセキュリティ）
```

| スケール時の論点 | 大規模で効く設計手法 |
|---|---|
| 経路数の爆発 | OSPF **マルチエリア**化＋**経路集約**（エリア境界で束ねる） |
| STPの複雑化 | **MST**（VLAN群をインスタンスに束ねる）、ディストリでL2/L3境界を切る |
| アドレス枯渇/管理 | 階層的IP設計（棟→フロア→VLANで集約しやすく） |
| 単一障害点 | 各階層を冗長化（コア/ディストリ二重化、上下リンク冗長） |
| 拠点増 | 支社テンプレを**横展開**（同じ設計を複製できる標準化） |

> 仮想環境（arm64/IOL）はノードが重いので、**「縮小モデルで原理を確認 → 設計書では大規模を表現」**
> という二段構えが現実的。ノードは原理確認に必要な最小数（例: コア2＋ディストリ2＋アクセス2）に絞る。

## 🏁 進め方（5フェーズ・詳細はテンプレ参照）
1. **要件定義** … 事業/規模/可用性(SLA)/セキュリティ/拠点間要件を文書化
2. **設計** … 論理（IP/VLAN/ルーティング/冗長/QoS/セキュリティ）＋物理（機器/結線/構成図）
3. **構築** … containerlab トポロジ化 → 段階投入 → Ansible自動化
4. **試験** … 疎通／障害(フェイルオーバ)／性能／セキュリティ検証
5. **ドキュメント** … パラメータシート／試験成績書／運用手順書

## ✅ 完了条件（これができたら「ゴール達成」）
- [ ] 要件定義書・設計書一式（論理/物理構成図、IP設計、VLAN設計、ルーティング設計）が揃う
- [ ] containerlab で全体が起動し、全VLAN/拠点が疎通する
- [ ] **障害試験**: ①ISP-A断 ②コアSW1断 ③拠点間VPN断 のいずれでも自動迂回/復旧する
- [ ] 主要通信のセキュリティ（ACL/NAT/VPN暗号化）が意図通り効いている
- [ ] 試験成績書・運用手順書まで完成し、第三者が再構築できる状態
- [ ] **（最終形）** 3階層・マルチエリアOSPF・経路集約・MST で、大規模化しても破綻しない設計を図示・説明できる
